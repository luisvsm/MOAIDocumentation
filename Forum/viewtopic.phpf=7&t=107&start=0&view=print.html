Hi All,

I am well into developing a little game, that is coming along really well. I have built a level builder and now I am onto the general management of levels in the game. I have two main questions:

1. Whats the appropriate method for reseting the CpSpace?

2. Are there any objects in the API that need special considerations when de-allocating, de-referencing when restarting a level?

At the moment I have found around a 200KB leak per level to try and sort out, and I have it closely locked down to a handful of possibilities. Considering I am only allocating around 4 objects in the level which are physics based this should be easy to track down. Any idea's/method to help resolve this would be greatly appreciated.

Cheers,

Ok. Found the problem :)  Sorry to reply to self, but I think this is a worthwhile 'gotcha' to remember.

I needed to apply :stop() to the Space before releasing it (setting to nil, then doing a collect).

Obviously the update functions still held a reference and so the space wasnt properly being removed - this is a good thing, so now my levels all reload without memory issues.

Dave
