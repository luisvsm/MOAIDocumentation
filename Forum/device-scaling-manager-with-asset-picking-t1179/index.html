Device scaling manager with asset picking

by moopf » Thu Sep 06, 2012 4:46 pm

Hi,

I've been trying out Moai for the last day or so and figured I'd get started learning the system by putting together a class to manage scaling on viewports to make the process of doing so easier depending on different needs. So here it is - I hope this is the right board to post in (it seemed the most likely).

It will have bugs, it's not optimised, but hopefully it's useful to some even as a starting point or to understand Moai viewport scaling better (putting it together is how I've got to grips with it). Here's what it does:

  • Provides a class to create scaled viewports on arbitrary devices based on your preferences for scaling and optionally create a window using MOAISim
  • Allows you to scale to best fit the screen or do "pixel perfect" scaling (so the scale will always be integer multiples). It is not designed to scale in a way that would not keep your desired aspect ratio.
  • Pixel perfect scaling can also be optional based on how much the real scaling would be above an integer, e.g. anything under 2.2, scale to 2 etc. Take the iPad for instance. Basing your game height and width on an iPhone, without pixel perfect scaling would lead to less than great visuals. Setting this value to .2 (the class will use the lowest scale from width and height for the decision) will ensure that the iPad is scaled perfectly.
  • Allow the viewport width and height to be expanded (according to the scale) to fill the whole screen to avoid letterboxing.
  • Supports both simulation and window coordinates (you choose)
  • Provides convenience variables for accessing coordinates based on the coordinate mode you have picked, e.g. manager.view.topleft.x etc. And accessing scale etc.
  • Allows granular control over picking the best assets via. asset decisions, e.g. if the scale is above 1.9, use assets from 'x' directory and/or with 'y' extension.
  • It's class based as there should be no reason why you couldn't have multiple viewports with different settings (haven't tested this yet!) I was going to extend MOAIViewport originally but decided against it once I'd put the asset management in.
  • All the examples I came across used the iPhone retina dimensions as the default leading to wasteful loading of resources on smaller screens. This class won't do that, you can use 320 x 480 (or anything) as your base dimensions enabling you to load lo-res assets on smaller screens.

I've attached a zip of ScaleManager with a sample main.lua to this post. Unzip it into your Moai SDK samples directory and everything should be setup accordingly. I've documented the class to explain the functions but, not being a documentation writer, if you get stuck just post here and I'll help you out.

Any comments welcome. This is firmly based on my usual needs (especially pixel perfect scaling on devices where it makes sense to improve visuals) but now I've put it together I thought I may as well share it.

If any Moai experts see something I'm doing incorrectly, or not in the best way, do please let me know - I've only been playing with it for a day or so, so it's more than likely that I have. I know for most here this is probably basic stuff but it's been a good first step in learning Moai for me

Anyway, enjoy.

I ran into a little issue where the viewport size wasn't correctly being set on the device and this turned out to be caused by some changes in the MOAI API since you've made your class (specifically 'MOAIEnvironment.screenWidth' is now called 'MOAIEnvironment.horizontalResolution' & 'MOAIEnvironment.screenHeight' is now called 'MOAIEnvironment.verticalResolution').

I updated the code and it seems to be working ok (simple find and replace), deployed & tested on my iPhone4.
