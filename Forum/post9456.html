off-topic Rant:

Maybe you should give yourself a chance to try both Blender and Inkscape in a native Linux environment - in such circumstances, I think both apps are pretty neat, have a very clean and appropriate UI and work quite well. (Blender 2.65 has come a *loooong* way from the days of 2.5 .. which I agree, was a bit messy, aesthetics-wise.) Also, if you want to become a power Blender user, I can't recommend getting an iPad set up with something like "KeyPad" for iOS, which extends the Blender interface onto the iPad .. with such a setup, Blender productivity goes through the roof .. at least I find it very helpful to have it running.

But certainly MacOSX has a heritage UI from the 90's too, so I'm not sure I agree with your assertion there, as OSX has a huge legacy cachet, with single-bar menu's, icons, WIMP, etc. Although .. I say this as a long-time Linux/Irix user who only picked up a tiBook and subsequent Macbook addiction/dependency because it was so godamn sexy .. However, I'm leaving OSX, as I do not agree with the political moves of Apple and their sandbox lately, plus the latest MBP hardware release was quite disappointing, imho. I am biased and freely admit it.

Anyway this is all off-topic for MOAI, but I encourage you to try to use Blender and Inkscape in a Linux-Mint VM or something .. it may well be you'll see that the reason for your abhorrence of the GUI is, mostly, OSX+XQuartz, which is .. frankly .. *designed* to dissuade people from using X11-based apps .. Apple have freely admitted that they've crippled X11 apps on OSX in the past.

As for replacing Photoshop, I've had a great time with Pixelmator, which has a decidedly "chic" user interface and does a pretty good job of pushing pixels around. As far as replacing Illustrator, Inkscape has its thorns but for my needs it is an amazing tool. Of course, I have a massive Linux workstation to run it on, and yes .. it looks way better there, and functions much, much more smoothly than it does on OSX. It'll be nice if an OSX-native port eventually gets made, but in the meantime .. switch to Linux for these things!

/off-topic-rant.

Re: Building Box2D Worlds?

by Retrofit » Fri Mar 08, 2013 5:48 am

I'm nearly done with adding template support to level director and would like some help creating a basic template for Moai.
This will link will show you a list of all the available objects that you'll be able obtain from Level Director, let me know if there are any properties that you may need;
http://www.retrofitproductions.com/export-templates

I've made a start with the following template but I do not have any experience with Moai so you're help would be much appreciated.

Thanks

Code: Select all
  1.  

  2. <%LEVELS%>

  3.         MOAISim.openWindow ( %LEVEL%.CanvasWidth, %LEVEL%.CanvasHeight )

  4.        

  5.         viewport = MOAIViewport.new ()

  6.        

  7.         viewport:setSize ( %LEVEL%.CanvasWidth, %LEVEL%.CanvasHeight )

  8.         viewport:setScale ( %LEVEL%.CanvasWidth, %LEVEL%.CanvasHeight )

  9.  

  10.         layers = {}

  11.                

  12. <%LAYERS%>

  13.         -- create a layer and sets its viewport

  14.         layer['%LAYER%.Name'] = MOAILayer2D.new ()

  15.         layer['%LAYER%.Name']:setViewport ( viewport )

  16.         MOAISim.pushRenderPass ( layer['%LAYER%.Name'] )

  17.        

  18.         layers['%LAYER%.Name'].name = '%LAYER%.Name'

  19.         layers['%LAYER%.Name'].objects = {}

  20.  

  21. <%OBJECTS%>

  22.         layers['%LAYER%.Name'].objects['%OBJECT%.Name'] = MOAIGfxQuad2D.new()

  23.         layers['%LAYER%.Name'].objects['%OBJECT%.Name']:setTexture("%OBJECT%.File")

  24.         layers['%LAYER%.Name'].objects['%OBJECT%.Name']:setRect(%OBJECT%.X,%OBJECT%.X,%OBJECT%.Width,%OBJECT%.Height)

  25.  

  26.         prop = MOAIProp2D.new()

  27.         prop:setDeck(level.layers['%LAYER%.Name'].objects['%OBJECT%.Name'])

  28.         prop:setLoc(0,0)

  29.  

  30.         layers['%LAYER%.Name']:insertProp(prop)

  31.                

  32. </%OBJECTS%>   

  33. </%LAYERS%>

  34. </%LEVELS%>

  35.

Re: Building Box2D Worlds?

by Retrofit » Tue Mar 26, 2013 3:34 pm

I'm very please to announce the release of Level Director V2 (http://www.retrofitproductions.com/level-director)

It would be great if the MOAI community could start writing some MOAI export templates.
The first person to create a fully working template will get a free licence to Level Director.

Please check out the website for more information but here is the release notes for your information;


• Add support for texture packer sprite sheets saved as ‘Generic XML’. The add asset button now provides two options, add image (as before) and add texturepacker xml file.
• Added template export feature which is documented here Export-Templates.
• New checkbox ‘New file per level’ added for when using templates.
• Fixed fit to window on body editor for circle objects.
• Save last zoom value on level.
• Moved example projects to user local documents folder under Level Director to fix permission issues with Windows8.
• Canvas margins can now be changed per level, see level properties.
• Added the ability to specify indivdual frames to an animation sequence e.g. 1,3,4,5,8.
• Multiple levels/scenes can now be created. A new level dropdown has been added above the layers window with new buttons to allow you to add and delete a level/scene.
• 25% Zoom level added.
• Project tab added to tools-options which allows you to specify the export folder and file extension when using an export template.
• New corona export template created along with a LD_Loader and LD_Helper files to aid in the object creation and asset management. This helps separate the level data from the creation code. These can be found in the Utils folder.
• Added some templates to output data in a generic LUA format and also a prototype MOAI format.
• Reworked rotation code so the host rectangle is resized correctly allowing the rotated images to show in proportion.
• A general ‘Class’ property added to asset and object to allow you to group objects and assets easily.
• Show bodies on text objects now supported.
• Fixed issue when editing text where delete button would remove the object.
• New ‘Path’ object added which allows you to specify points along a line and optionally hook-up to physics.
• LD_Helper attaches a ‘followPath’ method to each object which allows you to specify which path to follow (see LD_Helper.lua)
• Parallax properties added to Level - ParallaxEnabled and ParallaxInfinite.
• Parallax properties added to Layer - ParallaxRepeated, ParallaxSpeedX and ParallaxSpeedY
• LD_Helper will check to see if ParallaxEnabled and attaches two new methods ‘move’ and ‘moveCamera’ to the level object. moveCamera only moves layers so object wrapping not supported in this mode.
• X Flip property added to Image objects.
• Y Flip property added to image objects.
• EnableDrag property added, when set to true and used with LD_Helper a touch event will be added to the object.
• New V2Demo example project added to demo v2 features.
• *NOTE: The V2Demo example uses the CoronaTemplate export whereas all the other examples use the built-in corona export. So if you intend to make changes be sure to set the correct export method othewise the main.lua will not work correctly. Alternatively you could change the main.lua to work with the corresponding export. Please see the example project documentation on the website.
Retrofit
