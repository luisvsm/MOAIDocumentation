Hi complete noob at using Moai, having been introduced to Lua via the excellent CODEA IOS app.

So I have been mainly playing around with box2d in MOAI and have noticed it is not possible to set the dampingRatio or the frequency on a weld joint. This is possible in CODEA and I found in Corona docs this got introduced in box2d 2.2.1

http://docs.coronalabs.com/api/type/Joint/index.html

By playing around with these values it is possible to create springy structures, so for example a diving board is just a springy weld joint at the end of a box connected to a static object. It also allows for sagging bridges and so is pretty useful.

I would appreciate any feedback on how to work around this.

Re: box2d weldjoint frequency and damping ratio not supporte

by stun » Tue Jun 04, 2013 4:39 am

I'm not sure you'll be able to work around it. However the good news is that it's not hard to expose additional Box2D features through the MOAI layer to lua. It's also a good way to learn a little more about the framework underneath. Various people have done the same for various other missing pieces of Box2D functionality. Things like raycasting which is also missing, for example.

The MOAI Box2d classes are fairly simple wrappers for the Box2D code underneath. Take a look, hopefully you won't find it hard to extend for the functionality you want.
