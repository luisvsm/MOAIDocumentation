Re: Rotating Camera with a CamerFitter2D

by ezraanderson » Mon Apr 29, 2013 8:39 am

It should be pretty easy to figure it out by setting the parent of an object that reacts to another object

try this
camera:setParent(g_hero['prop'])
or
camera:setAttrLink ( MOAIProp.INHERIT_TRANSFORM, g_hero['prop'], MOAIProp.TRANSFORM_TRAIT )

or you could rotate the whole viewport
viewport:setRotation(-45)

I also read you could rotate layers but i haven't tried it, However i think it was broken in 1.4 build.

That's seams odd. I used a fitter for my tilemap, and the camera attached to the fitter and a main_hero. (i used an anchor before but i scrapped it, can't remember why). I do remember banging my head against the wall trying to do this.

Here are parts of my code.

--***************************************************************
--this is in my load camera function
Code: Select all
  1.  

  2.  

  3.  --******************************************

  4. --GLOBAL camera attached to fitter and hero

  5. camera = MOAICamera2D.new ()

  6.  

  7. --**********************

  8. --camera to tilemap layer

  9. myLayer:setCamera ( camera )

  10.  

  11.                        

  12. fitter = MOAICameraFitter2D.new ()

  13. fitter:setViewport ( viewportB ) --tilemap is in this viewport

  14. fitter:setCamera ( camera )

  15. fitter:setBounds (480*-1, 320*-1, 0, 0 ) --change this to different bounds for different scalling

  16. fitter:start()

  17.  

  18.  



--***************************************************************
--this is in my load hero function
Code: Select all
  1.  

  2.  

  3. --my prop rotates, if i want the tilemap to rotate i set this

  4. --camera:setParent(g_hero['prop'])

  5.  

  6. --my main hero is attached to a box2d body for movement,

  7. camera:setParent(g_hero['body'])

  8.  

  9. --to rotate the tilemap

  10. g_hero['body']:setAngularVelocity(math.rad(180))

  11.  

  12. --to move the tilemap

  13. g_hero['body']:setLinearVelocity(1,1)

  14.  

  15.  



--***************************************************************
--this is in my zoom function
Code: Select all
  1.  

  2. --**********************************************

  3. --make world to screen

  4. function F._func_zoom(int)

  5.  

  6. g_zoomCnt =     g_zoomCnt + int

  7.                  

  8. if g_zoomCnt == 4 then g_zoomCnt = 1 end

  9. if g_zoomCnt == 0 then g_zoomCnt  = 3 end

  10.                                  

  11. local width,height = 0

  12.                  

  13. if      g_zoomCnt == 1 then  width,height = 480,320

  14. elseif g_zoomCnt == 2 then  width,height = 480*2,320*2

  15. elseif g_zoomCnt == 3 then  width,height = 480*4,320*4

  16. -- elseif g_zoomCnt == 4 then  width,height = displaySetup['width'],displaySetup['height']

  17.  end

  18.                  

  19. zoomHW = {width,height}                    

  20. fitter:setBounds (width*-1, height*-1, 0, 0 )  

  21.  

  22. print('zoomCnt',g_zoomCnt,width,height)

  23.  

  24. end

  25.  

  26.  

  27.
