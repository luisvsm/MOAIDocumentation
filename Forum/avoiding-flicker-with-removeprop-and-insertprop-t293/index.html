Avoiding flicker with removeProp and insertProp

by patrick » Wed Dec 28, 2011 5:42 pm

OK, got it. If you are building from source, go into MOAISim.cpp and find MOAISim::Update (). At the bottom of that function I'm dispatching the http and data IO events:

MOAIUrlMgr::Get ().Process ();

this->mDataIOThread.Publish ();

To fix the flicker, just move those to the top of the function. Put them just after this line:

double interval = this->MeasureFrameRate ();

That should do it. The problem was, the http events were generating a callbacks *after* the dep graph update but *before* the draw. So the new textbox dep node wasn't getting updated until the following sim step.

Thanks for finding the bug and sending me the test script. It made tracking it down a snap. The fix will be in the next SDK release.
