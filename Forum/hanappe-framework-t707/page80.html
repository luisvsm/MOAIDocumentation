@Quedex

The TMXMapView of Hanappe, drawing layer is always divided.
This had disadvantages some performance basis.

TileMap of Flower was change in the Group instead of Layer.
Therefore, it is possible to arbitrarily set the drawing layer.

When using a single Layer, and excellent performance.
However, circumstances might be worse in some cases.
In this case, can also be modified to set a plurality of Layer.


TileLayer, TileObjectGroup is inherited classes of mere Group.
It is possible to add another object to the Group.

Code: Select all
  1.  

  2. image = flower.Image("cathead.png")

  3.  

  4. tileLayer = tileMap.mapLayers[1]

  5. tileLayer:addChild(image)

  6.  

  7. tileMap:updateRenderOrder()

  8. -- Drawing order, it is possible to arbitrarily set the Priority

  9.  



In this case, you must manage the objects they create.
I should create a TileMap of the game for you.

Code: Select all
  1.  

  2. GameMap = class(TileMap)

  3. ...

  4.  



In addition, it is as follows: If you want to generate a TileObject normally.

Code: Select all
  1.  

  2. local objectGroup = tileMap:findMapLayerByName("Object")

  3. local object = objectGroup:createObject({

  4.           name = "Enemy",

  5.           type = "Actor",

  6.           shape = "rectangle",

  7.           x = 192,

  8.           y = 448,

  9.           width = 0,

  10.           height = 0,

  11.           gid = 538,

  12.           visible = true,

  13.           properties = {

  14.             ["playAnim"] = "walkDown"

  15.           }

  16.         })

  17.

by makotok1123 » Mon May 13, 2013 5:21 pm

@n2oula

Widget must use the UIView.
This is for automating the configuration of the touch event processing and focus.

Code: Select all
  1.  

  2. function onCreate(e)

  3.     view = flower.UIView {

  4.         scene = scene,

  5.     }

  6.  

  7.     textbox1 = widget.TextBox {

  8.         size = {18*x, 9*y},

  9.         pos = {31*x, 25*y + (300/32)*y*i},

  10.         text = v,

  11.         textSize = 12,

  12.         parent = view,

  13.         -- backgroundTexture = "assets/skins/panelNew.png",

  14.     }

  15. end

  16.  



* When using the Widget, it must be the root UIView.
* If you want to get the Layer, to get from UIView.

@n2oula

Do you thing it with any name, we might want to get later?
If you do not want to use the group, who yourself have managed a good idea.

In general, it will be easier to create the map.

Code: Select all
  1.  

  2. local image = flower.Image("moai.png")

  3. image:setLayer(layer)

  4. images["a1"] = image

  5.  

  6. a1 = images["a1"]

  7.  



When using the Group, can be retrieved by name.

Code: Select all
  1.  

  2. local group = flower.Group()

  3. group:setLayer(layer)

  4.  

  5. local image = flower.Image("moai.png")

  6. image.name = "a1"

  7. group:addChild(image)

  8.  

  9. local a1 = group:getChildByName("a1")

  10.
