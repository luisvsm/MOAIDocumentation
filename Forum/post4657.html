by nosheet » Sat Jun 16, 2012 5:42 pm

@sgeos, unfortunately i am not sure i know how to do such an intense vector graphics, so that is probably not on the road right now.
the way i have been using illustrator and svg so far is to lay out scenes (such as menu, credits...) and game levels. So the workflow I plan to accomplish is like this:
- use illustrator to lay out scene ( gui layout or a game level)
- supported object types: rectangle, circle, polygon (no bezier so far, maybe later), text and images(texture atlases and spritesheets will be supported)
- use illustrator layers to group objects
- write down data for each object in Svg Interactivity panel. This data can be object's name, physics properties, stylesheet image path, animation sequence, as well as custom functions like "path:@line2 speed:5" which would mean that object will follow path with name line2 with speed of 5px per second...

The reason this is developing slowly is that I want to make this svg importer an integral part of a simple entity-component-system that would allow to define in svg not only how the objects are displayed but also how they behave and interact with each other.
