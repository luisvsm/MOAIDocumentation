by josh h » Fri Jun 28, 2013 12:29 pm

Thanks all for sharing your shaders. I've just started trying to get a few shaders working in Moai. After realizing that Moai's matrices are the transpose of what you find in most other examples I've gotten a couple working. Thanks to the comments in @nosheets example, I realized I had to swap from matrix * point to point * matrix.

Seems like the one thing that is missing in the shaders is the modelView matrix. Moai gives you MOAIShader.UNIFORM_WORLD is the model matrix, UNIFORM_VIEW_PROJ which is view * projection and UNIFORM_WORLD_VIEW_PROJ which is model * view * projection. For lighting shaders, I need model * view to get from model space to eye space since you usually do your lighting calculations in eyeSpace.

Re: 3D Shaders: Let's Share Code Here

by josh h » Tue Jul 16, 2013 8:57 pm

Looking at the source code, MOIA maintains 3 matrices: world, view and projection. From what I can tell world is the cumulative transform on the prop, view is the camera transform and projection is the usual projection matrix. So the equivalent of the usual modelview matrix in OpenGL programs would combine world and view.

Here is a snippet from MOAIShader::bindPipelineTransforms where you can see that UNIFORM_WORLD is just the world matrix without the view but UNIFORM_WORLD_VIEW_PROJ combines all three:

[code=C++]case UNIFORM_WORLD: {

this->SetValue ( world );
this->Bind ();
break;
}
case UNIFORM_WORLD_VIEW_PROJ: {

ZLMatrix4x4 mtx = world;
mtx.Append ( view );
mtx.Append ( proj );
this->SetValue ( mtx );
this->Bind ();
break;
}
[/code]

In a lot of scenes, there is no camera or the camera has only translation and if you are only doing directional lighting then it doesn't matter. I ran into it trying to do per-pixel lighting in eye coordinates with a point light source where the camera transform is important.

I added a new UNIFORM_WORLD_VIEW to my fork that gives the "real" modelview matrix which is at github.com/jhandley/moai-dev/

Code: Select all

  3. local vertexShader = [[

  4.  

  5.  

  6.  

  7. varying vec2 uv;

  8.  

  9. void main(void)

  10. {

  11.     gl_Position = vec4( gl_Vertex.xy, 0.0, 1.0 );

  12.     gl_Position = sign( gl_Position );

  14. }

  17.  

  18.  

  19.  

  20.  

  21.  

  22. local fragmentShader = [[

  23.  

  24.  

  25. uniform sampler2D tex;

  26. varying vec2 uv;

  27. const float sampleDist = 1.0;

  28. const float sampleStrength = 2.2;

  29.  

  30. void main(void)

  31. {

  32.                     float samples[10];

  33.                     samples[0] = -0.08;

  34.                     samples[1] = -0.05;

  35.                     samples[2] = -0.03;

  36.                     samples[3] = -0.02;

  37.                     samples[4] = -0.01;

  38.                     samples[5] =  0.01;

  39.                     samples[6] =  0.02;

  40.                     samples[7] =  0.03;

  41.                     samples[8] =  0.05;

  42.                     samples[9] =  0.08;

  43.                

  44.                     vec2 dir    = 0.5 - uv;

  45.                     float dist  = sqrt(dir.x*dir.x + dir.y*dir.y);

  46.                     dir                 = dir/dist;

  47.                

  48.                     vec4 color = texture2D(tex,uv);

  49.                     vec4 sum = color;

  50.                

  51.                     for (int i = 0; i < 10; i++)

  52.                         sum += texture2D( tex, uv + dir * samples[i] * sampleDist );

  53.                

  54.                     sum *= 1.0/11.0;

  55.                     float t = dist * sampleStrength;

  56.                     t = clamp( t ,0.0,1.0);

  57.                

  58.                     gl_FragColor = mix( color, sum, t );

  59. }

So I got hold of the brilliant and very helpful nosheet on email and he pointed me in the right direction with the calculations and troubleshooting info. With that I was able to finish up the dynamic point+ambient lighting shader I wanted for a 2D game and here are the results:



Apparently it's a lot like the lighting system used by Broken Age. Here's the final shader function if anyone else is interested. It's adapted to plug and play directly for props and quads rather than meshes, so it works nicely out of the box with frameworks like RapaNui and Hanappe:

Code: Select all
  1.  

  2. function point_lighting_shader(lightx, lighty, lightz, intensity, range, red, green, blue, am_red, am_green, am_blue)  

  3.   local vsh = [[  

  4.     attribute vec4 v_pos;

  5.     attribute vec2 v_uv;

  6.     attribute vec4 v_col;

  7.     // Note: NO NORMALS!

  8.  

  9.     // UNIFORMS

  10.     // X, Y and Z position of the point light.

  11.     uniform float light_x, light_y, light_z;

  12.     // Light range is a distance at which the light intensity will decay to zero.

  13.     uniform float light_intensity, light_range;

  14.     // Point light color

  15.     uniform float light_red, light_green, light_blue;

  16.     // Ambient light color

  17.     uniform float ambient_red, ambient_green, ambient_blue;

  18.  

  19.     // VARYINGS

  20.     varying vec4 frag_color;

  21.     varying vec2 frag_uv;

  22.  

  23.     void main () {

  24.       //vec4 vertex_position = v_pos

  25.    

  26.       // Find the distance between the vertex and the light source

  27.       float vertex_dist = length (vec3( light_x, light_y, light_z ) - vec3( v_pos.x, v_pos.y, v_pos.z ));

  28.  

  29.       // Decay light factor depending on the vertex distance from light source

  30.       float decay = max( 0.0, (light_range - vertex_dist )/ light_range );

  31.  

  32.       // Color of the point light

  33.       vec4 light_color = vec4( light_red, light_green, light_blue, 1.0 );

  34.  

  35.       // Color of the ambient light

  36.       vec4 ambient_color = vec4( ambient_red, ambient_green, ambient_blue, 1.0 );

  37.       frag_color = (v_col * light_color * light_intensity * decay) + ambient_color;

  38.      

  39.       // Alpha channel

  40.       frag_color.w = v_col.w;

  41.  

  42.       // Output UV

  43.       frag_uv = v_uv;

  44.      

  45.       // Multiply vertex position with projection matrix to get its on-screen position.

  46.       gl_Position = v_pos;

  47.     }

  48.   ]]

  49.  

  50.   local fsh = [[

  51.     // UNIFORMS

  52.     uniform sampler2D map_diffuse;

  53.  

  54.     // VARYINGS

  55.     varying LOWP vec4 frag_color;

  56.     varying MEDP vec2 frag_uv;

  57.  

  58.     void main() {

  59.       gl_FragColor = frag_color * ( texture2D( map_diffuse, frag_uv ) );

  60.     }

  61.   ]]

  62.    

  63.   local shader = MOAIShader.new ()

  64.   shader:reserveUniforms ( 11 )

  65.  

  66.   shader:declareUniformFloat( 1, 'light_x', lightx )

  67.   shader:declareUniformFloat( 2, 'light_y', lighty )

  68.   shader:declareUniformFloat( 3, 'light_z', lightz )

  69.   shader:declareUniformFloat( 4, 'light_intensity', intensity )

  70.   shader:declareUniformFloat( 5, 'light_range', range )

  71.   shader:declareUniformFloat( 6, 'light_red', red )

  72.   shader:declareUniformFloat( 7, 'light_green', green )

  73.   shader:declareUniformFloat( 8, 'light_blue', blue )

  74.   shader:declareUniformFloat( 9, 'ambient_red', am_red )

  75.   shader:declareUniformFloat( 10, 'ambient_green', am_green )

  76.   shader:declareUniformFloat( 11, 'ambient_blue', am_blue )

  77.  

  78.   shader:setVertexAttribute( 1, 'v_pos' )

  79.   shader:setVertexAttribute( 2, 'v_uv' )

  80.   shader:setVertexAttribute( 3, 'v_col' )

  81.  

  82.   shader:load ( vsh, fsh )

  83.   return shader

  84. end

  85.  



Since it's wrapped in a function, it's just a simple couple of lines in your main lua code instead of having to set attributes every time.

To make a flat shadow of a sprite on the floor, that seems to be "skewed" according to the light direction, you create a regular sprite (quad), assign the same texture/sprite animation you do for the main sprite and then:
- Let's call a shadow prop "my_shadow" in the further text.
- First you need to deform it. Use my_shadow:setShearByX(y,z) method that distorts the sprite the way you need it. Some experimentation with right values is needed but it works great.
- my_shadow:setColor(0,0,0,1). I didn't test if this is the correct value but I assume this will set all opaque pixels to black. Instead of 1.0 for alpha you can use other values, to get semi-transparent shadows.
- my_shadow:setBlendMode( MOAIProp. BLEND_MULTIPLY). In case you have semi-transparent shadow, this line will apply multiply blend that will darken the pixels below it.


Second example is also done without shaders, at least the way it looks. I see the effect is made using 3 quads with 3 radial gradient textures in red, blue and green. He set these quads to MOAIProp.BLEND_ADD so that when they overlap, they add color values and that's why their mutual intersection is white.

Both of things can be done with shaders, too, but for the sake of simplicity you might want the shader-less approach first. Let me know if

I tested it in my game :

and went with the following parameters :

Code: Select all
  1.  

  2. -- Specific to the sprites in use (64 pixels)

  3. FrameLayer.shadowXOffset = 18

  4. FrameLayer.shadowYOffset = -1

  5. FrameLayer.shadowScaleX = 0.9

  6. FrameLayer.shearFactor = 0.5

  7.  

  8. function FrameLayer:makeShadow()

  9.     local facing = self.facing -- change facing to mirror sprite and shadow to get other direction

  10.     local x,y,z = self:getLoc()

  11.     self:setShearByY(self.facing*FrameLayer.shearFactor,0)

  12.     self.scaleX = FrameLayer.shadowScaleX

  13.     self:setScl(self.scaleX*facing, 1,1)

  14.     self:setBlendMode(MOAIProp2D.GL_SRC_ALPHA,MOAIProp2D.GL_ONE_MINUS_SRC_ALPHA)

  15.     self:setColor(0,0,0,0.3)

  16.     self.isShadow = true

  17.     self:setLoc(FrameLayer.shadowXOffset, FrameLayer.shadowYOffset-facing, z)

  18. end
