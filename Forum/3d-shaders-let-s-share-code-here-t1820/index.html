3D Shaders: Let's Share Code Here

by nosheet » Thu May 02, 2013 7:33 pm

Hi
I started fooling around with 3D shaders, and thought it would be a good idea to make
a place where we can share some shader-related code, we can polish stuff together and eventually
include in Moai. Adding shaders that would handle effects such as lighting, reflections, etc... would greatly improve 3D support for Moai.

I'll start with a very simple vertex-lighting shader I made, that supports texture
coordinates and vertex colors. Here's a screenshot that show the results:



The model is light by somewhat diagonally positioned directional light (light direction
is hardcoded in vertex shader).
(note that the gradient that darkens vertical faces is made by vertex colors, and
is not the result of the shader).

Here's the code:
EDIT:Added precision in .fsh file, and control via attributes
------------------------------------------------------------------------------------------------
Vertex Format, we need to declare:
- vertex position (x,y,z)
- texture coordinates (u,v)
- color ( r,g,b,a )
- vertex normal (x,y,z)
Code: Select all
  1.  

  2. ...

  3. local vertex_format = MOAIVertexFormat.new()

  4.  -- vertex position: x, y, z:

  5. vertex_format:declareCoord ( 1, MOAIVertexFormat.GL_FLOAT, 3 )

  6. -- texture coordinates: u, v

  7. vertex_format:declareUV ( 2, MOAIVertexFormat.GL_FLOAT, 2 )

  8. -- color (array of 4 floats)

  9. vertex_format:declareColor (3, MOAIVertexFormat.GL_UNSIGNED_BYTE )

  10. -- vertex normal: x, y, z

  11. vertex_format:declareNormal( 4, MOAIVertexFormat.GL_FLOAT, 3 )  

  12.     ...

  13.  



------------------------------------------------------------------------------------------------
Shader creating in Lua (incomplete .lua code):
Code: Select all
  1. local shader = MOAIShader.new ()

  2.  

  3. -- loadShader function that loads .vsh and .fsh omitted for brevity

  4. local vsh, fsh = loadShader()

  5.  

  6. shader:reserveUniforms ( 7 )

  7.  

  8. shader:declareUniform( 1, 'mv_matrix', MOAIShader.UNIFORM_WORLD )

  9. shader:declareUniform( 2, 'view_proj_matrix', MOAIShader.UNIFORM_VIEW_PROJ )

  10. shader:declareUniformFloat( 3, 'light_x', MOAIShader.UNIFORM_FLOAT )

  11. shader:declareUniformFloat( 4, 'light_y', MOAIShader.UNIFORM_FLOAT )

  12. shader:declareUniformFloat( 5, 'light_z', MOAIShader.UNIFORM_FLOAT )

  13. shader:declareUniformFloat( 6, 'backlit_brightness', MOAIShader.UNIFORM_FLOAT )

  14. shader:declareUniformSampler ( 7, 'tex_diffuse', 1 )

  15.  

  16. shader:setVertexAttribute( 1, 'position' )

  17. shader:setVertexAttribute( 2, 'uv' )

  18. shader:setVertexAttribute( 3, 'color' )

  19. shader:setVertexAttribute( 4, 'normal' )

  20.  

  21. -- Store attributes for later use

  22. shader.ATTR_LIGHT_DIR_X = 3

  23. shader.ATTR_LIGHT_DIR_Y = 4

  24. shader.ATTR_LIGHT_DIR_Z = 5

  25. shader.ATTR_BACK_BRIGHTNESS = 6

  26.  

  27. shader:load ( vsh, fsh )

  28.  

  29.  



------------------------------------------------------------------------------------------------
How to set up light direction and brightness of the faces facing away from light
Code: Select all
  1.  

  2. -- Set light direction

  3. shader:setAttr( shader.ATTR_LIGHT_DIR_X, .5 )

  4. shader:setAttr( shader.ATTR_LIGHT_DIR_Y, .3 )

  5. shader:setAttr( shader.ATTR_LIGHT_DIR_Z, .9 )

  6. -- Set the lightness of the object facing away the light

  7. shader:setAttr( shader.ATTR_BACK_BRIGHTNESS, .5 )

  8.  


Back brightness affect the object faces that face away the light.
Set this to 0.0 to have them completely black, or 1.0 if you want the back
faces to be lit as much as their opposites (experiment for best results)

------------------------------------------------------------------------------------------------
Vertex shader file
Code: Select all
  1.  

  2.  

  3. uniform mat4 mv_matrix;                         // Sum of all MOAITransforms on object

  4. uniform mat4 view_proj_matrix;          // Projection matrix (camera)

  5. uniform float light_x, light_y, light_z, backlit_brightness;

  6.  

  7. attribute vec4 position;

  8. attribute vec2 uv;

  9. attribute vec4 color, normal;

  10.  

  11. varying vec2 frag_uv;

  12. varying vec4 frag_color;

  13.  

  14.  

  15. void main(void)  

  16. {    

  17.  

  18.         // Find the real vertex position (mv_matrix is transform applied to object)

  19.         vec4 vertex_position = position * mv_matrix;

  20.         //vertex_position.z = 0.0;      // For example, flatten all to z = 0

  21.  

  22.  

  23.         // Find the vertex normal after applying transform

  24.         vec3 eye_space_normal = normalize( vec3( vec4( normal.x, normal.y, normal.z, 0.0 ) * mv_matrix ));

  25.  

  26.         // IMPORTANT:

  27.         // Don't forget to always order in multiply operations:

  28.         // vector * matrix --> GOOD

  29.         // matrix * vector --> BAD

  30.  

  31.         // Light direction from attributes (normalized)

  32.         vec3 light_dir = normalize( vec3(light_x, light_y, light_z) );

  33.  

  34.         // Angle between light and normal

  35.         float light_factor = dot( light_dir, eye_space_normal );

  36.         // If facing away from light, don't make it black,

  37.         // instead still make some shading, only much darker than the

  38.         // opposite faces (the ones that face the light)

  39.         if (light_factor<0.0)

  40.                 light_factor = -backlit_brightness * light_factor;

  41.  

  42.         // Multiply vertex position with projection matrix to get its on-screen position.

  43.         gl_Position = vertex_position * view_proj_matrix;

  44.  

  45.         /* Always output color */

  46.         frag_color = light_factor * color;

  47.         frag_color.w = 1.0;

  48.  

  49.         /* Pass UV */

  50.         frag_uv = uv;

  51. }

  52.  




------------------------------------------------------------------------------------------------
Fragment shader file content
Code: Select all
  1.  

  2.  

  3. varying LOWP vec4 frag_color;

  4. varying MEDP vec2 frag_uv;

  5.  

  6. uniform sampler2D sampler;

  7.  

  8. void main()

  9. {

  10.     gl_FragColor = frag_color * texture2D( sampler, frag_uv );

  11. }

  12.

NEW SHADER: FAKE FRESNEL REFLECTIVE SHADER

Hey, I made another shader. It's a very quick and very dirty shader that adds fresnel reflection to objects (such as glass or wood). Here's a screenshot:


When I say "dirty" I really mean it. First, it's a vertex-based shader, to
improve the performance. Also since Moai doesn't support CUBE MAPs,
this shader is made to (very) approximately use regular 2d map (as with any Moai texture).

Now for the details, and actual shader:




------------------------------------------------------------------------
Very Important: SHADER REQUIREMENTS
1) MULTITEXTURE
Shader requires that mesh is assigned MOAIMultiTexture,
one texture for diffuse color, and other will be reflection map
(the stuff seen in reflection).
Multitexture:
1 = Diffuse Map
2 = Reflection Map

Note:
You can control reflection amount using alpha channel of the diffuse map.
Please note in the screenshot that the parts of texture that are "glass" are
reflective while white metal grid is not.
White pixels in diffuse map's alpha channel will be fully reflective,
and black will not reflect anything. Note that alpha channel will
not affect opacity!

Make a transparent .png for diffuse map to use this function (make sure
to use MOAIImage.TRUECOLOR when loading, to avoid premultiplying alpha ):
Code: Select all
  1. texture:load( "my_texture.png", MOAIImage.TRUECOLOR )





2) CAMERA LOCATION
Camera location x,y and z coordinates must be set to shader attributes
(see details below) every time the camera moves (ideally on every frame)

------------------------------------------------------------------------
Vertex Format
Code: Select all
  1.  

  2. ...

  3.  

  4. local vertex_format = MOAIVertexFormat.new()

  5.  -- vertex position: x, y, z:

  6. vertex_format:declareCoord ( 1, MOAIVertexFormat.GL_FLOAT, 3 )

  7. -- texture coordinates: u, v

  8. vertex_format:declareUV ( 2, MOAIVertexFormat.GL_FLOAT, 2 )

  9. -- color (array of 4 floats )

  10. vertex_format:declareColor (3, MOAIVertexFormat.GL_UNSIGNED_BYTE )

  11. -- vertex normal: x, y, z

  12. vertex_format:declareNormal( 4, MOAIVertexFormat.GL_FLOAT, 3 )  

  13. ...

  14.  




------------------------------------------------------------------------
Create Shader
Code: Select all
  1.  

  2. local shader = MOAIShader.new ()

  3. local vsh, fsh = loadShader( 'sh_vert_reflection' )

  4.  

  5. shader:reserveUniforms ( 8 )

  6.  

  7. shader:declareUniform( 1, 'mv_matrix', MOAIShader.UNIFORM_WORLD )

  8. shader:declareUniform( 2, 'view_proj_matrix', MOAIShader.UNIFORM_VIEW_PROJ )

  9. shader:declareUniformFloat( 3, 'cam_x', MOAIShader.UNIFORM_FLOAT )

  10. shader:declareUniformFloat( 4, 'cam_y', MOAIShader.UNIFORM_FLOAT )

  11. shader:declareUniformFloat( 5, 'cam_z', MOAIShader.UNIFORM_FLOAT )

  12. shader:declareUniformSampler ( 6, 'tex_diffuse', 1 )

  13. shader:declareUniformSampler ( 7, 'tex_reflect', 2 )

  14. shader:declareUniformFloat( 8, 'reflection_amount', MOAIShader.UNIFORM_FLOAT )

  15.  

  16. shader:setVertexAttribute( 1, 'position' )

  17. shader:setVertexAttribute( 2, 'uv' )

  18. shader:setVertexAttribute( 3, 'color' )

  19. shader:setVertexAttribute( 4, 'normal' )

  20.  

  21. -- Store attributes

  22. shader.ATTR_CAMERA_X = 3

  23. shader.ATTR_CAMERA_Y = 4

  24. shader.ATTR_CAMERA_Z = 5

  25. shader.ATTR_REFLECTION_AMOUNT = 8

  26.  

  27. shader:load ( vsh, fsh )

  28.  




------------------------------------------------------------------------
Set Camera Position

Following code must be executed every time the camera move
(or on every frame):

Code: Select all
  1.  

  2. local cx, cy, cz = camera:getLoc()

  3. shader:setAttr( shader.ATTR_CAMERA_X, cx )

  4. shader:setAttr( shader.ATTR_CAMERA_Y, cy )

  5. shader:setAttr( shader.ATTR_CAMERA_Z, cz )

  6.  


Set Reflection Amount
Code: Select all
  1.  

  2. shader:setAttr( shader.ATTR_REFLECTION_AMOUNT, 1. )

  3.  





------------------------------------------------------------------------
Vertex shader
Code: Select all
  1.  

  2. /*

  3.  

  4. VERTEX REFLECTION SHADER

  5. ----------------------

  6. Quick lighting shader.

  7. Light direction is hard-coded;

  8. */

  9.  

  10. uniform mat4 mv_matrix;                         // Sum of all MOAITransforms on object

  11. uniform mat4 view_proj_matrix;          // Projection matrix (camera)

  12.  

  13. uniform float cam_x, cam_y, cam_z;

  14.  

  15. attribute vec4 position;

  16. attribute vec2 uv;

  17. attribute vec4 color, normal;

  18.  

  19. varying vec2 frag_uv;

  20. varying vec4 frag_color;

  21. varying float fresnel_factor;

  22. varying vec2 reflection_uv;

  23.  

  24.  

  25. void main(void)  

  26. {    

  27.  

  28.         // Find the real vertex position (mv_matrix is transform applied to object)

  29.         vec4 vertex_position = position * mv_matrix;

  30.         //vertex_position.z = 0.0;      // For example, flatten all to z = 0

  31.  

  32.         // Find the vertex normal after applying transform

  33.         vec3 normal_with_transforms = normalize( vec3( vec4( normal.x, normal.y, normal.z, 0.0 ) * mv_matrix ) );

  34.         //vec3 camera_position = normalize( vec3( vec4( cam_x, cam_y, cam_z, 0.0 ) * mv_matrix ));

  35.         vec3 camera_position = vec3( cam_x, cam_y, cam_z );

  36.         vec3 eye_direction = normalize( vec3( vertex_position.x, vertex_position.y, vertex_position.z ) - camera_position );

  37.  

  38.         // Angle between light and normal

  39.         fresnel_factor = -(dot( normal_with_transforms, eye_direction ));

  40.         fresnel_factor = pow( 1.0 - fresnel_factor, 2.0);

  41.  

  42.         // Reflection vector

  43.         vec3 reflection_vector = normalize( reflect( eye_direction, normal_with_transforms ));

  44.  

  45.  

  46.         // Multiply vertex position with projection matrix to get its on-screen position.

  47.         gl_Position = vertex_position * view_proj_matrix;

  48.  

  49.         // Output color

  50.         frag_color = color;

  51.  

  52.         // Output texture coordinates

  53.         frag_uv = uv;

  54.  

  55.         reflection_uv = vec2( .5 * (1. + reflection_vector.x ), .5 * (1. - reflection_vector.z ) );

  56. }

  57.  






------------------------------------------------------------------------
Fragment shader
Code: Select all
  1.  

  2. varying LOWP vec4 frag_color;

  3. varying MEDP vec2 frag_uv;

  4. varying MEDP float fresnel_factor;

  5. varying MEDP vec2 reflection_uv;

  6.  

  7. uniform sampler2D tex_diffuse;

  8. uniform sampler2D tex_reflect;

  9. uniform MEDP float reflection_amount;

  10.  

  11. void main()

  12. {

  13.  

  14.         LOWP vec4 diff_color =  frag_color * texture2D( tex_diffuse, frag_uv );

  15.         LOWP vec4 refl_contr = texture2D( tex_reflect, reflection_uv );

  16.     gl_FragColor = ( 1.0 - reflection_amount * fresnel_factor ) * diff_color

  17.                                 + reflection_amount * fresnel_factor * diff_color.w * refl_contr;

  18.     gl_FragColor.w = 1.;

  19. }

  20.

@Rhoster,
I see now! I think that in first case, if it's a 2D shadow in a top/down like view, then you don't really need shaders.
To make a flat shadow of a sprite on the floor, that seems to be "skewed" according to the light direction, you create a regular sprite (quad), assign the same texture/sprite animation you do for the main sprite and then:
- Let's call a shadow prop "my_shadow" in the further text.
- First you need to deform it. Use my_shadow:setShearByX(y,z) method that distorts the sprite the way you need it. Some experimentation with right values is needed but it works great.
- my_shadow:setColor(0,0,0,1). I didn't test if this is the correct value but I assume this will set all opaque pixels to black. Instead of 1.0 for alpha you can use other values, to get semi-transparent shadows.
- my_shadow:setBlendMode( MOAIProp. BLEND_MULTIPLY). In case you have semi-transparent shadow, this line will apply multiply blend that will darken the pixels below it.


Second example is also done without shaders, at least the way it looks. I see the effect is made using 3 quads with 3 radial gradient textures in red, blue and green. He set these quads to MOAIProp.BLEND_ADD so that when they overlap, they add color values and that's why their mutual intersection is white.

Both of things can be done with shaders, too, but for the sake of simplicity you might want the shader-less approach first. Let me know if

@scottwb, @ibisum, also have in mind that having a 2D background with lighting done through fragment shader (per-pixel calculation) is an order of magnitude slower than having a real 3D background with vertex shader lighting (calculated per vertex). As per my testing on device, having a full screen (especially hi-res retina screens) processed through a fragment shader, even the simplest one (!) will easily drop the fps into 20-30fps area (based on my test on iPad 3). So be careful what you wish for, hehe

@JDance, thanks, yep I'm also just learning shader, and lack of any debug possibility (except for black screen) is really a problem. But the real world of weirdness starts when you Google search "shader optimisation on mobile" or similar terms, as you can see that even differences between:
Code: Select all
  1.  

  2. if condition

  3.    continue;

  4.  


and
Code: Select all
  1.  

  2. if not condition

  3.    do_some_stuff;

  4.  


makes a substantial difference in performance!
(for a real-world example of unexpected performance gains/losses see my comment on Seebs' question about Shader inputs)
