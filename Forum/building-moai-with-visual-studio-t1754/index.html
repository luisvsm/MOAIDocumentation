by ezraanderson » Sun Mar 31, 2013 10:24 am

1. clean solution
2. if you have changed any of the file structure it will break ie:(moai folder)\bin\win32 must be accessible
3. check project properties
project>moai properties> linker> general> addition linker libraries
project>moai properties> linker>input> addition dependencies
4. start from the begin re-download dev-moai, see if you can build with a fresh copy in a new folder.

by stun » Sun Mar 31, 2013 1:03 pm

Which projects in the solution are you trying to build? I'm building MOAI with VS2010 and not every project will build 'out of the box' due to requiring third party DLLs that aren't shipped with MOAI. (FMOD for example).

Other than that the basic MOAI core and several of the host projects should build fine, you can safely ignore the other host projects as long as you can build the one you need which will typically be 'moai' or 'moai-untz' I believe.
