by nosheet » Wed Oct 02, 2013 3:22 pm

Yes, as far as I know, you need first to create MOAIFrameBufferTexture, and then cal the grabNextFrame.
Ie:
Code: Select all
  1.  

  2. myImage = MOAIImage.new()

  3. myFrameBufferTexture:grabNextFrame( myImage, function()

  4.     -- In this function you want to delete your frame buffer texture

  5. end)

  6.  



Please have in mind that grabNextFrame() is a very slow method (unlike renderFrameTexture that comes with no performance penalty), probably due to copying memory block into MOAIImage. In my tests, for Retina iPad it would take a second or something to finish operation.

by seebs » Wed Oct 02, 2013 8:47 pm

I should expand on this. I was using a MOAIFrameBuffer. To cause the frame buffer to be rendered, ever, you have to tell the render manager about it. From some diffs from an old version of a thing:

Code: Select all
  1.  

  2.   n.framebuffer:grabNextFrame(n.captured, function() Neon.rendered(n) end)

  3.   local t = MOAIRenderMgr.getBufferTable()

  4.   if t then

  5.     t&#91;#t + 1&#93; = n.framebuffer

  6.   else

  7.     MOAIRenderMgr.setBufferTable( { n.framebuffer } )

  8.   end

  9.  



And then Neon.rendered() had to do another grabNextFrame to another function that would actually get the results. Which is why I'm not using that now.

Re: One time offscreen render

by nosheet » Thu Oct 03, 2013 2:37 pm

Yes it can render an entire screen at 30, even 60fps, but there is a huge slowdown when copying the memory from the framebuffer (GPU memory) into the CPU memory of MOAIImage (at least I imagine that's the reason for slowdown, I didn't look at the implementation of the grabScreenshot).
I imagine that, if you don't need to manipulate pixels of the screenshot (or save them to disk), that you can code a shortcut that would not output framebuffer to the MOAIImage, but to MOAITexture, which should avoid copying the memory block. Even if you do copy memory via gltexsubimage2d, it shouldn't take too long (in my tests using gltexsubimage2d to copy the video frames to texture, I was getting some 20-30fps for 1024x1024 video).
