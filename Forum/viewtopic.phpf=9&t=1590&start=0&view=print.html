Long story short:

I was looking at the samples, and some had a "run.sh" which basically did .../../bin/moai main.lua.

That doesn't work for any of the iOS or Android specific things -- those all require that you be actually building for that target. And while MOAIDialog isn't iOS or Android specific, it doesn't have a generic implementation without reference to a specific host, so the simulator can't do it.

No, not any part of the MOAI SDK or code base. The "simulator" I refer to is, for instance, the iPhone/iPad simulator which is part of Xcode.

If you use Apple's tools to build a Moai project, you end up with an executable you can run on Apple's simulator (or hardware), and that will have the MOAIDialog available.

If you use the Android tools to build for Android, you can use the Android simulator or hardware, and that will have the MOAIDialog available.

I don't know about tutorials, I just opened the moai.xcodeproj and kicked it a few times and it worked.
