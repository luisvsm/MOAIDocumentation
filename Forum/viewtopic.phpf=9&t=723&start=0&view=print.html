Baked in Localization Support?

Posted: Sat Apr 28, 2012 12:29 pm
by sgeos
Does Moai have any support for localization baked in? Features that come to mind include text IO and a mechanism for detecting the locale.

I have a fairly robust messaging system and a cascading resource override system in my own Lua based project. Both take advantage of the iOS locale settings. The resource override system tries the find the localized version of any file before looking in the root project directory. (~/myapp/en/FILE or ~/myapp/ja/FILE before ~/myapp/FILE, finally ~/documents/FILE and probably a couple of other places.) This way you can localize something like the title screen, drop it in the right place and everything just works.

I'm not sure about detecting locale but Moai has a virtual filesystem, you can mount an archive into it. So each of your locale can be an archive.

Also, Moai is quite low-level, it gives you all the pieces but you have to build a resource management system yourself. Prepending something to a resource name should be trivial.

Re: Baked in Localization Support?

Posted: Mon Apr 30, 2012 7:07 pm
by ibisum
Don't even bother worrying about it - Lua is where you should be doing localization, and leaving MOAI for its job as a 2D game engine.

And by 'Lua is where you do localization", I mean: put your locale strings (which you'd expect to be organizing, anyway..) into plain old Lua resources, set up the right Font resource, and pass the strings around for the users language selection. End of problem.
