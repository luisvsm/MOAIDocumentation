Interesting question. I'm surprised the attribute linking didn't work in your first attempt?

Instead of a coroutine, have you considered using a MOAIScriptNode and linking it to the Box2dBody? Then perform your manual sensor 'chasing' in the update callback? As that's part of the MOAI node tree I wonder if it would still experience the lag you mention. Although of course it might since Box2d is really a separate simulation to the MOAI code.

Box2D has the Prismatic Joint as well which limits movement to along one axis. I wonder if you could fix the sensor to your rolling body with a prismatic joint to force it to stay upright. I've only ever used prismatic joints that have been fixed to a static ground body at one end however, so can't say definitely if that would work or not.

Re: rolling box2d body with sensor

by Vavius » Tue Sep 24, 2013 5:32 pm

Before dropping that rolling thing completely you should try one more time with approach 2 - joints.
Try it this way:
1) Position feet sensor correctly at players bottom and attach it with revolute joint to player circle. Use circle center as joint anchor.
2) Disable rotation for sensor (setFixedRotation(true)).

Now you should have player circle that can freely rotate and sensor that stays always at bottom, since it cannot rotate itself.
Sensor shouldn't really affect the circle. One thing is a center of mass will be lower a bit. If it creates problems try setting sensors density to some small value, maybe even 0.

Re: rolling box2d body with sensor

by niom » Tue Sep 24, 2013 7:04 pm

Thank you for the both answers. I actually went with the idea using the revolute joint and that solved the whole problem.

Thinks I needed to do is:
- set the sensor body to the correct position of the circle
- use the revolute joint between the player and the sensorBody (joint to the center of the player)
- sensorBody:setFixedRotation(true)
- sensorFixture:setMassData(0) -- this is needed if you don't want to have an elastic and bouncing sensor.

One step closer to a working prototype
