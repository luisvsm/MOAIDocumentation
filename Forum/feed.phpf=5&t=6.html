MOAIFileSystem.mountVirtualDirectory ( 'virtual', 'path/to/archive.zip' )

MOAIFileSystem.setWorkingDirectory ( 'virtual/folder/in/archive' )

The virtual path will work with any of the Moai commands that access the file system.

We've been using this to mount .apk's so we don't have to extract the app resource folder on install. There's still a known bug in the current SDK version, but it's been fixed in the development branch and should be ready for prime time in the next release.

Re Apple IAP, if you look in the Xcode reference project you should see an ios specific extension. There's also an StoreKit example under iphone-samples. (You know, I just realized that the MOAIApp docs aren't being generated; will look into that).

I was asking about iAds simply because we haven't heard of a lot of people seeing viable revenue there, unless they have millions and millions of users. TapJoy style incentives for in-app currency seem to be a lot more successful - the numbers there can be really good. If you haven't looked at TapJoy, you should. We're also expanding support for their service - we have baseline support now, but should have a much fuller implementation (ios for now, Android soon) in the next Moai SDK release or two.

IAP on Android isn't in there yet. We're seeing a good deal of fragmentation already. As Wolf Toss goes to market we'll have to figure out which one(s) we'll want to use (and will therefore support). In the mean time still thinking about ways to offer a more consolidated IAP experience across all platforms, not just Android.
