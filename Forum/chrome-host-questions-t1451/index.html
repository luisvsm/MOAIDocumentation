by JDance » Thu Dec 06, 2012 7:24 am

Well I log by setting these environment variables
Code: Select all
  1. NACL_DEBUG_ENABLE = 1

  2. NACL_EXE_STDERR = DEBUG_ONLY:dev://postmessage

  3. NACL_EXE_STDOUT = DEBUG_ONLY:dev://postmessage



These cause NaCl modules to post messages for all regular prints in native. Then you have to catch these in your webpage and route them to the console. For the original moai_base.js you add something like this:

Code: Select all
  1. Moai.Base.prototype.naclMessageHandler = function ( message_event ) {

  2.        

  3.         if ( message_event.data && message_event.data.search ( 'URL:Open:' ) != -1 ) {

  4.                 ...

  5.         }

  6.         else if ( message_event.data && message_event.data.search ( 'DEBUG_POSTMESSAGE:' ) != -1 ) {

  7.                 console.log(message_event.data.substr(18));

  8.         }

  9. }



To the end of the if-structure, add another else if for the DEBUG_POSTMESSAGE prefix. Now it will be all logged to the javascript console.

My keyboard input doesn't work either, it seems Chrome uses different key codes. I can post again when I get around to that.

In general I haven't found it very easy to get started with the Chrome build. Reading up on Native Client and how it really works has been very helpful. Right now I'm adding support to for my lua code to communicate with the web page through PostMessage since that wasn't built in.

Great with some feedback! Yeah it feels a bit strange to work with Pepper 17 when latest version is 23 or something. But it's still working so no panic yet. But it's just a matter of time before it will be necessary to update it.

I got a nice communication going between lua and javascript now by prefixing messages with "LUA:". I encode each message in JSON since it is supported in both the website and Moai, so I can send a whole object structure and not just strings. In my little API I also add a message ID to each message so if a listener returns a value, I send a reply message back to the other side with the orginal message ID and invoke a callback for that message.

So for example I call lua from javascript to get information from the game and I also pass a callback. In lua the invoked listener returns a value and through the messaging system the callback automatically gets invoked in javascript with the return value as argument.

Thanks for the reply!

We ending up getting this to work when invoking Chrome from terminal. Here are the setting used:

Code: Select all
  1.  

  2. export NACL_DEBUG_ENABLE=1

  3. export PPAPI_BROWSER_DEBUG=1

  4. export NACL_PLUGIN_DEBUG=1

  5. export NACL_PPAPI_PROXY_DEBUG=1

  6.  

  7. export NACL_EXE_STDERR=DEBUG_ONLY:dev://postmessage

  8. export NACL_EXE_STDOUT=DEBUG_ONLY:dev://postmessage

  9.  



Then run Chrome from the Terminal with this command:

Code: Select all
  1. /Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --enable-nacl



Now log messages get dumped to Terminal.
