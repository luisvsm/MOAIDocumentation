by ibisum » Wed Apr 03, 2013 2:47 pm

Pawel: You don't get the OpenGL ES acceleration in your Android emulator unless you've specifically set it up for your machine, to support whatever hardware-acceleration your machine provides (there's a shim layer that has to be enabled). Double-check that your expectation that acceleration is configured for your Android emulator is true!

Besides that, I have tried your app with my local Android project folder, and it works without any problems whatsoever (okay I added the setClearColor workaround also) - this is using the latest moai from github.com/moai/moai-dev. Double-check that the arialbd.ttf file is being packed into your resources, also the moai.png file, and so on.

Also, for future reference - its nearly impossible to debug issues like this for forum members working on Android if the logcat output is not included. "adb logcat" should provide you with a lot of clues about whats happening ..

EDIT: ONE thing that I just noticed that is different from my test of this code with Android-build locally here, is that I do not have my main.lua and other resources in a sub-folder (.ie. "hello-moai/" in your case), but instead I just have everything in the assets/lua folder. Could it be that your local-config is set up to run main.lua from "assets/lua", but you've inadvertently forgotten to add the "hello-moai" to your path? Try to simplify this and just put everything in "assets/lua" instead, and see if that produces more viable results. Oh, and 'adb logcat' would be helpful if you don't debug it and need further forum assistance ..

by ibisum » Wed Apr 03, 2013 3:46 pm

MOAI supports Android v4.2.2 - you are making the wrong conclusions. My Nexus7 is running it, I have absolutely no problem putting my MOAI work on it and running it with the latest moai release.

I think the real problem is that you are going past serious problems that are going to thwart your progress. The nexus4 not being picked up by logcat is definitely suspicious - do you have Developer Mode turn on, on the device? If I were you I would not do anything with MOAI until that is sorted out - because honestly, you cannot expect to build and release for Android without having access to logcat during development/platform testing. Fix that, or let us help you fix it, first.

Second, the Android emulator - did you enable hardware acceleration for the emulator? You *MUST* do this for MOAI to run in that environment - see here:

http://developer.android.com/tools/devi ... l-graphics

And:

http://stackoverflow.com/questions/7831 ... s-emulator
http://stackoverflow.com/questions/1156 ... on-support


Lastly: yes, please do build the MOAI host yourself. You will *HAVE* to do this anyway when your app is ready for distribution: doing it earlier rather than later is going to smooth things out a lot.

EDIT: if you want more help, join us in #moai on IRC at irc.freenode.net and look for user torpor (thats me).

spectralcanine wrote:Have to?


Just tweaking your app bundle settings for provisioning on iOS, for example, will trigger a re-build of the host, libmoai, and so on..
