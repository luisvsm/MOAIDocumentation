Demo: Package game in one executable file

by thebullno1 » Sat Apr 05, 2014 11:35 pm

This is a script and demo of how to pack your code and resources to an executable without rebuilding. It's slightly more complex than just appending an archive to the host because Moai's ZLVfsZipArchive is stricter than most zip archiver and does not accept such archive.

Download: https://dl.dropboxusercontent.com/u/7948800/MoaiPackDemo.zip

Currently, the demo only works on windows but the script should work fine on Linux. You need cygwin with zip and lua installed.

Too see how a game file is created, run build-win.sh. game.exe will be created in the current folder. You can move it to any directory and it will still work.

bin/pack.sh handles the packing. The syntax is:
Code: Select all
  1. bin/pack.sh <src-dir> <path-to-host> <output>



For example:
Code: Select all
  1. bin/pack.sh src bin/MyHost.exe MyGame.exe



The host is a modified version of Moai 1.5's glut host:

Right after:
Code: Select all
  1. GlutRefreshContext ( argc, argv);



I added:
Code: Select all
  1. AKUMountVirtualDirectory(".", argv[0]);



This will be part of a larger toolchain so no repo for now. Feel free to use the code for any purposes.

Some ideas regarding deploying on desktop:
  • lua file can be compiled to bytecode using luac
  • To add more protection, some obfuscation can be done on bytecodes. A custom loader will be inserted into package.loaders to deobfuscate.
  • custom code loader can be even written in C++ for speed and stronger encryption
  • zip everything up and pack to a host with this script
  • non code assets can be left alone or zip to another file called assets.dat
  • Build your host once, with an unique "obfuscation key". Reuse that for multiple projects.
  • Alternatively, build once per project. Reuse for different builds.
