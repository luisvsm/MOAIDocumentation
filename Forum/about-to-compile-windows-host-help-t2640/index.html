After a couple of weeks trying to avoid it, I'm gonna try to compile Moai 1.4 for Windows (moai-untz, more precisely). There are a few things I want to do, and I'd appreciate a lot if you could guide me through them:

-[SOLVED] Hiding mouse pointer.
--> Open GlutHost.cpp, go to line 320 and add:
Code: Select all
  1. glutSetCursor(GLUT_CURSOR_NONE);



----------------------------------------------------------------------------------------

-[SOLVED] Make the executable load a file when double clicking it (main.lua or a packaged luac) so I don't need to make a launcher.bat to load my lua code.
-->Moai 1.4: After the line in the previous modification, add:
Code: Select all
  1. AKURunScript("main.lua");



-->Moai 1.5.
I finally moved to Moai 1.5 for a bug I had after rebuilding 1.4. To compile Moai 1.5, first I had to uninstall "Microsoft Visual C++ 2010 x86 Redistributable" and "Microsoft Visual C++ 2010 x64 Redistributable", then downloaded the latest DirectX SDK (DXSDK_Jun10.exe) and installed it. Then I went to "bin" directory in moai 1.5 source, launched a Windows Command Prompt and run "create-projects-win.bat vs2010" (I had Visual Studio 2010). My project was generated in "<moai source dir>\cmake\projects\vs2010".

I opened the solution but building would still fail, as first I had to add the DirectX libs path to the project, in Linker->General section->Additional Library Directories. The DirectX libs path depends on wether you're building for 32 bits or 64 and it should be something like "C:\Program Files (x86)\Microsoft DirectX SDK (June 2010)\Lib\x86" or "C:\Program Files (x86)\Microsoft DirectX SDK (June 2010)\Lib\x64". Now if you build the solution, you should have moai.exe (no more glut32.dll, cool) in "<visual studio solution directory>\host-glut\Release". But this still doesn't allow loading a luac file by double clicking moai.exe. We'll get into that now.

First, my main.luac file renamed to data.dat wouldn't work, because Moai 1.5 uses LuaJit, so I downloaded luajit sources from here and followed the installation instructions here. Finally I packed my code to a single lua file, copied it to my luajit.exe directory (had to set an environment variable later), and in the command prompt, I typed "luajit -b main.lua data.dat".

Then I noticed that in Moai 1.5, there's this code in Modules->host-modules->Source files->aku_modules.cpp:
Code: Select all
  1.         #else

  2.        

  3.                 if ( argc < 2 ) {

  4.                         AKURunScript ( "main.lua" );



I changed main.lua to data.dat and now double clicking moai.exe runs data.dat. Finally!

----------------------------------------------------------------------------------------------------------

-[SOLVED] Disable console window.
--> Simply change GlutHostMain.cpp to have this main:
Code: Select all
  1.  

  2. int main ( int argc, char** argv ) {

  3.         //prevents debuggin

  4.         #pragma comment(linker, "/SUBSYSTEM:windows /ENTRY:mainCRTStartup")

  5.         #ifdef _DEBUG  

  6.         #endif

  7.         return GlutHost ( argc, argv );        

  8. }




-Add a custom icon to the .exe.
--> Changing the exe icon in Visual Studio 2010 was easy, and the exe has my icon, but in the taskbar it still shows the OpenGL icon (http://i.imgur.com/asFr20A.png). I guess the icon I should change is the OpenGL one, but how?


Thanks a lot in advance!
