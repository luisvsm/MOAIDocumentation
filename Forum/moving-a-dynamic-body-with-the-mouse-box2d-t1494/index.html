Re: Moving a dynamic body with the mouse ( box2d )

by stun » Sun Dec 16, 2012 8:33 am

There's a box2d feature called 'mouse joint' that can be used to follow a mouse around. Generally it's not recommended for production use, at least according to the box2d manual, by it is handy for testing things. Google 'box2d mouse joint' and you'll find lots of example code you can translate into MOAI. Depending on your requirements even 'setLinearVelocity' might be tricky to use, the other option is just to applyLinearForce every frame in the appropriate direction (the vector from your prop's current point to the desired point) to 'guide' it in a gentle fashion.

The replacement for 'setParent' is attribute links as you mention. For getting a prop to match the transform of a box2d body, just use the following:

prop:setAttrLink(MOAIProp2D.INHERIT_TRANSFORM, body, MOAIProp2D.TRANSFORM_TRAIT)

Reading the MOAI C++ code will show you the various 'attributes' each MOAI class supports btw.

To get from a prop to a body you have to do it yourself. The easiest way I've found so far is simply to assign a custom property to the prop in question. Something like this:

prop.myBox2dBody = body

I use wrapper classes, so am typically assigning a class instance to my custom prop variable (which will include the box2d body, fixtures and a bunch of other game related data). In lua there's nothing stopping you from assigning (and indeed replacing) properties and functions of MOAI entities. Just make sure you use a distinct name so you don't stomp on something important. Hope that helps.
