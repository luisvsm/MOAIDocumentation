Problems with make-host on Android

Posted: Mon Mar 24, 2014 7:15 pm
by Kantholz
Hello and sorry to bother you with such a basic problem, but I'm pretty new to Linux and just can't seem to get the android host working.

The history:
After a few days of experimenting with MOAI on a Windows 7 machine, I noticed that the version I had downloaded did not have Untz enabled. The MOAI-Untz folder was missing a binary, so I decided to switch to Linux and make one. Compiling a host for Linux worked (still no sound yet, but that's another thing), but trying to make a host for android just keeps giving me errors.

What I did, step by step:
1) Laptop running freshly installed Ubuntu 12.04 x64
2) Checked out latest version of MOAI from github and setup environment as explained in the readme here: https://github.com/moai/moai-dev
3) Installed Android NDK (32bit) and SDK
4) Exported environment variables (by adding to ~/.profile)
- MOAI_BIN
- MOAI_CONFIG
- ANDROID_NDK
Variables show up (printenv), I double checked the paths and they seem correct (navigated to every one of them to make sure).
5) Successfully built the Linux-Host. The Hello-Moai Sample is working fine, as is my own test-project. So far, so good.
6) But when I try to run build-android.sh from the /bin directory, I get mainly errors:
Code: Select all
  1. ./build-android.sh: line 3: /home/markus/.bash_profile: No such file or directory


But mostly lots of permission-denied errors like this one:
Code: Select all
  1. Building C object third-party/tlsf/CMakeFiles/tlsf.dir/home/markus/moai-dev/moai-dev/3rdparty/tlsf-2.0/tlsf.c.o

  2. /bin/sh: 1: /media/markus/storage/development/android/android-ndk-r9d/toolchains/arm-linux-androideabi-4.8/prebuilt/linux-x86/bin/arm-linux-androideabi-gcc: Permission denied

  3. make[3]: *** [third-party/tlsf/CMakeFiles/tlsf.dir/home/markus/moai-dev/moai-dev/3rdparty/tlsf-2.0/tlsf.c.o] Error 126

  4. make[2]: *** [third-party/tlsf/CMakeFiles/tlsf.dir/all] Error 2

( ".../moai-dev/moai-dev/..." is admittedly stupid, but it's the correct path)

I tried setting permissions (chmod -R 777) on the moai-dev folder as well as the android-ndk folder, but unfortunately the error remains.

I'm guess I'm just missing something obvious?
Any help would be greatly appreciated!

Posted: Wed Mar 26, 2014 10:57 am
by Kantholz
** PROBLEM FIXED **

Thanks for the fast reply, halfnelson!
Unfortunately, running build-android.sh from the sdk root still gave the same permission denied errors.
So the next thing I tried was moving the Android-NDK from the shared storage partition to the Linux partition.
This replaced all the "permission denied" errors I got with "not found" errors instead, like:
Code: Select all
  1. arm-linux-androideabi-gcc: not found


After confirming that all those files were exactly where they should be, I googled the "not found" messages and stumbled upon the solution: seems like I was missing "ia32-libs".
So I installed ia32-libs, or rather three packages to replace it: lib32z1, lib32ncurses5 and lib32bz2-1.0

Build-android now finally works!
