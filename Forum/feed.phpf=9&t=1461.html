]]> 2013-09-19T17:22:11+00:00 2013-09-19T16:41:28+00:00 /forums/viewtopic.php?t=1461&p=11627#p11627 <![CDATA[Re: Lightweight library.(Flower)]]> Hi.

Flower will continue to cache the MOAIFont that was used once.
There is a possibility that this effect does not release the internal texture.

Basically, in order to maximize the performance, is called the preloadGlyphs function.
If you use only alphanumeric characters, it does not dynamically generated texture.
Therefore, there is no internal texture that is generated many times.

How internal texture has been generated in your game?
If it is generated frequently, there is some cause.

If you sent me the source to me, it is possible to investigate the cause.

EDIT:
Continue to cache the MOAIFont It is intentional.
Because fonts reuses frequently, it is necessary to continue to cache while long time.

Problem of memory usage has not happened in the range to be used by me.
