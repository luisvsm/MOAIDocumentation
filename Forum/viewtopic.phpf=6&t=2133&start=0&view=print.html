IDE is PyQT based, develop iteration is really fast. Because python has a very big community, I've add some good features with very little efforts ( e.g image processing/ios device access/file system watcher ).
Python-Lua communication is done through Lupa( https://pypi.python.org/pypi/lupa ), which is almost seamless.
The engine part is a unity-like component framework, it's almost done entirely in Lua without C++ side modification. I made a class system which supports introspection in IDE.

It's not stable at this moment. It would take quite some time to get decent enough to be open to public ( after current project at least ).

Moai is wonderful, big thanks to Patrick!

Gianmichele wrote:This looks really nice! Any chance of more info or a video showing your tool chain?

Basically, it's a unity-clone. You manage your asset in IDE, and write component in Lua with external editor.
I tried to avoid building an IDE and keep it lightweight ( like a cli toolset ), but assets have too many connections to each other, so IDE is just a better idea.

Component is build using my custom class system, you can setup data model with some kind of DSL. The model is used for introspection & serialization.
Some example:
Code: Select all

  2. import 'script.TextBox'

  3. import 'script.effect.PopText'

  4. import 'script.effect.EffectShake'

  5.  

  6. EnumCreatureTeam = {

  7.         {'enemy', 'enemy'},

  8.         {'hero',  'hero'},

  9.         {'npc',   'npc'},

  10. }

  11.  

  12. CLASS: Creature ( mock.FSMController )

  13.         :MODEL{

  14.                 '----';

  15.                 Field 'hp'     :int();

  16.                 Field 'mhp'    :int();

  17.                 Field 'atk'    :int();

  18.                 Field 'def'    :int();

  19.                 Field 'textHP' :type(TextBox);

  20.                 Field 'team'   :enum( EnumCreatureTeam );

  21.         }

  22.  

  23. mock.registerComponent( 'Creature', Creature )

  24.  

  25. function Creature:__init()

  30. end

  32. function Creature:onAttach( entity )

  34.         self.msgListener = entity:addMsgListener(

  35.                 function( msg, data, source )

  36.                         return self:onMsg( msg, data, source )

  37.                 end

  38.         )

  39. end

  40.  

  41. function Creature:onDetach( entity )

  42.         mock.FSMController.onDetach( self, entity )

  44. end

The editor is built under some module protocol. The UI interaction is mainly done in python, and the data part is done in Lua.
It supports hot reloading for some asset types( texutre, decks, etc...). It's designed to be customizable for each project, so you can easily write different asset type manager as needed.
Game component script can be hot reloaded, too. The editor provides some simple remote command API, so you can control the editor in your coding tool. I wrote some basic controller for sublime text ( code reload, game preview start/stop... )

There are many important missing features, like prefab inheritance( on of the best part of unity ), undo/redo support, elegant exception handling. Hopefully they will be eventually added some day.

I wish I have time to write more about it, maybe some time later.

I'm using FMOD designer module contributed by double fine (IIRC).
There're some issues in the module, I fixed some of them.
When playback with FMOD on iOS, there will be some little delay, you have to change dsp buffer size for FMOD initialization (http://www.gamedev.net/topic/589845-fmo ... d-latency/).
What I'm facing now is sound time synchronizing. You can get time of playing sound from Event:getTime(), but it's not accurate. According to FMOD forum, you have to implement some interpolation to solve this problem( http://www.fmod.org/forum/viewtopic.php ... dsp#p48923 ).
That's all. Hope it can help.
