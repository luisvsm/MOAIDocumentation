Rolling many source files into one file can be done.

by sgeos » Thu Jun 21, 2012 4:03 am

If for whatever reason anyone wants to roll a bunch of modules up into a single lua file, this can be done by concatenating the files.
Code: Select all
  1. $ cat Module1.lua Module2.lua Module3.lua main.lua > bundle.lua


The module declarations must explicitly set the module names if you want to concatenate files.
Code: Select all
  1. -- This works in a concatenated file

  2. module ( "MyModuleName", package.seeall )

  3.  

  4. -- This does NOT work

  5. module ( ..., package.seeall )


Here is a proof of concept that can be run as one file:
Code: Select all
  1. -- Message.lua

  2. module ( "Message", package.seeall )

  3.  

  4. function new(pMessage)

  5.   local result   = {}

  6.   result.message = pMessage

  7.   function result:print()

  8.     print(self.message)

  9.   end

  10.   return result

  11. end

  12.  

  13. -- Hello.lua

  14. module ( "Hello", package.seeall )

  15.  

  16. require "Message"

  17.  

  18. function new(pTarget)

  19.   return Message.new("Hello " .. pTarget .. "!")

  20. end

  21.  

  22. -- GoodBye.lua

  23. module ( "GoodBye", package.seeall )

  24.  

  25. require "Message"

  26.  

  27. function new(pTarget)

  28.   return Message.new("Good bye, " .. pTarget .. "!")

  29. end

  30.  

  31. -- main.lua

  32. require "Hello"

  33. require "GoodBye"

  34.  

  35. local target = "World"

  36. Hello.new(target):print();

  37. GoodBye.new(target):print();


This can be run as a concatenated file or not.
Code: Select all
  1. $ cat Message.lua Hello.lua GoodBye.lua main.lua > bundle.lua

  2. $ lua main.lua

  3. Hello World!

  4. Good bye, World!

  5. $ lua bundle.lua

  6. Hello World!

  7. Good bye, World!

  8. $
