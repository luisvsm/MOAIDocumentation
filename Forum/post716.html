Hey Everyone,


Some additional information on getting the nacl branch up and running for the adventurous.




Getting it up and running:



- Since the OpenGL interfaces just moved out of Dev to Beta, the 0.5 SDK listed on Google's site won't work. Use the Beta SDK at http://code.google.com/chrome/.....a-sdk.html. 

 

- To build just run the scons.bat on windows ( I haven't kept the unix up to date, but the shell script should just be the default scons.sh in the SDK ).


- Only Fmod is supported right now for sound, Fmod officially supports native client and you can download their beta at http://www.fmod.org/index.php/download, you have to add the libraries to the toochain. 


- If you do not want sound you need to remove the Fmod library dependencies from the build. The build system used is Scons, you can adjust build settings by looking in the build.scons. Eventually I'll setup a build without sound.


Notes/Issues:




- File saving and Socket communication is not well supported right now and will probably crash your application.


- Do not force anything to load 'on demand' ( I believe the only way to do this is to force releasing of textures still in use ). Due to the way the native client API and File I/O work this will cause hangs/crashes.




The code will be undergoing some major re-factoring soon now that the APIs are stable, so hopefully the above problems will be resolved soon.

by adam » Thu Oct 13, 2011 4:31 am

Some tips on running: 



I find it easier to run as an unpacked extension. You can also run from a web server the same way their examples do.

 
For an unpacked extension:

- Create a manifest http://code.google.com/chrome/.....ifest.html

- Run Chrome

- Click on the wrench icon in the upper right and go to Tool->Extensions

- The should be a checkbox on the page that says "Developer mode", check that box

- The option to "Load upacked extension..." should appear", click this and navigate to the directory where your .nexe and webpage are


Alternatively you can host the directory you have your .nexe and webpage on a webserver and simply access that from chrome. Just be sure to clear the browsing data. ( You can also run in incognito mode to prevent it from being cached )


The basic workflow is:

- Run the Scons script ( only for C++ changes )

- Run chrome

- Go to options and clear browsing data

- Run the extension
