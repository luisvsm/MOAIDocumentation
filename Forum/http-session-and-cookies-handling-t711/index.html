Re: HTTP session and cookies handling?

by maggus » Wed May 09, 2012 5:20 pm

a problem with your fix;
"cookie.txt" file does not get created until application closes. So the first time you run the app, http tasks do not see cookie file. And thus all http tasks on first run does not share same cookies, and server treats them as separate sessions.
Example:
replace last line from your example "util-http-cookies" to this:
Code: Select all
  1.  

  2. --sendHTTPRequest ( "www.google.com" )

  3. MOAISim.openWindow ( "test", 32, 32 )

  4. local timer = MOAITimer.new ()

  5. timer:setMode (MOAITimer.LOOP)

  6. timer:setSpan ( 1 )

  7. timer:setListener ( MOAITimer.EVENT_TIMER_LOOP,

  8.         function ()

  9.                 sendHTTPRequest ( "www.google.com" )

  10.         end

  11. )

  12. timer:start ()

  13.  


run it and while it's running notice that no "cookie.txt" file gets created. It only gets created once you killed the app.

The problem is still exist even if I try async request with moai1.4p0.
Cookie file does not get created until application closes. You may try to delete a cookie file to see this file is not get created on first try.

Current workaround is just not use cookie file. Manually get cookie from response header 'Set-Cookie', and set it to request header 'Cookie'.
