Box2D setCollisionHandler Edges

by noon » Mon Nov 25, 2013 11:26 am

Why can't I set a collision handler to Edges?
For instance this code doesn't work:

body = world:addBody ( MOAIBox2DBody.STATIC )
fixture = body:addEdges { -5, -5, 5, -3 }
fixture:setCollisionHandler ( onCollide, MOAIBox2DArbiter.BEGIN + MOAIBox2DArbiter.END, 0x00 )

It gives me an error:

attempt to call method 'setCollisionHandler' (a nil value)

I can set collision handlers to other types of fixtures like polygons and circles.
I really need to detect when collision with edges happens.

Re: Box2D setCollisionHandler Edges

by stun » Mon Nov 25, 2013 4:00 pm

Just from the Box2D manual about 'Edge Shapes': "These are provided to assist in making a free-form static environment for your game. A major limitation of edge shapes is that they can collide with circles and polygons but not with themselves.", that may be why. I suspect you're expected to set collision handlers on the things colliding with edges, rather than the edge fixtures themselves. That certainly works as I've done it while building a simple Box2d cavern game for 1GAM.

If you haven't already I'd really recommend recommend reading the Box2D manual itself from top to bottom. Box2D has some quirks, but they're all described in the manual: http://www.box2d.org/manual.html

Also beware that MOAI doesn't expose all of the Box2D functionality you might want either
