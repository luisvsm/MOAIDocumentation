Debugging MOAI source on Linux

by akak » Sun Nov 10, 2013 10:59 am

A quick note of setting up qtcreator for debugging MOAI source.

Set up I used:
- Ubuntu 13.10
- qtcreator 2.8.1

1. Select Open project in qtcreator and point it to <moaidir>/cmake
2. Set up qtcreator cmake options:

Build directory:
Code: Select all
  1. <moaidir>/cmake/build


Arguments:
Code: Select all
  1. -DBUILD_LINUX=TRUE -DMOAI_BOX2D=TRUE -DMOAI_CHIPMUNK=TRUE -DMOAI_CURL=TRUE -DMOAI_CRYPTO=TRUE -DMOAI_EXPAT=TRUE -DMOAI_FREETYPE=TRUE -DMOAI_JSON=TRUE -DMOAI_JPG=TRUE -DMOAI_MONGOOSE=TRUE -DMOAI_OGG=TRUE -DMOAI_SQLITE3=TRUE -DMOAI_TINYXML=TRUE -DMOAI_PNG=TRUE -DMOAI_SFMT=TRUE -DMOAI_VORBIS=TRUE -DMOAI_UNTZ=TRUE -DMOAI_OPENSSL=FALSE -DMOAI_LUAEXT=FALSE -DCMAKE_C_FLAGS="-lGL" -DCMAKE_BUILD_TYPE=Debug ..


- options copied from bin/build_linux_x64; plus -lGL flag to fix gl library error; plus enabling debugging

After you compile moai with qtcreator:

Select Projects tab, select Run:
Run configuration: moai (not moai-test-runner)
Arguments: main.lua
Working direectory: your game dir

You should be able to debug moai source now.

If you can debug and make breakpoints but can't see any variables in Locals and Expressions view and if there is this error in Debugger Log:
Code: Select all
  1. >&"Undefined command: \"bb\".  Try \"help\".\n"


problem is with gdb being compiled with Python 3 which qtcreator 2.8.1 doesn't support.

If you see something like this, you should be ok:
Code: Select all
  1. % ldd `which gdb`

  2.         linux-vdso.so.1 =>  (0x00007fff44bfe000)

  3.         libdl.so.2 => /lib/x86_64-linux-gnu/libdl.so.2 (0x00007f9a1e33b000)

  4.         libncurses.so.5 => /lib/x86_64-linux-gnu/libncurses.so.5 (0x00007f9a1e118000)

  5.         libtinfo.so.5 => /lib/x86_64-linux-gnu/libtinfo.so.5 (0x00007f9a1deef000)

  6.         libz.so.1 => /lib/x86_64-linux-gnu/libz.so.1 (0x00007f9a1dcd6000)

  7.         libm.so.6 => /lib/x86_64-linux-gnu/libm.so.6 (0x00007f9a1d9d1000)

  8.         libpthread.so.0 => /lib/x86_64-linux-gnu/libpthread.so.0 (0x00007f9a1d7b4000)

  9.         libpython2.7.so.1.0 => /usr/lib/x86_64-linux-gnu/libpython2.7.so.1.0 (0x00007f9a1d250000)

  10.         libexpat.so.1 => /lib/x86_64-linux-gnu/libexpat.so.1 (0x00007f9a1d025000)

  11.         libc.so.6 => /lib/x86_64-linux-gnu/libc.so.6 (0x00007f9a1cc5d000)

, but if it's something like "libpython3._.so__" then you should recompile gdb. To do this execute:

Code: Select all
  1. wget "ftp://sourceware.org/pub/gdb/releases/gdb-7.6.1.tar.gz"

  2. tar xvzf gdb-7.6.1.tar.gz

  3. cd gdb-7.6.1

  4. ./configure -with-python

  5. make

  6. sudo checkinstall -D # generate deb package

  7. wajig install ./gdb*.deb # or dpkg -i ./gdb*.deb

  8. sudo ln -s /usr/local/bin/gdb /usr/bin/gdb # qtcreator looks for /usr/bin/gdb



That's all: http://i.imgur.com/54KNulq.png
