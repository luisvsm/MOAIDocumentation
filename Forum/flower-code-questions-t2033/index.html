Re: Flower - Code Questions

by btatarov » Wed Sep 04, 2013 10:52 am

I'm using Flower for all my projects. At first I had the same concerns - keeping my codebase small and trying not to use any libraries. But at some point I found myself cloning more and more from Flower's code to make simple things like groups, scene management, etc. That's why I made the switch. I haven't seen any slowdowns because of the library even with a lot of fast moving action elements like bullets and enemies.

naturally wrote:Are there any tutorials for Flower?

No. Currently you're stuck only with the git repo samples. On the plus side Flower follows the same principles most of the frameworks do so you can look into RapaNui samples or even samples from other engines like Corona SDK and Gideros Mobile and make the same using the Flower namespace.

naturally wrote:Is it possible to use the sublibraries of Flower, such as the TMX library, separately?

Flower uses smart OOP tricks that the author implemented in all his extensions so if you want to use an extension separately, you'll need to at least copy the table and class parts from Flower.
