by ezraanderson » Tue Apr 02, 2013 7:53 pm

i am also relatively new to moai and lua but this is what i do, make sure to load textures and release them, make sure to set things to nil when finished using them, and use MOAISim.forceGarbageCollection() between scenes. this seams to keep the memory in check. I am also sure there are other object in the moai framework that need to be cleanup differently, i just have not ran into these situation yet.


Code: Select all
  1.  

  2. image_texture = MOAIImageTexture.new()

  3. image_texture:load( 'image.png' )

  4.  

  5.  

  6. local image_quad= MOAIGfxQuad2D.new ()

  7. image:setTexture (image_texture)

  8. image:setRect ( -16, -16, 16, 16 )

  9.  

  10. image_texture:release()

  11. image_quad = nil

  12. MOAISim.forceGarbageCollection()--only run when you want to do a clean up

  13.  




have you verified that your game has memory leak?

the only thing that i can advise is try managing the particles/sprites/weapons systems in a pool structure to recycle them. I have 100 sprites(props) , with 100 particles systems, with 100 textures, when the sprite dies they explode, they go off screen and swap texture, swap particle system, and comes back as a new sprite. then in the process of doing this find out how many sprites/paricle systems the device can handle.

Hopefully someone else can offer advice that can help.
