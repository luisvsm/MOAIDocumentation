https://github.com/latestpost/Moai-SDK-Player



You could pull down this eclipse project and have a play with it to get you started.



The deployment method on android takes a the file in the

/res/raw/ called bundle



This file is actually a zip file without .zip on the end. I was originally creating a zip file with the lua files and assets and renaming from bundle.zip to bundle - and then re-deploying through eclipse

At runtime the android harness unpacks this file and then executes the files - (Note: it doesn't like to unpack folders so just use flat files)



To create the above project for Eclipse in a nutshell on a Windows platform - 

pre-requisites - Eclipse and Android SDK installed

1) Get ant working

2) install cygwin

3) download the android source

4) run the build.sh script to create a working android project

5) import the project into eclipse

@taylorringo Apologies - the documentation for Android on the wiki is sorely outdated as I have not been able to keep up with the rapid changes in the code and process. However, I will be finished updating it early this week. 

In the meantime, there are some READMEs that should help you get started. If you're using the binary SDK downloaded from getmoai.com, then look in the hosts/android/ folder. If you're enlisted in the github source, then look in the android/ folder. Basically, you need to run the make-host script, which will create a new Android host for you. Inside the untitled-host/ directory that it creates, you will find a run-host script that will package up a full Android project into it's build/ sub-directory - the contents of that directory can be loaded into eclipse. DO NOTE that every time the make-host or run-host scripts is run, it obliterates its working directory, so if you intend to edit/debug the project in eclipse, I highly recommend that you take the untitled-host/build/ folder and move it elsewhere on your computer before importing it into eclipse.

I'll be tweeking/improving the Android process more in the next couple of weeks, so if you have any suggestions, complaints or other feedback, please let me know and I'll do my best to address it in upcoming SDK releases.
