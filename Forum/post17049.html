Re: Level Editor - your thoughts please

by thebullno1 » Sat Apr 05, 2014 1:19 am

I don't like most of the generic level editors out there. They always lack what I think is the most important: domain logic.

For example, teleporters should come in connected pairs. Doors should come with keys ... There's no way to validate and see it clearly in the editor.

Moreover, the placeholder graphics in the editor might not always match what happens in the game.

For those reasons, I am an advocate of creating your own level editor. In the past, I've made one for a physic puzzle game which looks like this:


My current project doesn't need anything complex so I just use a text file for tile editing(really). But for my next one, I'm planning to write it in Moai. You get WYSIWYG for free. I'll try to make it generic and extensible. It will be sort of like Unity.

If you are willing to get creative you can do some tricks with Tiled to store some logic. For example using a 'data layer' with numeric tiles, could be used to match a keycard to a door, for example. Tiled also supports Objects with arbitrary data. The problem comes with using a generic level-loader, will not be able to handle such data. You will have to custom-load the levels.

Re: Level Editor - your thoughts please

by Galdred » Sun Apr 20, 2014 9:33 pm

I started using Tiled, toying with using objects zones, or tile objects for a lot of game logic, but I am now working on my own, mostly for the following reasons:
- It would be awkward for someone else than me to use the editor.
- It would be much better if the interface is the same as the game's for viewing and editing game objects properties.
- Flower gui already gets a lot of the interface works out of the box, and can easily be extended with menubars, pop up boxes, and drop down lists. So the main issue of needing a more desktop oriented GUI is no more an obstacle.

That said, Tiled was pretty amazing, and I will go back to it if writing my own editor takes too much time.

by Galdred » Tue Mar 31, 2015 7:04 am

update : the new features of tiled are pretty neat, and removed my urge to write my own editor entirely
The automap feature allows you to set up input conditions (ie : what is in tiles for the region to check), and have tiled generate the corresponding output (tiles in the original layer or a new one).
So basically, it took care of all my border and rectangular walls in an hexagonal grid problem.
The only thing I don't like is the handling of objects, so I will probably not use tiled for the level logic.
