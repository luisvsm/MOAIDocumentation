Instruction for Building with Ubuntu 12.04.

by ezraanderson » Thu May 08, 2014 6:12 pm

~
I finally built MOAI 1.5 master with Ubuntu 12.04(32bit) last night.
  • https://github.com/moai/moai-dev/wiki/B ... untu-13%29

Download Ubuntu, I had a copy of 12.04 on DVD that I installed
  • http://www.ubuntu.com/download/desktop

I keep all my source on a windows machine, so I needed to mount a network drive in Ubuntu with the correct permissions. This allows me to edit my source on windows but build on ubuntu.
  • https://wiki.ubuntu.com/MountWindowsSharesPermanently


Code: Select all
  1.  

  2. gksudo gedit fstab

  3.  

  4.  

  5.  



Install Dependencies
  • https://github.com/moai/moai-dev/wiki/B ... untu-13%29
Code: Select all
  1.  

  2. $sudo apt-get install git-core cmake build-essential

  3. $sudo apt-get install freeglut3-dev libglew-dev libglu1-mesa-dev libxmu-dev libxi-dev libsdl-dev

  4. $sudo apt-get install libxxf86vm-dev

  5.  



Now remove the old version of cmake that was just installed
Code: Select all
  1.  

  2. sudo apt-get remove cmake

  3.  



Download the new version of cmake, cmake-2.8.12.2-Linux-i386.tar.gz (not the top link)
  • http://www.cmake.org/cmake/resources/software.html

Export the path to cmake that you previously downloaded
Code: Select all
  1.  

  2. export PATH=$PATH:/home/ezra/cmake/bin

  3.  



Try to build moai
  • https://github.com/moai/moai-dev
Code: Select all
  1.  

  2. # Build

  3. cd moai-dev

  4. ./bin/build-linux-sdl.sh

  5.  

  6. # Run

  7. $MOAI_BIN/moai main.lua

  8.  



if the build failed, you are probably missing dependencies.
edit--> moai-dev\bin\build-linux-sdl.sh --> line 44:
When building this will show the error in the console near the bottom, try to Google and find the missing Dependency.
  • https://github.com/moai/moai-dev/blob/m ... nux-sdl.sh
Code: Select all
  1.  

  2. //change make = j2/j4 --> to --> make -j1

  3. make -j1

  4.  




if lua-jit is the error disable luajit
  • https://github.com/moai/moai-dev/blob/m ... nux-sdl.sh
Code: Select all
  1.  

  2. line 39: -DMOAI_LUAJIT=FALSE \

  3.  





You can try this to get missing Dependency
  • moai-make-error-on-debian-7-1-32bit-t2018/#p11346
Code: Select all
  1.  

  2. sudo apt-get install freeglut3-dev libglew-dev libglu1-mesa-dev libxmu-dev libxi-dev chipmunk-dev libjpeg8-dev libpng-dev libfreetype6-dev liblua5.1-0-dev libjansson-dev libtinyxml-dev libcurl4-openssl-dev zlib1g-dev libexpat1-dev cmake build-essential freeglut3 libxmu-dev libxi-dev cmake build-essential zlib1g-dev libssl-dev sqlite3 libsqlite3-dev libexpat-dev libcurl4-openssl-dev libasound2-dev libbox2d0 libbox2d-dev libogg-dev libvorbis-dev libpulse-dev libsdl1.2-dev libbox2d-dev libxxf86vm-dev



If your updating your C++ source and need to rebuild
Code: Select all
  1.  

  2.  cd ../cmake/build

  3. make moai/fast -j4

  4. gets put into cmake/build/host-****/moai

  5.  




Notes:
1. Getting sudo privileges with gedit & nautilus
Code: Select all
  1.  

  2. gksudo gedit  /etc/fstab  

  3. gksudo nautilus

  4.  





Thank you to seebs, naturally and halfnelson for helping figure all this out lastnight
