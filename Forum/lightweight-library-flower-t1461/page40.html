Re: Lightweight library.(Flower)

by Vavius » Tue Sep 17, 2013 1:12 pm

@n2oula
Latest flower already manages scene switching nicely. It will destroy your scene module when scene is closed.
Something is retaining references to your textures, check the following:
1) If you call flower.openScene(sceneName, params), then the old scene is not closed - it will remain in scene stack. In order to close current scene call flower.gotoScene(sceneName, params);
2) If you run coroutines or other actions in loop on some objects, then you should stop them when scene is closed. I usually perform it in scene:onClose() callback.
3) Maybe some global variables still contain references to unused objects. Can be a typo with forgetting "local" keyword somewhere.

Edit: forgot about the main problem =) Currently there is a big research going on in main git repo. Moai was crashing when GC was collecting objects that it shouldn't collect. A fix to this bug introduced a new one - now GC cannot handle circular references. A fix to this is not currently in any branch.
As a workaround you can try this:
Code: Select all
  1.  

  2. // comment MakeStrong in MOAILuaObject.cpp line 408

  3. void MOAILuaObject::OnRetain ( u32 refCount ) {

  4.     UNUSED ( refCount );

  5.  

  6.     //if ( refCount == 1 ) {

  7.     //    this->mUserdata.MakeStrong ();

  8.     //}

  9.  }


If it is not introducing crashes for you, then you can try to release the game =)

by makotok1123 » Thu Sep 19, 2013 4:41 pm

@n2oula
Hi.

Flower will continue to cache the MOAIFont that was used once.
There is a possibility that this effect does not release the internal texture.

Basically, in order to maximize the performance, is called the preloadGlyphs function.
If you use only alphanumeric characters, it does not dynamically generated texture.
Therefore, there is no internal texture that is generated many times.

How internal texture has been generated in your game?
If it is generated frequently, there is some cause.

If you sent me the source to me, it is possible to investigate the cause.

EDIT:
Continue to cache the MOAIFont It is intentional.
Because fonts reuses frequently, it is necessary to continue to cache while long time.

Problem of memory usage has not happened in the range to be used by me.
