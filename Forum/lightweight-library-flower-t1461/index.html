by makotok1123 » Sat Jan 19, 2013 4:03 pm

@mattguest

Hi.
The recommended method is as follows.

Code: Select all
  1.  

  2. -- Player.lua

  3.  

  4. --------------------------------------------------------------------------------

  5. -- Import

  6. --------------------------------------------------------------------------------

  7.  

  8. local flower = require "flower"

  9. local class = flower.class

  10. local MovieClip = flower.MovieClip

  11.  

  12. --------------------------------------------------------------------------------

  13. -- Class

  14. --------------------------------------------------------------------------------

  15.  

  16. local M = class(MovieClip)

  17.  

  18. local ANIM_DATAS = {

  19.     {name = "walkRight", frames = {1, 2, 1}, sec = 0.20},

  20.     {name = "walkDown", frames = {3, 4, 3}, sec = 0.20},

  21.     {name = "walkLeft", frames = {5, 6, 5}, sec = 0.20},

  22.     {name = "walkUp", frames = {7, 8, 7}, sec = 0.20},

  23.     {name = "standDown", frames = {9, 10, 9}, sec = 0.20},

  24.     {name = "standUp", frames = {11, 12, 11}, sec = 0.20},

  25. }

  26.  

  27. --------------------------------------------------------------------------------

  28. -- Initializes the player object.

  29. -- @param layer layer to place the prop on

  30. --------------------------------------------------------------------------------

  31. function M:init(layer)

  32.     -- flower.MovieClip:init(self, "assets/player.png", 2, 6)

  33.     MovieClip.init(self, "assets/player.png", 2, 6) -- Do not need a semicolon.

  34.     self:setAnimDatas(ANIM_DATAS)

  35.     self:setIndex(9)

  36.     self:setLayer(layer)

  37.     self:setPos(100, 100)

  38. end

  39.  

  40. --------------------------------------------------------------------------------

  41. -- Initiate the walk up animation

  42. --------------------------------------------------------------------------------

  43. function M:walkUp()

  44.    self:playAnim("walkUp")

  45. end

  46.  

  47. return M

  48.  



When the call is as follows.

Code: Select all
  1.  

  2. module(..., package.seeall)

  3.  

  4. -- classes

  5. local Player = require "classes/Player"

  6.  

  7. -- variables

  8. local player

  9.  

  10. function onCreate(e)

  11.     -- layer

  12.     layer = flower.Layer()

  13.     layer:setTouchEnabled(true)

  14.     layer:setScene(scene)

  15.  

  16.     player = Player(layer) -- Player:new(layer)

  17.     -- player:init(layer) -- The constructor is called when generating.

  18. end

  19.  

  20. function onStart(e)

  21.     player:walkUp();

  22. end

  23.

by makotok1123 » Fri Feb 01, 2013 4:48 am

@arreche

Hello.
There are no plans in particular.
At least, I'm not going to add a GUI to flower.lua.

Flower design philosophy is to make the library easy to use for everyone.
It is to provide a flexible mechanism.

GUI necessary for the game is different from a generic GUI.
Be created generically leads to code bloat, you will produce extra bug.

I think rather than providing a generic GUI, you had better to create a GUI needed for the game is good.
