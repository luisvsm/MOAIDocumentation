Re: A new forum...

by dooms101 » Thu May 03, 2012 4:31 pm

I really like the idea of using a StackOverflow style implementation. However, that is for a purely Q&A purposed site, which this site is not and should not be. I've been using StackOverflow for some time, and although it is great for getting my questions answered or finding an already answered question, it does not encourage discussion or community building that is essential to this site.

The argument that "90% of the content in here are questions" may be valid, I am not arguing that. But, I think it only points out a more subtle flaw in the existing forum system in place here. People often first come to a forum in search of answer to a common question, so someone's first post is almost always asking a question that has been asked before. Adding some sort of flag system that marks a thread as "Open", "Answered", etc would greatly reduce the amount of redundant threads here. The system would best be enforced by moderators but should be available to the OP as well.

So IMHO I think a 'traditional' forum system should be used for simple fact that they work well and have been proven work well for many diverse online communities. As for what specific software to use, phpBB has always been a favorite of mine and many forums I've frequented over the years have relied on it. I don't see the community here as having any special needs above other online communities that use 'traditional' forums software, so I say stick with what is proven to work.
