Code: Select all
  1. ----------------------------------------

  2. -- in main.lua

  3.

  5. function mainLoop ( pState )

  6.   MOAIUntzSystem.initialize ( )

  10. ----------------------------------------

  11. -- Sound.lua

  12. module ( ..., package.seeall )

  14. local bgm

  16. function playSound(pFileName, pLoadIntoMemory, pVolume, pLoop)

  17.   local result = MOAIUntzSound.new ()

  18.   result:load( pFileName, pLoadIntoMemory )

  19.   result:setVolume( pVolume )

  20.   result:setLooping( pLoop )

  21.   result:play()

  22.   return result

  23. end

  24.  

  25. function playSFX(pFileName)

  26.   local loadIntoMemory = false

  27.   local volume         = 2

  28.   local loop           = false

  29.   sfx = playSound(pFileName, loadIntoMemory, volume, loop)

  30.   return sfx

  31. end

  32.  

  33. function playBGM(pFileName)

  34.   local loadIntoMemory = false

  35.   local volume         = 0.5

  37.   if bgm then

  38.     bgm:stop()

  39.     bgm = nil

  40.   end

  41.   bgm = playSound(pFileName, loadIntoMemory, volume, loop)

  42.   return bgm

  43. end

Re: Untz Sound on Android

by thebullno1 » Sat Sep 15, 2012 11:33 am

If you look at the Untz source code, you will see that it doesn't even have MP3 decoding built in.
Untz uses C/C++ to decode WAV and OGG and push the raw data to the platform's playback API.

On platforms like Windows, Mac, iOS, it also uses the platform native decoder so MP3 is supported. (and a lot more like aac or wma)

MP3 is a problematic format with its patents anyway.
