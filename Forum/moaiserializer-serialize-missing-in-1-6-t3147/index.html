Re: MOAISerializer:serialize() missing in 1.6

by orkj » Sat May 30, 2015 3:48 pm

Just a quick reply in case anyone has the same problem.

This seems to fix a port from 1.5 to 1.6 for me:

Code: Select all
  1. -    serializer:serialize ( self.data )

  2. -    local gamestateStr = serializer:exportToString ()

  3. +    local gamestateStr = serializer.serializeToString (self.data)



(that's a diff and not a code sample, if you were wondering)

Re: MOAISerializer:serialize() missing in 1.6

by naturally » Sat May 30, 2015 4:13 pm

Thanks, I have a solution now:

Code: Select all
  1. function Menus:SaveSettings()

  2.         Menus:PullSettings()

  3.     local level_file = Game.documentDirectory..'settings.lua'

  4.     local serializer = MOAISerializer.new ()

  5.         local user_data_Str

  6.         if serializer.serialize then

  7.                 serializer:serialize ( Engine.settings )

  8.                 user_data_Str = serializer:exportToString ()

  9.         else

  10.                 user_data_Str = serializer.serializeToString (Engine.settings)

  11.         end

  12.         local user_data_file = io.open (level_file, 'wb' )

  13.         if (user_data_file ~= nil) then

  14.                 user_data_file:write ( user_data_Str )

  15.                 user_data_file:close ()

  16.         end

  17. end



This works across both 1.6 and 1.5
