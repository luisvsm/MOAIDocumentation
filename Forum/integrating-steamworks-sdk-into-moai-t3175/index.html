Integrating Steamworks SDK into Moai

by sshukul » Wed Jul 15, 2015 4:12 pm

So my first game finally got greenlit on Steam and I was looking to integrate some Steam features like Achievements, Leaderboards etc. using their SDK before release. This would be similar to Moai's existing Google Play Game Services and Apple Game Center integrations, except this would be for the desktop builds rather than the mobile ones.

My question is how would I go about adding something like that to the moai codebase in the "standard" way? Here are the basic instructions they have:

First, download the Steamworks SDK.
Unzip into a folder relative to your source tree
The following files need to go in your run-time directory (next to your game executable, or in your the dll search path)
steam_api.dll (this is redistributable with your game )
The following library needs to be included in any project you access
steam_api.lib
The following header files need to be included in your C++ project
steam_api.h
steam_gameserver.h (for multi-player games only)
Build!


So do I unzip the SDK into moai's 3rd party folder, then make a CmakeLists.txt file and tie these into the build scripts etc. somehow? And the integration code that calls the API functions would go into some new classes like src/moai-steam/MoaiSteam.cpp and MoaiSteam.h or somewhere? And how would these be tied in to the host builds using visual studio / XCode etc.?

Any help listing all the major steps would be appreciated as I'm still a novice in C++ / Cmake etc. and haven't done a third party integration like this into Moai before. Thanks.
