The TMXMapView of Hanappe, drawing layer is always divided.
This had disadvantages some performance basis.

TileMap of Flower was change in the Group instead of Layer.
Therefore, it is possible to arbitrarily set the drawing layer.

When using a single Layer, and excellent performance.
However, circumstances might be worse in some cases.
In this case, can also be modified to set a plurality of Layer.


TileLayer, TileObjectGroup is inherited classes of mere Group.
It is possible to add another object to the Group.

Code:
  1.  

  2. image = flower.Image("cathead.png")

  3.  

  4. tileLayer = tileMap.mapLayers[1]

  5. tileLayer:addChild(image)

  6.  

  7. tileMap:updateRenderOrder()

  8. -- Drawing order, it is possible to arbitrarily set the Priority

  9.  



In this case, you must manage the objects they create.
I should create a TileMap of the game for you.

Code:
  1.  

  2. GameMap = class(TileMap)

  3. ...

  4.  



In addition, it is as follows: If you want to generate a TileObject normally.

Code:
  1.  

  2. local objectGroup = tileMap:findMapLayerByName("Object")

  3. local object = objectGroup:createObject({

  4.           name = "Enemy",

  5.           type = "Actor",

  6.           shape = "rectangle",

  7.           x = 192,

  8.           y = 448,

  9.           width = 0,

  10.           height = 0,

  11.           gid = 538,

  12.           visible = true,

  13.           properties = {

  14.             ["playAnim"] = "walkDown"

  15.           }

  16.         })

  17.
