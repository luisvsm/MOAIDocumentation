White lines rendering between tilemap borders.

by wincekjb » Sun Feb 12, 2012 7:34 pm

Hello, I am using a tile map on iOS 5 iPhone 4, my tilemap is 15x45 tiles with each tile being 64x64 pixels. There are mainly two colors in the tilemap, green and blue, with no transparency.

I've been moving the tilemap with tilemap:movLoc(x*-20,y*-20,0.033333332, MOAIEaseType.FLAT) where x and y are the accelerometer values. That's in a function that runs every time the accelerometer propagates an event which is set at 30Hz.

When the lines occur seems to be somewhat random, they don't follow any sort of easily noticeable pattern or period. But two things I did notice is the lines only happen in between the green parts of the tiles and the lines are also only in the horizontal border between tiles.

I tried moving the character and the camera instead of the map itself, same issue.
I tried putting the tilemap on a different layer from the character, same issue.
Making sure the thing moving always moves at least 1 pixel when it moves doesn't change anything.

When searching the forum to see if anyone else had this issue I found this post about adding content to the wiki. Item 2 under gamer programming seems to suggest that maybe this is a known issue that just hasn't been documented publicly yet.

Re: White lines rendering between tilemap borders.

by patrick » Wed Feb 15, 2012 9:51 pm

Yep. It's caused by the way OpenGL samples textures. The solution is to add 'bleeds' between the tiles in your texture. You can do this with the bleedRect () function in MOAIImage. Once you've set up the bleeds you then configure the MOAIGrid used by the your tile deck to leave a 'gutter' between each tile - your grid cell size will be 2 pixels larger than your tile size. So if you wanted 32x32 pixels tiles in your map, your tile deck will use a cell size of 34x34 with a tile size of 32x32 which gives you a 2 pixel gutter and room for a 1 pixel bleed around each tile.

It sounds harder than it is. I'll ask someone on the studio team to post an example if they have time.

Re: White lines rendering between tilemap borders.

by jon » Wed Feb 15, 2012 11:12 pm

As Patrick says, it's not as hard as it sounds. I wrote this function which does the trick pretty well:
Code: Select all
  1.  

  2. function TilesetBleeder ( inputfile, imgwidth, imgheight, tilewidth, tileheight, columns, rows )

  3.         local img_in = MOAIImage.new ()

  4.         img_in:load ( inputfile )

  5.        

  6.         local img_out = MOAIImage.new ()

  7.         img_out:init ( imgwidth, imgheight )

  8.        

  9.         for r = 0, rows - 1 do

  10.                 for c = 0, columns - 1 do

  11.                         local srcx = c*tilewidth

  12.                         local srcy = r*tileheight

  13.                         local dstx = 1 + srcx + ( 2 * c )

  14.                         local dsty = 1 + srcy + ( 2 * r )

  15.                         img_out:copyBits ( img_in, srcx, srcy, dstx, dsty, tilewidth, tileheight )

  16.                         img_out:bleedRect ( dstx, dsty, dstx+tilewidth, dsty+tileheight )

  17.                 end

  18.         end

  19.        

  20.         return img_out

  21. end

  22.  

  23. out = TilesetBleeder ( "test.png", 256, 256, 64, 64, 3, 3 )

  24. out:writePNG ( "test_bled.png" )

  25.  



This takes the tileset image "test.png" (which is a 3x3 tileset of 64x64 tiles) and outputs it to "test_bled.png".

Once I have my bled tileset, I can load it like this:
Code: Select all
  1.  

  2. tiledeck = MOAITileDeck2D.new ()

  3. tiledeck:setTexture ( "test_bled.png" )

  4. tiledeck:setSize ( 3, 3, 66/256, 66/256, 1/256, 1/256, 64/256, 64/256 )

  5.  



And there you have it, no more shimmering edges. I hope this was of some use to you.

by ezraanderson » Mon Jan 28, 2013 3:25 am

This is really useful however I had 2 small problems.

1.
Problem: img_out:init ( imgwidth, imgheight ) would not work for me every time the image would come out as a blank white png,
solution: Load an image then it worked fine --> img_out:load ("512.png" )

2.
Problem: This loads the tiles reversed for me --> tiledeck:setSize ( 3, 3, 66/256, 66/256, 1/256, 1/256, 64/256, 64/256 )
solution: Reverse the tiles with --> tiledeck:setUVRect(-0.5, 0.5, 0.5, -0.5)


Edit Number 1 was caused, because I did not have a buffer row. You cant convert a full 512x512 tilesheet to a 512x512. You would need a 32pixel row on the right and on the bottom.

I did the first pass at adding the function bleed_tileset() in to my map-maker.

  • Download --> http://ezraanderson.com/moai/bleed/bleed_up.rar



1. Add text file --> into --> \tileset\mytilesheet.txt
Code: Select all
  1.  

  2. input=C:\Documents and Settings\Administrator\Desktop\bleed_up\tileset\tileB.png

  3. output=C:\Documents and Settings\Administrator\Desktop\bleed_up\output\tileB.png  

  4. tile_w=16

  5. tile_h=16

  6.  




  • Picture A: This is how I set up my tilesheets --> image 512x512 --> tile area 480x480
  • Picture B: If your tileset set is setup like this --> image 512x512 --> tile area 512x512 --> un-check the clip button and the tilesheet will be bleed to a 544x544 image.

Notes:
  • Always backup your tileset before using
  • Works fine for my needs
  • Should work with all equal dimensions (256X256,512x512,1024x1024,2048X2048)
  • Have not tested it when width & height dimensions of the tileset are not the same (512x1024)
  • Have not tested it when width & height dimensions of the tiles are not the same (32x64)
  • Its built for windows users but if I have time I will build it with delphi-firemonkey for osx users
  • the grid-check-box just separates the tiles but doesn't bleed them, could be used for adding borders
  • I just set the output, in the text file to export the tilesheet to where my MOAI game is
  • Haven't tested it on nvidia-cards but it should work

    Source code:
    • Unit1.pas --> included in download
