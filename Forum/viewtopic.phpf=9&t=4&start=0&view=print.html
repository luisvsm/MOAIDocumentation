Here's another tutorial request: I'm a little stumped on how MOAI's use of timed animations should interact with a game loop.



My (admittedly limited) experience on 2D game animation involved putting a call to character or object animation in the game loop, where it's called every frame or loop step, and moving the object a tiny amount. Then I can tweak that small movement number to get the movement to look right. MOAI's timed animations are confusing me here, I'm not sure how to go from



"every loop, if the player is pressing X, move the cathead 3 pixels to the right"

to

"every loop, if the player is pressing X, move the cathead to a certain point over a set period of real time"



Maybe these things aren't supposed to work together? In the Rocket Lobster example, when the player fires a missile, once the missile has its target, it just does its "seek target" process for a few seconds, and the game loop doesn't really have to pay attention to it anymore. I guess I'm wondering how to update the speed, direction, etc. of an animation from one loop step to the next when the animation is supposed to play out over a set timeline. I'm interested in seeing an example of a loop that continually takes player input and adjust the animation parameters to that input.



Hope this isn't too basic a question - I have a lua book arriving in the mail this week, maybe there are basic structural things about lua I just don't have down yet.

You can take either approach depending on what you're comfortable with. In fact, the first draft Rocket Lobster tutorial used stepwise animation instead of the animation helpers.

Try creating a MOAIThread and in the main function use prop:getLoc () and prop:setLoc () to move your objects. The default simulation step is 60fps, but there are methods in MOAISim to adjust that.
