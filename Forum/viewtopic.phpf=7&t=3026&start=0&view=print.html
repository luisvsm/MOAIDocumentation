1.6 Windows/OSX Binary SDK package (preview)

Posted: Mon Jan 19, 2015 6:21 am
by halfnelson
I have created a binary sdk release for windows. This release contains all moai modules precompiled, a moai host for windows and host templates for all supported platforms on windows (android, html, vs2013):

https://www.dropbox.com/s/aug4knz8y1imdio/moai-sdk-binary-windows.7z?dl=0

I have one for OSX too (ios, osx app store, android, html):
https://www.dropbox.com/s/s4e4r6b677e59fe/moai-sdk-binary-osx.tar.bz2?dl=0

There is a moaiutil command called "host" that allows the hosts be generated for your project and to customise the build. Checkout the documentation in the readme.md file in the zip or here https://github.com/moaiforge/moai-sdk/blob/binary_release_1.6/util/package-cmake-sdk/moai-sdk/Readme.md

short version:
with your lua in "testproject/src":
  • add "<sdk-root>/util" to your path
  • from testproject folder run "moaiutil host init"
  • Edit the resulting hostconfig.lua.
  • You can build with "moaiutil host build windows" or "moaiutil host build android-gradle" or "moaiutil host build html".
  • You can run with "moaiutil host run windows" or "moaiutil host run android-gradle" or "moaiutil host run html"
[*]List available hosts with "moaiutil host list"
All host projects are created in the hosts subfolder, and can be edited and changed from the relevant ide (android studio, visual studio 2013, xcode)

This release is geared towards someone who just wants to run their lua on as many platforms as possible, and isn't interested in editing the moai source, or creating a complex custom host etc.

halfnelson wrote:What do you think of the workflow (where each project has its own set of per platform host projects that can be regenerated from config file). Does it sort of match what you do at your studios? If not, what is your workflow for managing multiple apps


At the moment we do what I guess most of us do and that is have separate host 'templates' where we just copy over the lua files and build the hosts. Most of what changes between the apps is the lua files and associated assets. The peripheral stuff, chartboost, game centre etc is all in our custom libraries which we just include.

We are currently migrating all of our apps from 1.3 to 1.6 so we have a lot of custom stuff at the moment which has been superseded by some of the 1.6 works.

One slight change for us is that we ship with the @ezraanderson android host as we find the out of the box performance not acceptable. Even with 1.6 improvements we couldn't get the performance we get out of Ezra's host.

The turn key template approach is definitely the way forward.
