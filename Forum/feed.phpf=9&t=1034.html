]]> 2012-07-28T01:29:59+00:00 2012-07-28T01:29:59+00:00 /forums/viewtopic.php?t=1034&p=5603#p5603 <![CDATA[Re: building the linux branch]]>
Just to explain this. There's no installation script since the expected use is that you will ship the Moai binaries with your game anyway, so it won't install to a global area. This is due to the fact that repositories and resolving dependencies on Linux is a real pain since you have to do it for every individual distro.

You will need to include the compatibility layer for most of the samples to work since they haven't been updated yet. In addition, although Moai normally automatically includes the compatibility layer, since there's no step in the build process to copy it to the output directory, the file doesn't exist when you run Moai and hence it doesn't get automatically included under Linux. As pygy79 showed, you simply have to include it manually on the command line, along with the config file and the name of the sample / game you want to run.
