by BacioiuC » Sun May 04, 2014 1:45 pm

If anyone plans on using moaicli (latest release) with moai 1.5 and moaigui you need to update the pathjoin function (/gui/support/fileSystem.lua - line 38) to:
Code: Select all
  1.  

  2. function _M.pathJoin(_one, _two)

  3.         local s = ""

  4.         local arg = { }

  5.         arg[1] = _one

  6.         arg[2] = _two

  7.         for i, v in ipairs(arg) do

  8.                 s = s .. v .. "/" --MOAIFileSystem.getDirSeparator()

  9.         end

  10.         return

  11. end

  12.  



For some reason in 1.5 you cannot use ... to pass multiple arguments without having them specified. Took me a while to realise this is the problem that kept crashing my game on startup on 'droid. Same thing applies to any host build with moaicli.

by halfnelson » Sun May 04, 2014 9:56 pm

This is not really a moaicli issue, it has to do with luajit. Luajit supports and implements the lua 5.1 syntax for varargs :
http://www.lua.org/manual/5.1/manual.html#2.5.9 but not the deprecated lua 5.0 syntax. (lua 5.1 that we use still supports the old syntax as well as the new syntax apparently this is against the spec!)

so you can solve the problem by using '...'

Code: Select all
  1.  

  2. function pathJoin(...)

  3.         local s = ""

  4.         local arg = {...}

  5.      

  6.         for i, v in ipairs(arg) do

  7.                 s = s .. v .. "/" --MOAIFileSystem.getDirSeparator()

  8.         end

  9.         return s

  10.  

  11. end

  12.  

  13. print(pathJoin("one","two"))

  14.  



You still need to be careful when you can possibly get 0 arguments, see this discussion here /forums/post13843.html?hilit=args#p13843

I am glad you managed to get up to 1.5, let me know of any show stoppers or annoyances in either moaicli or 1.5

not sure what is broken with your vs2010 compiler (moaicli is tested as working with vs2012), the latest moaicli defaults to visual c++ compile, to build for mingw (like the old version) you can add --platform mingw to the build command line
Code: Select all
  1. moaicli build sdl --platform mingw



There was a massive refactoring to improve startup time before the 1.5rc2, looks like I left out a require, I will look into it and roll out a new build, the sledge host should still work after I fix that.
