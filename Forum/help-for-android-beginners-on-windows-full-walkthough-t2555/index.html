Help for Android beginners on Windows (full walkthough)

by molul » Sun Mar 02, 2014 10:38 am

After a couple of days getting a hard time with starting to work on an Android test, I'd like to post my steps and mistakes, in case it helps somebody else. This is everything I needed to do to be able to run my Windows Moai game in my cellphone (not sure if it will work for every cellphone, mine is Samsung Galaxy S3 Mini).

MY SETUP
I'm working on Windows 8.1, and so far I've worked just with the binaries posted here (Moai 1.4p0). My dev path is C:\Users\luis\dev and those Moai binaries are installed in C:\Users\luis\dev\moai-sdk.


GETTING MOAI SOURCE CODE
First, I downloaded the 1.4p0 source code from here and copied it to C:\Users\luis\dev\moai\sdk\src (the final path would be C:\Users\luis\dev\moai\sdk\src\moai-dev-Version-1.4p0). If you need another version -> here.


SETTING UP YOUR PC
After some reading, I think this was the most useful guide, but it was a bit disordered and lacks some important info ( it was more useful than the official guide in the wiki, though). So here is what I would do step by step:

1- First thing to do is installing Cygwin. This part of the guide lacks a couple of things: 1) The installer must be the x86 (or at least I couldn't make it work with x64 version), and you can get it from here. The install process then goes as the guide says (it veeeeery long after you select the Devel package), and finally 2) I'd add one thing to the process: set an environment variable for Cygwin binary path. I added CYGWIN_BIN = c:\cygwin\bin .

2- Then I went to installing Java JDK (Cygwin would finish long after I did the rest of these steps, so you can go on as well). The guide is fine here except for the installer. Download the Windows x86 version from here. Install it and do the JAVA_HOME thing as said in the guide.

3- Now Android SDK. This is perhaps the blurriest part in the guide I've followed. Here's the link. Instead, do this:

a) Download the Android Developer Tools (ADT) 32bit (x86) bundle: http://developer.android.com/sdk/index.html#download

b) Follow the ADT setting up guide until "That's it!".

c) Go to your adt bundle directory, run SDK Manager and configure it as described here BUT choose the latest Android instead of 4.0.3 (mine was 4.4.2): http://developer.android.com/sdk/instal ... kages.html

4- Now Android NDK. This part of the guide is almost perfect. I'd add that you have to download the x86 version, and that you better add the new paths to PATH at the beginning, not the end (at least is one of the three things I've done to be able to keep working).

5- Finally, installing Apache Ant. You can trust the guide here


BUILDING THE ANDROID HOST
Now go to the beginning of the guide and skip the blue links. Go to "Creating an Android Host" but don't follow the first steps. Instead, go to your Moai source code directory, open "ant" and edit "make-host.bat". Replace the code with this:

Code: Select all
  1.  

  2.         @echo off

  3.         setlocal

  4.         echo %CYGWIN_BIN%

  5.         set CYGWIN=%CYGWIN_BIN%

  6.         %CYGWIN_BIN%\bash.exe make-host.sh –p com.mygamename.moai

  7.         @pause

  8.  



Then run make-host.bat and get back to the guide where it says "Once the script completes, it will have created a directory in the /ant folder called untitled-host.". Right after that, some Android SDK config is described. As I have downloaded the ADT bundle, I have the Android SDK path set as this:

android_sdk_root="/cygdrive/c/luis/dev/adt-bundle-windows-x86-20131030/sdk"

Make sure the "src_dirs" path has no spaces and follow with the settings-global.sh. I edited it to have just " requires=( "miscellaneous") " and added info about my app. Then I run "./run-host.sh". If you get an error, doble or triple-check the paths you typed in settings-local.sh (I had to triple check the src_dirs ).


RUNNING THE PROJECT IN ECLIPSE
I followed the rest of the guide and got stuck in the moment of building the project in Eclipse. First I got an error like "can't find target android-10" or something similar. To solve it, I right-clicked the project, then go to Properties, and in "Android" checked my "Project Build Target" (in my case I chose "Android 4.4.2").

Now I'm asked about creating a new Android Virtual Device because no Android Device was detected. I don't want to do it now, so I stop to add my Samsung Galaxy S3 Mini.

First I have to enable debugging in the phone (go to Menu->Settings->Advanced. Turn the toggle at the top on if it's not, then check "USB Debugging"). Then I think I just run the project in Eclipse again and my phone appeared in a menu (if not, I don't remember what I did but I'm sure it's on Google). I double clicked it and, after loading some stuff, it started running.


IT'S MY GAME!
...But no. There seems to be some bugs in my code (references to keyboard and other Windows stuff, something to be expected), so I can't get to the end, but at least I got the environment set! I'll try to edit the code later just to see the app running a bit, and then I'll go back to the main development branch (Windows).

I hope this guide works for some other newcomer

EDIT: after a few hours of fixing problems with resolutions and mouse/touch screen setting, I've finally seen my game in my phone as I see it on Windows, but without sound. I think accepted some option in Eclipse to route the app output through Windows (quite weird). I'll investigate on that tomorrow

Nice post! I tried to get MOAI and Android working under Windows last year, but in the end simply abandoned it as a bad effort. I've been developing under Ubuntu since with no difficulty at all. And it doesn't require Eclipse either, I'm using ZeroBraneStudio.

I found that under Linux, the Android SDK, NDK and the MOAI SDK just plain work. There is minimal set up compared to the anguish you'll feel trying to get it running under Windows. If you don't want to dual boot the other option of course is an Ubuntu VM.

I've also found that the Android Emulator is basically annoying and broken when it comes to MOAI, at least on my Nvidia GPU laptop. Clicking 'use host GPU' results in the emulator doing nothing under Ubuntu for some reason. I've just abandoned it and do my device testing on actual devices, although of course I only have a small set of those in the house.
