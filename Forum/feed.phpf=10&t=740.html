Note that this->mPrimSize should be 0 for GL_POINTS.

The second bug is a bit harder to spot. In fact, if the only primitive you draw is a point array then it works. This is because MOAIGfxDevice->mPrimType is being initialized as GL_POINTS but inside of MOAIGfxDevice::SetPrimType a check is being made

Code:
  1.  

  2. if ( this->mPrimType != primType ) {    // line 1157 of MOAIGfxDevice.cpp

  3.  



that causes critical code to be skipped over.

The solution is to initialize mPrimType to a truly invalid value like -1

Code:
  1.  

  2. mPrimType ( -1 ),       // line 748 of MOAIGfxDevice.cpp

  3.  



Be aware that the type of mPrimType is GLenum and thus you will get signed/unsigned warnings if you put in a -1.
