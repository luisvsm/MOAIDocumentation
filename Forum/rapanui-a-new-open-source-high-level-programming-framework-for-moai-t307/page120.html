Hi,

>I was just wondering what the process was for making contributions to Rapa Nui. I am working on a ScrollView similar to >RNListView that scrolls in both x and y.Is this something that would be valuable to include in Rapa Nui?
It seems great! (a cell scroll handler which can scroll in every direction, autoresizable and everything)
Sooner or later we should have done this
>And if so, what is your preferred process to get this into the main branch?
Everything which is in rapanuidev will be merged into the master branch, so simply open a pull request in rapanuidev.
Just be sure to create a sample of it in rapanui-samples/list to let us see it on action, too

Also, if you want to add it as a new native "Foo" element in RapaNui, remember that each native "Foo" element:
1) is named RNFoo and placed in rapanui-sdk directory
2) is loaded in rapanui.lua
3) has its own sample or samples in rapanui-samples.
4) we should have documentation comments for it ready for luadoc (we are sorry, again for the actual lack of docs).
(In this case you should know RapaNui well, to not make any mistake)

Else you can add it as an external scripts given out with rapanui which users can manually load like by creating a new folder in samples named like "contribs/foo" in which you'll add both your Foo.lua and FooSample.lua.
This way is a little bit free because it's simple a way to "extend" rapanui with external scripts.

Choose the way which suits you better.
