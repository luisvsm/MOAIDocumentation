newRect "helper" module

by patrick » Mon Oct 31, 2011 6:53 pm

That's right. Both the layer and the collision world keep a Lua reference to their members. If you remove the object from its layer and world then it should get garbage collected. Or if the layer and world are also set to nil...

One other note: the Lua GC will collect userdata objects in two steps, so I'm not sure if one call to collectgarbage is sufficient. There's a MOAISim.forceGarbageCollection that will run the GC in a loop until no more Moai objects get collected. We've been using a combination of the object allocation histogram and forceGarbageCollection to test for leaks.
