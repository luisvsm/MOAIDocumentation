Guide - How to setup MOAI SDK for Windows/Android

Posted: Fri May 11, 2012 8:52 pm
by bach
Hey everyone,

Well I wanted to setup MOAI on my machine today. Starting out, I was confident that this would be a relatively easy task. I mean, come on I've been doing this for years now. It's just another SDK.. Oh boy, let's just say I was in for a treat

To save the next person the pain I've been through I decided to write a nice little guide with which will allow you to run MOAI on your desktop as well as on your Android phone never even leaving the IDE. It also provides a nice little template project to work with.

This guide focuses solely on a Windows 7 dev environment. However I'm sure that adapting this to Mac and other versions of Windows shouldn't be a big deal.

I looked around for the best IDE for a while. After trying out many different ones (intellij, eclipse, etc.) I ended up with Sublime2. Unless someone can convince me otherwise, it's the best option for MOAI developing right now. I used a lot of different resources I found from around the forums, but mainly I made great use of Moai Nao made by thebullno1. I adjusted some of his scripts to create a nice workflow with Sublime2. Hope he's ok with it

Here we go:

1. Download and install Sublime2(http://www.sublimetext.com/2)

I assume this step explains itself. Have a look around the IDE and play around with the different color schemes under Preferences -> Color Schemes. You can also turn on folder view as a sidebar under View -> Sidebar -> Show Sidebar.

2. Download the Android SDK (http://developer.android.com/sdk/index.html)

The sdk just comes as a zip file. Extract it to a folder of your choice. For simplicity reasons let's take "C:\android-sdk". As a next step we need to add the android sdk platform tools to our PATH environment variable. In windows 7 this is done by going to Control Panel -> System -> Advanced System Settings. At the bottom of the window that pops up click "Environment Variables". Under "System" (the bottom list) find the "Path" variable and edit it. Add this to the end of the string (DON'T replace):

";C:\android-sdk\platform-tools" (minus the quotes, but it needs the semi-colon)

3. Download the MOAI SDK (https://dashboard.moaicloud.com/signup.php)

Login and download the sdk. Again this is not an installer, just a zip file. Extract the zip file to your location of choice. For this guide let's use "C:\moai-sdk".

Now, let's have a quick look at the sdk folder structure:

/3dparty (some glut libs)
/bin (this folder contains all the hosts (i.e what will run your lua scripts)
/docs (the api reference guide)
/hosts (has stuff in there for hosts building, we'll ignore in this guide)
/include (some lua includes)
/samples (the sdk samples)

Before we go any further, you should go into the samples folder and run some samples. This will work without any extra setup. Just go into any samples folder and double click the run.bat file and it'll execute the sample. Familiarize yourself with all the different functionalities the sdk has to offer. There are a lot of samples!

Next we need to add the sdk to the "Path" environment variable. Follow the instructions in point 2. and add the following to the end of the string:

";C:\moai-sdk\bin\win32" (again, minus the quotes)

4. Install 7zip (http://www.7-zip.org/)

This is needed for the Android deployment. Just run the installer. By default it'll install to "C:\Program Files (x86)\7-Zip".

Go ahead and also add this folder to the "Path" enviroment variable.

";C:\Program Files (x86)\7-Zip"

5. Download my project template (http://www.thegreystudios.com/moai/project-template.zip)

This zip file gives you a clean basic project that you can use to work off. It has everything ready setup for deploying to Android (using Moai Nao). Of course you can also use this to test on and develop on desktop.

Alright, extract the zip file to a folder of your choice. Have a quick look at the extracted folder structure. You should have a folder structure like this:

/main
../assets (some test asset)
../main.lua
../config.lua
/tools(various batch files and txt files in here)

6. Get the project running on Desktop

Now the fun can begin. Open Sublime2 and go to File -> Open Folder. Navigate to the location you just extracted the template to and open up the root folder (the folder that has main & tools in it).

This will open up the project template in Sublime2. You should have the above folder structure visible on the left now (make sure you have the sidebar visible, View -> Sidebar -> Show Sidebar).

The next thing we need to do is define a build for MOAI-Desktop. You only have to do this one time. This will be saved with your Sublime2 install so all subsequent projects you create will have it already.

Go to Tools -> Build System -> New Build System .. This will open up a new file. Replace the text with the following:
Code: Select all
  1.  

  2. {

  3.         "cmd": ["${project_path:${folder}}/tools/run.bat"]

  4. }

  5.  



Press save and specify an appropriate name for the build, i.e MOAI-Desktop. Verify that the new build configuration has been selected by going to Tools -> Build System -> (x) MOAI-Desktop.

Now navigate to main.lua via the sidebar and open it by double clicking. You can now either start this by going to Tools -> Build or simply use Ctrl+B. This will start the script running in the windows host. If everything has worked you should seet the moai logo spinning in front of you.

7. Get the project running on Android

First, download MoaiNoa from here: http://www.host-a.net/u/bullno1/moainao.zip

The only thing we care about is the MoaiNao-debug.apk. Move that file over to your Android device somehow (via adb, or dropbox, gdrive) and install the apk.

Next, we need to connect the device to the PC via adb (Android Developer Bridge). You can do this via USB or wifi. Have a read through this if you've never done it before: http://developer.android.com/guide/developing/device.html#setting-up

I personally use adbWireless which allows a wifi connection to adb and gets rid of the annoying usb cable But you need to have your phone rooted in order for that to work.

Alright, assuming you have a connection setup we're going to create another Build System specifically for Android. Same process as for Desktop, Tools -> Build System -> New Build System

Replace the text with this:
Code: Select all
  1.  

  2. {

  3.         "cmd": ["${project_path:${folder}}/tools/deploy-to-device.bat"]

  4. }

  5.  



Press save and specify a name for the build system, i.e. MOAI-Android. Verify that it's the current build system via Tools -> Build System -> (x) MOAI-Android.

Navigate to main.lua and press Ctrl+B or Tools->Build. This should now automatically deploy your script to Android and run it. MAGIC! (not really)


And that's the end of this longer than expected guide. I hope this helps the next person trying out this incredible little SDK. Let me know if this has worked for you and if not where you got stuck!

I'm still only just starting out (first day today ) so I may have done things in a weird, uncommon way and I'm more than happy to be shown a better way. However I've found this setup works surprisingly well and allows for very quick testing on Android (<4s startup).

Thanks to thebullno1 who really made this possible.

Cheers,
Bach

No worries.

For those who end up in the same boat as myself, here's what I ended up doing.

1. Add an "ANDROID_SWT" environment variable. Have this point to the location for the swt file appropriate for your OS. These are located at "android-sdk/tools/lib/x86" and "android-sdk/tools/lib/x86_64". I recommend rebooting afterward, as Windows often doesn't recognize new environment variables otherwise.
2. Open a command window, and navigate to "android-sdk/tools". The easiest way is to hold shift, right-click on the "tools" folder in Windows explorer, and select "Open command window here".
3. Type "android.bat update sdk --no-ui", without the quotes. I got a couple errors at the beginning, saying it can't find a .jar file, and "The system cannot find the path specified." I'm not entirely sure how to fix these, or if they're important. However, after a few moments, it should start downloading the files. This will take a while.

However, while updating, it seemed to halt several times, so I've had to rerun the command. It looks as though its able to restart from where it left off. I'm still waiting for all the updates, and will report back what's happened.
