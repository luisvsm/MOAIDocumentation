Moai 1.0 beta roadmap

by todd » Sun Apr 03, 2011 4:56 am

The Moai 1.0 beta is expected to last 60 days.



Moai SDK features in the beta are:




  • Support for iOS & Android apps

  • Device compatibility

  • Font engine

  • Internationalization

  • 2D animation

  • Physics engine

  • Collision detection




Moai Cloud features for the beta are:




  • Individual Lua applications for your server logic

  • Database for persistent world state and app content




Items that didn't make the cut for the 1.0 beta but are being worked on include:




  • Moai SDK sound engine for iOS and Android

  • Custom app URL's for Moai Cloud

  • Moai Cloud API key integration

  • Moai Cloud Lua sandbox for testing in the browser




Got additional questions? Post them here!

Moai 1.0 beta roadmap

by patrick » Wed Apr 13, 2011 10:12 pm

Welcome to the forum. :)

1 – Moai can merge with Cocos2D inasmuch as you can link Moai SDK in with your Cocos2D app. That said, Moai SDK is really an alternative to a library like Cocos2D. I'll say more in the answer to 3.

2 – I think that's a great idea. Moai Cloud should make an ideal back end for your Cocos2D game. If you need rich logic and data persistence on the server, you can write it all in Lua and talk to it over HTTP. Up to you how to set things up and return your data.

3 – Moai SDK is similar to Cocos2D in a lot of ways. The differences are that Moai is written in 100% C/C++ and has Lua integration built in from the ground up. You can use Moai SDK as a C++ library (just as you might use Cocos2D as an obj-c library), but the quicker path is to use the Lua interface for most of your game setup and logic and to extend Moai using either Lua wrappers or by writing custom additions to the C++ framework. Even if you're a C++ expert, you can still put things together a lot faster using Lua and just go back to C++ as needed. 

The C++ implementation means Moai SDK can run natively on iPhone, Android, Mac, PC, etc. I think there's a current Cocos2D port to Android, but it's written in Java, so I'd have questions about performance.

But yeah, if you're comfortable with obj-c and iPhone, Cocos2D is fine. You can still use Moai Cloud to write your back end in Lua.
