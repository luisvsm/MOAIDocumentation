Re: Help needed! HTML5 host.

by halfnelson » Tue Oct 07, 2014 3:03 am

it is a change in the current emscripten version that gets upset if stdin doesn't have a filedescriptor of 0, stdout 1 and stderr 2.

This can happen if before emscripten spins up the filesystem, there are still files open (hence 0,1 or 2 are taken). So either the host code isn't closing the filesystem properly after writing its files before launching, or the filesystem function that opens the streams isn't getting called first.

I don't have time at the moment, but two workarounds are:
1. make sure createStandardStreams ( https://github.com/kripken/emscripten/b ... s.js#L1262 ) is called before moaihost starts writing to the filesystem, (you can breakpoint next_fd() (the function that gets next file descriptor) in moaijs.js if you can find it (turn off minification in the build script) and see what is calling it and not freeing it.

2. roll back to an earlier version of emscripten that is known to work with moai (from memory 1.8.0 was good, or the late 1.7.x)
