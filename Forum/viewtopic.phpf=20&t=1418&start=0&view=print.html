spacepluk branched the latest moai-dev, and added the ability to make the linux host with cmake. You can get his branch here:

https://github.com/spacepluk/moai-dev

Something like this:

Code: Select all
  1.  

  2. $ git clone https://github.com/spacepluk/moai-dev moai-dev.spacepluk

  3. $ cd moai-dev.spacepluk

  4. $ mkdir build

  5. $ cd build

  6. $ cmake ../cmake

  7. $ make

  8.

I'm going to guess that you don't have 'liblua5.1-0-dev' installed, you can check via:

Code: Select all
  1. aptitude search lua | grep 'i '



The grep is optional - i=installed, p=package available (there's a deliberate space after 'i ' in the grep statement). Issue a 'sudo aptitude install liblua5.1-0-dev' if you find you don't have the package installed.

Now, I know it'll seem like I'm pushing my spacepluk-linux-cmake Bash build gist 'moaiLinuxUnoffical.sh' again, however, you might find the package dependency part of my script useful.

Originally, I used spacepluk's own list of required Ubuntu pakages, as he outlines in his 'Compiling MOAI on Linux' blog post. His list of packages was:

Code: Select all
  1. # Install the dependencies:

  2. sudo aptitude install freeglut3-dev libglew-dev libglu1-mesa-dev libxmu-dev libxi-dev chipmunk-dev libjpeg8-dev libpng-dev libfreetype6-dev liblua5.1-0-dev libjansson-dev libtinyxml-dev libcurl4-openssl-dev zlib1g-dev libexpat1-dev cmake build-essential



To get my build script working, I had to add 3 more dependencies, libasound2-dev, libvorbis-dev,
and libpulse-dev:
