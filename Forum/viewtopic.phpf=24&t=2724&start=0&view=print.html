We use Flurry. Works good for basic things: user count, session length, audience info. It has support for custom events with parameters, but lacks any kind of querying mechanism. Sometimes we may want to know how many users buy more than one in-app or what was the last level that user played before he leaves the game for more than one week.
Flurry is completely free even for large loads of data.
