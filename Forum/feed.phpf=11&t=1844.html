Oh damn, I just realized the images I've posted no longer works due to some misfortunate event with my webserver. I need to find and upload those images again, it makes the process much easier.

The way I build host, I just run in Terminal following lines:
Code:
  1.  

  2. cd moai/xcode/libmoai

  3. bash clean.sh

  4. bash build.sh



Instead of moai/xcode/libmoai I put absolute path to the folder where libmoai.xcodeproj is residing.
This will first clean the build (if there is something built already) and then build all the targets of libmoai xcode project.
Before even running the build.sh script, I would normally edit it in text editor, to avoid building stuff that I don't want to use, namely facebook, fmod etc...So the relevant part of build.sh would be:
Code:
  1.  

  2. ios_schemes="libmoai-ios libmoai-ios-3rdparty libmoai-ios-luaext libmoai-ios-untz libmoai-ios-zlcore"

  3. ios_sdks="iphoneos iphonesimulator"

  4. ios_architectures_iphonesimulator="i386"

  5. ios_architectures_iphoneos="armv7 armv7s"

  6.  





The build process will create library files, and it will save it somewhere in /tmp/moai... folder (see the output in terminal to find the folder). In order to open that temp folder in Finder (because it will be hidden by default), simply copy the folder path and press CMD+SHIFT+G in Finder to open the "Go To Folder" dialog box and paste the path there. You want to look for Release (not Debug) folder, which will have following subfolders:
/armv7
/armv7s
/i386
/universal
Then I would normally copy the contents of that build folder (the 4 above subfolders) to somewhere on hard disk where I want to keep it. The content of folder /universal is the one I include in my XCode project (as in step 3 in my tutorial: Link Binaries) - I normally don't do anything with other 3 folders (maybe I should?)
In addition to that, univeral folder is what I want to point with using my MOAI_DIR_LIB Xcode variable.

As far as AKU headers go, I would normally point to the moai/include/ folder of the downloaded sources.

Now depending on errors you get, we might be able to troubleshoot.
