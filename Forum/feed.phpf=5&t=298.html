I would love to help implement this suggestion.  We can start by collecting the required information here, determine what is missing and what needs to be improved, and then update the Wiki and included documentation.

To me, one thing that gets a lot of new users is the fact there is a Moai SDK with precompiled libs in the Dashboard (which I use and am grateful for), and the full, latest, beta project in GitHub — and they each have different set ups and issues.

To begin,

(1) How was your attempt to learn how to start, and where did you feel it should be improved? 

(2) I recently posted 9 steps (click here) to run the samples in a Mac.  Would something like this satisfy you, and if not, how could it be improved?

(3) Regarding libraries, I myself would like more feedback in this area.  For example, I require MD5, and there is a lua library for it (click here).  However, I am not completely excited about implementing it since my Apps are for iOS, Android, Chrome, OSX, and Win32.  I believe I would have to add it to each platform's project — this makes me wish for a Lua Script version, so I know it would work on all platforms with no implementation issues.  Perhaps for me, this MD5 issue is the perfect gateway to document creating Lua libraries.  There is the Extending Moai SDK section. 

I look forward to any feedback.
