Re: How to build an Android APK , step-by-step

by ibisum » Mon Sep 03, 2012 12:47 pm

Did you use the make-host.bat script to make a new Android project for your sources, using the Template? "MyDomain" and "MyApp" look like the wrong names for these paths .. they would better be something like "src/com/SH_Black_Hole/some_app_name/MoaiActivityJava" .. where "some_app_name" is of course a proper name.

In general, you must use 'make-host' for every Android project target. This script file creates, from a non-working Template (*), a new Android project, initially in a folder named "untitled-host/", although this can be changed. The sources in that folder are *then* usable to build the MOAI Executable for Android, bundle it with some Lua sources, and execute as a MOAI App - it is a project directory which can both be imported into Eclipse, *OR* built from the command line. Ideally, you wouldn't need to mod much Android stuff in this project folder, and use it simply as a host for MOAI, but you can if needed, modify the onboard Android OS host code, as well. Ideally, not: you're just using it to build your MOAI executable for Android, and not much more than that.

This Android project, named and path'ed as you specify in the "make-host.bat" argument list, also at some point needs to point to your MOAI Sources for the project you are building, because it is a real Android project to build the moai executable and then package it up into a working .apk for distribution on Android, doing absolutely nothing if not given a main.lua file for execution, as well as all your images and whatnot that compose the basis of your MOAI Application.

Please note that the bundling of your sources into the Android APK can go wrong, even if everything else is okay, meaning that you can build an Android project successfully, yet still get nothing on-screen, simply because your Lua sources are missing/not-working.

So, there are few bits that need connecting. In your case, "MyDomain" and "MyApp" don't seem right.

So, for me every time I want to start an Android build for a MOAI project, it goes a bit like this (on Linux):

Code: Select all
  1.  

  2. ~/moai/moai-dev/ant/ $ #My project name is 'fot' ..

  3. ~/moai/moai-dev/ant/ $ ./make-host.sh -a all -p at.w1xer.droids.fot

  4.  



(This will produce an "untitled-host/" folder .. I then rename untitled-host/ to "fot-android-host/", move it to my buildserver, and edit the local configuration files (settings-global.sh/settings-local.sh), before then doing:

Code: Select all
  1.  

  2. ~/moai/build-android/fot-android-host/ $ ./run-host.sh

  3.  



.. to test THE BUILD FIRST.

If I get a spinning MOAI logo, I'm all good, because the Template is set up to run a demo. So I then edit "moai-target", point it at my project code sources instead, and re-build .. making sure of course to be watching logs with 'adb log' and so on, along the way.

If you can do something similar with the Cygwin+Windows side of things, sure it will work. I guess you just have to get through the 'unix-isms' of the MOAI build setup as it currently stands (I think most MOAI core devs are MacOSX/Linux guys, with a few Windows bravehearts sort of straggling along.. it really is very easy to do cross-platform dev with MOAI, as long as you're using either an OSX or Linux host somewhere in your development chain ..

(*EDIT _ Re: Template is non-working - yes, the sources in $MOAI_HOME/ant/host-source *do not work*. If you find yourself trying to do anything with those files, in Eclipse, or whatever, struggling to get an Android project working, forget it. You *HAVE* to use make-host.sh/make-host.bat to generate a new project, from the Template sources in this folder. If you monkey with $MOAI_HOME/ant/host-source, the only possible reason to do so, is to make modifications to the Template for all future invocation of the "Lets Start This Project On Android" #1 script: make-host.[sh,bat])

EDIT2: "what is "../../include/lua-modules"" .. this is a remnant from earlier invocations of sample code, I believe .. "lua-modules" isn't strictly part of MOAI, its just a place for lua addons .. I for example, have lua-enumerable.lua in my modules list, and anyone can hard-code other paths at the end of this if they also have external Lua dependencies .. thus I believe that this is 'left in' as a hint that one need not only have a single path to sources in the config ..)

Because:

Re: How to build an Android APK , step-by-step

by ibisum » Sat Sep 15, 2012 1:43 pm

When you use 'make-host' to make an Android build project from a template, you inherit the setup of the Android workspace and Manifest, which isn't maybe set up for how your app is supposed to work: and thus you should *always* double-check the host configuration/setup as a last-step when making the template work for a build for your App. If you need Landscape orientation, don't assume that the MOAI host for Android is set up for that; same is true for iOS, and all other host build projects that typically go into making a cross-platform release.

MOAI runs under the constraints of the Host, in a little GL ES window. The orientation and physical configuration of the device is done in the Host level, MOAI then just plays along. If all looks good in your MOAI code, but yet you're still getting dizzy results, head into the Host project and double-check, nay triple-check, all your expectations of behaviour.

Code: Select all
  1.  

  2. -- get MOAI's settings

  3. screen_width = nil or MOAIEnvironment.horizontalResolution

  4. screen_height = nil or MOAIEnvironment.verticalResolution

  5.  



There are many other things in MOAIEnvironment you'll probably want to know about, such as device features and so on, so worth digging deeper. Also, beware: it had some refactor'ing that didn't make its way to samples, fields have been changed, so use the docs, bruce .. but also check the code.
