by Galdred » Wed May 20, 2015 9:38 pm

I am working on

Name : Zodiac Wars (temporary name)
Summary : PC game, mixing tactical RPG and strategy, with a strategic map and battle taking place on a tactical map. . You can think of it as a medieval fantasy X-Com. It is in isometric perspective, using hexagons, and pixel art.
The sprites have been made with graphicsgale(characters and animations) or gimp (tilesets).
The textures have been assembled with TexturePacker
The levels are built with Tiled (and a very heavy set of autorules).
MOAI version : 1.5 + flower
Status : Basic level building, and game functions (movement, combat, but no skill yet, and very little items) are in, as is the paperdolling system.
I still have a lot to implement : lighting and shadows (using spriteIlluminator), quest objectives, AI, strategic level layer, UI.
I plan to launch a kickstarter campaign in October.

The website is not fully functional yet, but it should be soon :

I'm doing a couple of small MOAI projects, as well as an embedded/C++ project on the side, mostly these days. I wish I could find a way to do more MOAI gigs locally here in Vienna, but alas .. as we all know .. MOAI is a totally unknown quantity to most dev-shops, who aren't interested in anything 'but the standards stuff, what everyone else is using', etc.

Still, there are a surprising number of ways MOAI can be used, its not just a game engine .. I hacked up a quick embedded interface for a friends' robot project recently, it worked vey smoothly .. and was nothing 'game' about it .. MOAI+Hanappe is a great prototyping/GUI environment, I find, and I'm sort of used to using MOAI as my tooling engine for whenever I need a button in some script somewhere ..
