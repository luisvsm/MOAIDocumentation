Re: Upscaling a framebuffer?

by Vavius » Tue Nov 18, 2014 4:21 pm

You can create MOAIFrameBufferTexture with the given width/height and then render it as quad with any size.

Code: Select all
  1. local frameBuffer = MOAIFrameBufferTexture.new ()

  2. frameBuffer:setRenderTable ({ layer })

  3. frameBuffer:init ( 100, 100 )

  4. frameBuffer:setClearColor ( 1, 0, 0, 1 )

  5. MOAIRenderMgr.setBufferTable ({ frameBuffer })

  6.  

  7. // set this quad to some prop to render it

  8. gfxQuad = MOAIGfxQuad2D.new ()

  9. gfxQuad:setTexture ( frameBuffer )

  10. gfxQuad:setRect ( -100, -100, 100, 100 )

  11. gfxQuad:setUVRect ( 0, 0, 1, 1 )



Also you may want to upscale the default framebuffer, this depends on host.

What are you trying to achieve? Stretching to fit the screen size is achieved with MOAIViewport settings. But framebuffer will be created with the size of the whole screen.
If you really need to make a small framebuffer and then upscale it on display - I know a way to make this on iOS. This can be usefull for pixel-art games: you can draw just 1/16 of pixels gaining lots of performance due to less fragment shader computations.

Re: Upscaling a framebuffer?

by nosheet » Wed Nov 19, 2014 12:24 pm

I think the idea for setting up the frame buffer texture is the following:
- Create a viewport with the small size you need (ie. 256x256px). This viewport will be rendered into MOAIFrameBufferTexture.
- Create a layer (or multiple layers) for that viewport where you will put the objects you want to render into frame buffer texture.
- Create a MOAIFrameBufferTexture with the size of the viewport.
- MOAIRenderMgr.setBufferTable({ myFrameBufferTexture })
- myFrameBufferTexture:setRenderTable( {myLayer1, myLayer2})

So what you will have now is the content of everything rendered for that particular viewport will be rendered into a small MOAIFrameBuffer texture.
Then you can use *ANOTHER VIEWPORT, ANOTHER LAYER* to put a MOAIQuad there, stretch it to any size you want, and apply myFrameBufferTexture to it.

Very vague I know, but it's more/less the workflow.

Some code from my engine that may help:

Code: Select all
  1. Game.buffer_render = true/false

  2.  

  3. -- SINGULAR VIEWPORT

  4. Game.viewport = MOAIViewport.new()

  5. if not Game.buffer_render then

  6.         Game.viewport:setSize(Game.screenWidth, Game.screenHeight)

  7.         Game.viewport:setScale(Game.worldWidth, Game.worldHeight)

  8. else

  9.         Game.viewport:setSize(Game.worldWidth, Game.worldHeight)

  10.         Game.viewport:setScale(Game.worldWidth, Game.worldHeight)

  11. end

  12.  

  13. -- RENDERING LAYERS

  14. function Game:GetRenderTables(layers)

  15.     Engine:GetRenderTables(layers)

  16.     Menus:GetRenderTables(layers)

  17.     if Game.console and Console.visible then

  18.         Console:GetRenderTables(layers)

  19.     end

  20. end

  21.  

  22. -- RENDER TABLES

  23. function Game:SetRenderTables()

  24.     local layers = {}

  25.     Game:GetRenderTables(layers)

  26.         if not Game.buffer_render then

  27.                 MOAIRenderMgr.setRenderTable(layers)

  28.         else

  29.                 Game.buffer_frame:setRenderTable (layers)

  30.         end

  31. end

  32.  

  33. -- RENDER BUFFER

  34. function Game:BuildRenderBuffer(sizex, sizey, scalex, scaley)

  35.        

  36.         sizex = 640 or sizex

  37.         sizey = 320 or sizey

  38.        

  39.         local buffer_frame = MOAIFrameBufferTexture.new ()

  40.         buffer_frame:init(sizex, sizey)

  41.         buffer_frame:setClearColor()

  42.         MOAIRenderMgr.setBufferTable({buffer_frame})

  43.        

  44.         local buffer_viewport = MOAIViewport.new ()

  45.         buffer_viewport:setSize ( sizex*scalex, sizey*scaley )

  46.         buffer_viewport:setScale ( sizex, sizey )

  47.         buffer_viewport:setOffset ( -1, 1 )

  48.        

  49.         local buffer_layer = MOAILayer2D.new ()

  50.         buffer_layer:setViewport ( buffer_viewport )

  51.        

  52.         local final_table = {}

  53.         table.insert(final_table, buffer_layer)

  54.        

  55.         local buffer_gfxQuad = MOAIGfxQuad2D.new ()

  56.         buffer_gfxQuad:setTexture ( buffer_frame )

  57.         buffer_gfxQuad:setRect ( 0, 0, sizex,sizey )

  58.         buffer_gfxQuad:setUVRect ( 0, 0, 1, 1 )

  59.  

  60.         local buffer_prop = MOAIProp2D.new ()

  61.         buffer_prop:setDeck ( buffer_gfxQuad )

  62.         buffer_prop:setScl(1)

  63.         buffer_prop:setLoc(0, -sizey)

  64.         buffer_layer:insertProp ( buffer_prop )

  65.        

  66.         Game.buffer_frame = buffer_frame

  67.         Game.buffer_viewport = buffer_viewport

  68.         MOAIRenderMgr.setRenderTable(final_table)

  69. end

  70. if Game.buffer_render then

  71.         Game:BuildRenderBuffer(Game.worldWidth, Game.worldHeight, 2, 2)

  72. end

Re: Upscaling a framebuffer?

by Vavius » Thu Nov 20, 2014 12:53 pm

Spy wrote:How?

Requires modification of iOS host. In OpenGLView.mm find -(void)createContext and -(bool)createBuffers:
1) There you can use custom mWidth and mHeight for framebuffer
2) Scaling can be done through glLayer.constentsScale
3) Also useful in glLayer: minificationFilter, magnificationFilter - can be 'linear' or 'nearest'; contentsGravity and contentsRect to adjust onscreen location.

Doing like so can greatly increase performance of pixel-art games, cause it requires 1/4 or 1/16 of your normal fillrate and performs upscaling on gpu after fragment shaders are applied. This is iOS only, there is no way to use "nearest" filter on Android.
