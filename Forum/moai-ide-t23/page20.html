Re: Moai IDE?

by Hach-Que » Sat Feb 25, 2012 5:02 pm

thebullno1 wrote:How to use the debug feature? I built from the latest revision and when I clicked run with debug it says something along the line of "cannot find MOAI executable". The preference and tool options are grayed out so I could not config the path to moai host.

You need to copy the Moai engine to the folder manually at the moment, since there's no settings back-end.

Once extracted, you must copy the Moai engine executable and libraries to the Deployment\Engines\Win32\Debug such that the path Deployment\Engines\Win32\Debug\moai.exe will exist.

In addition, you need to ensure the Moai engine version you're using has the GLUT_AKU_DEBUGGER definition enabled while compiling so that the IDE will be able to connect to the engine. Unfortunately, the current SDK binaries that ship do not have this enabled, so you will need to compile the engine yourself.

I'm hoping that once the build server is set up for the Moai engine itself, it'll automatically build a version with the debugger enabled, which the IDE will be able to automatically download and set it up for you.

Vad1mo wrote:Hey guys I don't want to be a killjoy but did you take a look at http://www.eclipse.org/koneki/ldt/this is some big step in lua IDE landscape. James put a lot of effort in developing the moai IDE but why not concentrating on the plugin development for the koneki or intellij then developing a completely new ide?

There's a number of features that this IDE can support which aren't implemented yet (but the structure permits them to be developed). For example, this IDE can support remote Lua debugging on devices themselves and on remote computers.

Given that the Lua debugging mechanism that is in place inside the Moai engine is not the same kind of debugging connection that those IDEs will use to inspect Lua, nor is that debugging connection likely to work over network or to mobile devices, it would mean that not only would I need to learn how to implement plugins for those IDEs, but I'd also have to rewrite the Lua debugging systems anyway.

That said, you're free to do so. The debugger API in Moai is open-source, just like the rest of it and can be used by any application (not just this IDE). If you want to, you can implement plugins for Eclipse that connects to the Moai debugger (it'll just end up being a lot of work).
