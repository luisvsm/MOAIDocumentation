@monkan When I first set out to make a platformer the 'unicycle' method came up a lot on google seraches so I first tried doing something like that. I am used to writing my own simple physics systems for platformers and I've gotten quite used to modifying x,y velocity components exclusively for all movements so using forces felt awkward. I also didn't like some of the effects caused by the wheel like climbing up walls and such. Since I update the velocity every simulation loop iteration the movement appears just as smooth as any other (from what I can tell anyways). I also like having full control over the character's movements instead of hoping Box2D simulates physics interactions in a desirable way.

That being said, I am no pro Box2D developer so maybe after using it for a while I'll find that using forces is more favorable. I guess we'll see.

I tried to make a platformer with box2d before and the solution I used was to have the player character have a wheel attached with a revolute joint (I read somewhere on the internet about someone doing the same thing). To move left and right I would set motor speed in the direction I wanted. This actually produced some really nice results and avoided any dodgy physics errors you can get by setting the velocity yourself. It also meant that the character moved very smoothly across the ground which gave good visual results. Some hurdles that I faced though included not having control when jumping and also sometimes rolling up walls. These were both solved with some work-arounds but it wasn't ideal.

I'm trying to make a platformer type thing at the moment using box2d and so far I have just opted to add a force in the direction I want to move and then add a braking force in the opposite direction to bring the character to a stop. It works ok for now but I think I might give your method a go so I can get more control over the movement,it looks like you got some nice simple ways to overcome some of the problems.

]]> 2012-11-24T21:36:16+00:00 2012-11-24T20:34:22+00:00 /forums/viewtopic.php?t=1415&p=7541#p7541 <![CDATA[Free Box2D Platformer Game Code =)]]>
It seems like platformers are often what people attempt to make for their first Box2D (or chipmunk or whatever) game. It seems logical to emulate the control scheme and game play 'feel' of the classics like Mario and such. However, what many don't realize is that traditional platformer games don't use real physics for much of the character control. To name a few 'fake' physics properties of these games: air control, where the player can move left/right mid-air; being able to instantly turn on a dime without slowing down to turn; being able to double jump. So how is it possible to implement physically impossible things in a realistic physics environment? Simple, you just have to get creative and mess with it enough until you get what you want. There is no exact formula to doing this (as far as I know anyways...) you just have to keep trying different solutions until one works.

That being said, here's my implementation of a simple platformer game using Box2D. The controllers are:
'a' moves left
'b' moves right
'w' jumps
You can press 'w' again in the air to double jump.
Code:
  1.  

  2. -- scale set so screen is 20 meters tall

  3. scale = 10

  4.  

  5. -- screen dimensions

  6. Screen = {

  7.         w = 1216,

  8.         h = 760

  9. }

  10.  

  11. -- stage dimensions

  12. Stage = {

  13.         w = 320,

  14.         h = 200

  15. }

  16.  

  17. -- open sim window

  18. MOAISim.openWindow( 'platformer_test', Screen.w, Screen.h )

  19.  

  20. -- setup viewport

  21. viewport = MOAIViewport.new()

  22. viewport:setSize( Screen.w, Screen.h )

  23. viewport:setScale( Stage.w, Stage.h )

  24.  

  25. -- setup Box2D world

  26. world = MOAIBox2DWorld.new()

  27. world:setGravity( 0, -10 )

  28. world:setUnitsToMeters( 1 / scale )

  29. world:setDebugDrawFlags( MOAIBox2DWorld.DEBUG_DRAW_SHAPES + MOAIBox2DWorld.DEBUG_DRAW_JOINTS +

  30.                                                  MOAIBox2DWorld.DEBUG_DRAW_PAIRS + MOAIBox2DWorld.DEBUG_DRAW_CENTERS )

  31.  

  32. -- main rendering layer

  33. layer = MOAILayer2D.new()

  34. layer:setViewport( viewport )

  35. layer:setBox2DWorld( world )

  36.  

  37. -- char code for fonts

  38. charCode = 'abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789 _+-()[]{}|\/?.,<>!~`@#$%^&*\'":;'

  39.  

  40. -- to scale fonts

  41. fontScale = Screen.h / Stage.h

  42.  

  43. -- status textbox

  44. status = MOAITextBox.new()

  45. status:setRect( -160 * fontScale, -100 * fontScale, 160 * fontScale, 100 * fontScale )

  46. status:setScl( 1 / fontScale )

  47. status:setYFlip( true )

  48. status:setColor( 1, 1, 1 )

  49. status:setString( 'status' )

  50. status.font = MOAIFont.new()

  51. status.font:load( 'verdana.ttf' )

  52. status.font:preloadGlyphs( charCode, math.ceil( 4 * fontScale ), 72 )

  53. status:setFont( status.font )

  54. layer2 = MOAILayer2D.new()

  55. layer2:setViewport( viewport )

  56. layer2:insertProp( status )

  57.  

  58. -- setup ground

  59. ground = {}

  60. ground.verts = {

  61.         -160, 10,

  62.         -120, 10,

  63.         -120, -10,

  64.         -15, -10,

  65.         -15, 5,

  66.         5, 5,

  67.         20, 20,

  68.         40, 20,

  69.         40, -18,

  70.         140, -18,

  71.         140, 20,

  72.         160, 20

  73. }

  74. ground.body = world:addBody( MOAIBox2DBody.STATIC, 0, -60 )

  75. ground.body.tag = 'ground'

  76. ground.fixtures = {

  77.         ground.body:addChain( ground.verts )

  78. }

  79. ground.fixtures[1]:setFriction( 0.3 )

  80.  

  81. -- setup player

  82. player = {}

  83. player.onGround = false

  84. player.currentContactCount = 0

  85. player.move = {

  86.         left = false,

  87.         right = false

  88. }

  89. player.platform = nil

  90. player.doubleJumped = false

  91. player.verts = {

  92.         -5, 8,

  93.         -5, -9,

  94.         -4, -10,

  95.         4, -10,

  96.         5, -9,

  97.         5, 8

  98. }

  99. player.body = world:addBody( MOAIBox2DBody.DYNAMIC )

  100. player.body.tag = 'player'

  101. player.body:setFixedRotation( true )

  102. player.body:setMassData( 80 )

  103. player.body:resetMassData()

  104. player.fixtures = {

  105.         player.body:addPolygon( player.verts ),

  106.         player.body:addRect( -4.9, -10.1, 4.9, -9.9 )

  107. }

  108. player.fixtures[1]:setRestitution( 0 )

  109. player.fixtures[1]:setFriction( 0 )

  110. player.fixtures[2]:setSensor( true )

  111.  

  112. -- setup platforms

  113. platforms = {}

  114. platforms[1] = {}

  115. platforms[1].body = world:addBody( MOAIBox2DBody.KINEMATIC, 70, -44 )

  116. platforms[1].body.tag = 'platform'

  117. platforms[1].body:setLinearVelocity( 20, 0 )

  118. platforms[1].limits = {

  119.         xMax = 130, xMin = 70,

  120.         yMax = -43, yMin = -45

  121. }

  122. platforms[1].fixtures = {

  123.         platforms[1].body:addRect( -10, -4, 10, 4 )

  124. }

  125.  

  126. platforms[2] = {}

  127. platforms[2].body = world:addBody( MOAIBox2DBody.KINEMATIC, 50, -44 )

  128. platforms[2].body.tag = 'platform'

  129. platforms[2].body:setLinearVelocity( 0, 10 )

  130. platforms[2].limits = {

  131.         xMax = 51, xMin = 49,

  132.         yMax = -44, yMin = -74

  133. }

  134. platforms[2].fixtures = {

  135.         platforms[2].body:addRect( -10, -4, 10, 4 )

  136. }

  137.  

  138. -- platform movement thread

  139. platformThread = MOAIThread.new()

  140. platformThread:run( function()

  141.         while true do

  142.                 for k, v in ipairs( platforms ) do

  143.                         local x, y = v.body:getWorldCenter()

  144.                         local dx, dy = v.body:getLinearVelocity()

  145.                         if x > v.limits.xMax or x < v.limits.xMin then

  146.                                 dx = -dx

  147.                         end

  148.                         if y > v.limits.yMax or y < v.limits.yMin then

  149.                                 dy = -dy

  150.                         end

  151.                         v.body:setLinearVelocity( dx, dy )

  152.                 end

  153.                 coroutine.yield()

  154.         end

  155. end )

  156.  

  157. -- player foot sensor

  158. function footSensorHandler( phase, fix_a, fix_b, arbiter )

  159.  

  160.         if phase == MOAIBox2DArbiter.BEGIN then

  161.                 player.currentContactCount = player.currentContactCount + 1

  162.                 if fix_b:getBody().tag == 'platform' then

  163.                         player.platform = fix_b:getBody()

  164.                 end

  165.         elseif phase == MOAIBox2DArbiter.END then

  166.                 player.currentContactCount = player.currentContactCount - 1

  167.                 if fix_b:getBody().tag == 'platform' then

  168.                         player.platform = nil

  169.                 end

  170.         end

  171.         if player.currentContactCount == 0 then

  172.                 player.onGround = false

  173.         else

  174.                 player.onGround = true

  175.                 player.doubleJumped = false

  176.         end

  177. end

  178. player.fixtures[2]:setCollisionHandler( footSensorHandler, MOAIBox2DArbiter.BEGIN + MOAIBox2DArbiter.END )

  179.  

  180. -- keyboard input handler

  181. function onKeyboard( key, down )

  182.         -- 'a' key

  183.         if key == 97 then

  184.                 player.move.left = down

  185.         -- 'd' key

  186.         elseif key == 100 then

  187.                 player.move.right = down

  188.         end

  189.        

  190.         -- jump

  191.         if key == 119 and down and ( player.onGround or not player.doubleJumped ) then

  192.                 player.body:setLinearVelocity( player.body:getLinearVelocity(), 0 )

  193.                 player.body:applyLinearImpulse( 0, 80 )

  194.                 if not player.onGround then

  195.                         player.doubleJumped = true

  196.                 end

  197.         end

  198. end

  199. MOAIInputMgr.device.keyboard:setCallback( onKeyboard )

  200.  

  201. -- player movement thread

  202. playerThread = MOAIThread.new()

  203. playerThread:run( function()

  204.         while true do

  205.                 local dx, dy = player.body:getLinearVelocity()

  206.                 if player.onGround then

  207.                         if player.move.right and not player.move.left then

  208.                                 dx = 50

  209.                         elseif player.move.left and not player.move.right then

  210.                                 dx = -50

  211.                         else

  212.                                 dx = 0

  213.                         end

  214.                 else

  215.                         if player.move.right and not player.move.left and dx <= 0 then

  216.                                 dx = 25

  217.                         elseif player.move.left and not player.move.right and dx >= 0 then

  218.                                 dx = -25

  219.                         end

  220.                 end

  221.                 if player.platform then

  222.                         dx = dx + player.platform:getLinearVelocity()

  223.                 end

  224.                 player.body:setLinearVelocity( dx, dy )

  225.                 coroutine.yield()

  226.         end

  227. end )

  228.  

  229. -- update function for status box

  230. statusThread = MOAIThread.new()

  231. statusThread:run( function()

  232.         while true do

  233.                 local x, y = player.body:getWorldCenter()

  234.                 local dx, dy = player.body:getLinearVelocity()

  235.                 status:setString( 'x, y:   ' .. math.ceil( x ) .. ', ' .. math.ceil( y )

  236.                                          .. '\ndx, dy: ' .. math.ceil( dx ) .. ', ' .. math.ceil( dy )

  237.                                          .. '\nOn Ground: ' .. ( player.onGround and 'true' or 'false' )

  238.                                          .. '\nContact Count: ' .. player.currentContactCount

  239.                                          .. '\nPlatform: ' .. ( player.platform and 'true' or 'false' ) )

  240.                 coroutine.yield()

  241.         end

  242. end )

  243.  

  244. -- render scene and begin simulation

  245. world:start()

  246. MOAIRenderMgr.setRenderTable( { layer, layer2 } )



You will need a copy of the 'verdana.ttf' font for the status box to work. You can find it in one of the sample folders that comes with the MOAI SDK (just search for it because I don't remember where it is exactly...)

Some notes:
- In order to have pretty much ultimate control of the player's movement I choose to directly manipulate his linear velocity components as opposed to applying linear impulses in the desired direction of movement. To do this I needed a way to store the desired movement of the character. So I attached some values tot he player object that tells me if he is on the ground (player.onGround), and if he should move left or right or both (player.move.left and player.move.right). The onGround value is updated by placing a sensor on the player's feet area and setting a collision handler; this collision handler sets onGround to false when his feet aren't touching something and to true when they are. The left and right values are set according to what keys are being pressed by using a keyboard input callback function; so when 'a' is being held down left is true, when 'd' is held down right is true. The next step was to make a function that runs every frame update and manipulates the player's velocity according to these values. To do this I made a MOAIThread that is always running using a while loop with a forever true condition; you'll notice a call to coroutine.yield() which allows the rest of the program to run after this function has updated the player's velocity - otherwise nothing would happen and the program would be stuck in an infinite loop. You should also note that the player has 0 friction which allows him to slide along without slowing down.

For beginners looking at this, read it carefully and take sometime to try and figure out what each piece is doing. Try playing with different numbers (should probably change only one value at a time) and running it to see what the effect is.

For more experienced people looking at this, feel free to post comments and suggestions on how to improve this code so people can learn from it better.

Enjoy =)
