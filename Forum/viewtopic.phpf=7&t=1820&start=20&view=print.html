Thanks all for sharing your shaders. I've just started trying to get a few shaders working in Moai. After realizing that Moai's matrices are the transpose of what you find in most other examples I've gotten a couple working. Thanks to the comments in @nosheets example, I realized I had to swap from matrix * point to point * matrix.

Seems like the one thing that is missing in the shaders is the modelView matrix. Moai gives you MOAIShader.UNIFORM_WORLD is the model matrix, UNIFORM_VIEW_PROJ which is view * projection and UNIFORM_WORLD_VIEW_PROJ which is model * view * projection. For lighting shaders, I need model * view to get from model space to eye space since you usually do your lighting calculations in eyeSpace.

by josh h
Looking at the source code, MOIA maintains 3 matrices: world, view and projection. From what I can tell world is the cumulative transform on the prop, view is the camera transform and projection is the usual projection matrix. So the equivalent of the usual modelview matrix in OpenGL programs would combine world and view.

Here is a snippet from MOAIShader::bindPipelineTransforms where you can see that UNIFORM_WORLD is just the world matrix without the view but UNIFORM_WORLD_VIEW_PROJ combines all three:

[code=C++]case UNIFORM_WORLD: {

this->SetValue ( world );
this->Bind ();
break;
}
case UNIFORM_WORLD_VIEW_PROJ: {

ZLMatrix4x4 mtx = world;
mtx.Append ( view );
mtx.Append ( proj );
this->SetValue ( mtx );
this->Bind ();
break;
}
[/code]

In a lot of scenes, there is no camera or the camera has only translation and if you are only doing directional lighting then it doesn't matter. I ran into it trying to do per-pixel lighting in eye coordinates with a point light source where the camera transform is important.

I added a new UNIFORM_WORLD_VIEW to my fork that gives the "real" modelview matrix which is at github.com/jhandley/moai-dev/

  4.

  7. varying vec2 uv;

  9. void main(void)

  10. {

  11.     gl_Position = vec4( gl_Vertex.xy, 0.0, 1.0 );

  12.     gl_Position = sign( gl_Position );

  14. }

  17.

  19.  

  20.  

  21.  

  22. local fragmentShader = [[

  23.  

  24.  

  25. uniform sampler2D tex;

  26. varying vec2 uv;

  27. const float sampleDist = 1.0;

  28. const float sampleStrength = 2.2;

  29.  

  30. void main(void)

  31. {

  32.                     float samples[10];

  33.                     samples[0] = -0.08;

  34.                     samples[1] = -0.05;

  35.                     samples[2] = -0.03;

  36.                     samples[3] = -0.02;

  37.                     samples[4] = -0.01;

  38.                     samples[5] =  0.01;

  39.                     samples[6] =  0.02;

  40.                     samples[7] =  0.03;

  41.                     samples[8] =  0.05;

  42.                     samples[9] =  0.08;

  43.                

  44.                     vec2 dir    = 0.5 - uv;

  45.                     float dist  = sqrt(dir.x*dir.x + dir.y*dir.y);

  46.                     dir                 = dir/dist;

  47.                

  48.                     vec4 color = texture2D(tex,uv);

  49.                     vec4 sum = color;

  50.                

  51.                     for (int i = 0; i < 10; i++)

  52.                         sum += texture2D( tex, uv + dir * samples[i] * sampleDist );

  53.                

  54.                     sum *= 1.0/11.0;

  55.                     float t = dist * sampleStrength;

  56.                     t = clamp( t ,0.0,1.0);

  57.                

  58.                     gl_FragColor = mix( color, sum, t );

  59. }

  60.  

  61.  

  62. ]]
