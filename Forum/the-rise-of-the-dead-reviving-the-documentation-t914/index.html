Re: The Rise of the Dead - Reviving the Documentation

by bWard » Sat Jun 23, 2012 4:23 am

Right off of the top of my head, I can point out only a few things:

The documentation should reflect the current status, what happens when a typical user sets up Moai: example, many of us can't make an Android build following the instructions on the wiki. It should be noted as incomplete or updated with a working solution. As it is, most pages show a simplified idealistic notion of what should happen, not what does happen.

In short, test out Moai/building/exporting and update its wiki page.

Honestly, it's not that things are noticeably wrong, just un-updated. The best place to start would be looking at the update notes from Moai 1.0 up until now, and using the search function to find key words that will lead you to the pages that need updated. Of course, you can find them manually too.

From there, it would be best to read the wiki pages while following the moai files to see if it adds up. The only things I spotted were updates to the folder structure or being moved to a new folder.

I've found smaller things that don't matter so much: the most recent Moai, Build 54, I've noticed that it runs Moai.exe from a local path rather than a global config path. Hello-Moai/Run.bat and run.sh goes one folder up too far, being the only(?) sample directory not two folders down and doesn't work without a little editing. And the newest version.txt has broken linebreaks.

Everything else I can contribute are my experiences running Moai: problems and errors I've encountered, how to get around some. But the documentation thread isn't the place for submitting errors.
