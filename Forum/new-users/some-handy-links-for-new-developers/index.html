Ah. Lua provides the ':' operator when calling functions as syntactic sugar for passing the table being indexed as the first parameter of the function.

So

fixture2 = body2:addRect ( -5, -5, 5, -3 )

is exactly equivalent to

fixture2 = body2.addRect ( body2, -5, -5, 5, -3 )

Note the use of the '.' versus the ':' in the second example.

The documentation is just telling you that the MOAIBox2DBody function addRect expects a MOAIBox2DBody as its first parameter, which you can pass either explicitly (with the dot) or implicitly (with the colon).
