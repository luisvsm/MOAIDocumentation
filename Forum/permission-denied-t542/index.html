by NoobsArePeople2 » Fri Mar 02, 2012 6:21 pm

I'm not sure that's true. The link you provided only mentions that certain OpenGL methods were implemented, not that they are supported in the emulator.

The OpenGL ES2.0 page in the Android docs explicitly states that OpenGL is not supported in the emulator (it mentions 2.2, not 2.3 though. Maybe this changed?)

Of course, the way to put this to the test is to update the Android host to use Gingerbread rather than Froyo. I would note that the Android emulator is notoriously slooooooow so it's probably not up to testing out a game.

Changing line 53 in "AndroidManifest.xml" in your Android host should do the trick. SDK version 8 is Froyo, changing it to 9 should update the host to use Gingerbread.

Then delete your host and try building it again. Hope it helps.

Re: Permission Denied

by sean » Tue Apr 03, 2012 8:47 pm

I need to tweek the scripts to run better out-of-the-box on Windows. Several different people have seen several different issues. But, that's not going to happen until an SDK update later this month. For now, IFF you have rsync in your cygwin install, I have a workaround. If you open the script in a text editor, you'll see several rsync commands that are commented out using the # character). If you go to each one, remove the # to uncomment the command and then add a # in front of the subsequent find command to comment it out, you might get past the errors you're seeing. Basically, we used to use rsync to do some magic file copying, but then switched to find to remove the dependency on rsync. However, I left the old rsync commands in the script in case people had issues.
