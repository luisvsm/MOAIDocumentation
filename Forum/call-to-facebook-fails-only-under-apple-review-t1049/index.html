by stephensullivan » Sat Jul 28, 2012 12:34 am

We are seeing an issue where calls to facebook fail only on apple's test devices. We have it running on over 20 test devices internally. From the log they gave us it says the following.

Jul 27 16:33:38 unknown wifid[13] <Error>: WiFi:[365124818.457236]: Client itunesstored is background application
Jul 27 16:34:30 unknown wifid[13] <Error>: WiFi:[365124870.803931]: Client itunesstored is background application
Jul 27 16:34:40 unknown UIKitApplication:com.playnomi.picturethisfree[0x30a2][2210] <Notice>: * Could not resolve host: graph.facebook.com (Timeout while contacting DNS servers)
Jul 27 16:34:40 unknown UIKitApplication:com.playnomi.picturethisfree[0x30a2][2210] <Notice>: * Closing connection #0

The behavior is the call hangs until it times out.

Here is the outbound call:

url = "https://graph.facebook.com/me?fields=picture,name,id,email,gender&sdk=ios&sdk_version=2&access_token=" .. MOAIFacebook.getToken() .. "&format=json";

-- go and grab the current games and put them in a list
local task = MOAIHttpTask.new ()

task:setVerb ( MOAIHttpTask.HTTP_GET )
task:setUrl ( url )
task:setCallback ( onGetProfileFinish )
task:setUserAgent ( "Moai" )
task:setTimeout ( 30 )
task:setVerbose ( true )
task:performAsync ()
