Are you sure you've got your Android host set up with the proper screen dimensions and orientation for your game? You have to double-check this yourself in the Android native host code, and be sure that the assumption you're making in your MOAI app about screen orientation is actually being presented to the MOAI host by way of the Native Android code. If you haven't looked at this: please do so, because its the #1 cause of problems with the Android host. Assumptions about the native host code making everything ready are not going to be productive until you've fully investigated and verified the code ..

Re: MOAI Android - Touch events not working

by ibisum » Sun Apr 28, 2013 4:26 am

The Android host sources are in the $MOAI_HOME/ant folder, and you must build them yourself for an Android host .. you have to use the 'make-host.sh' script to generate a copy of the template Android folder, than modify the settings-local.sh and settings-global.sh files according to your needs (set the path to your MOAI sources, Android SDK, and so on) .. once you have done this, and can successfully build your app for Android using the run-host.sh script, then you will be able to modify the Android host sources to use the screen orientations that your app requires. By default the Android host has some weird half-assed configuration for landscape mode, and if your MOAI app thinks its in portrait, you will get busted touch events, for sure.

So I'd say you should be building your own Android MOAI host first, to solve this problem, then investigate what the screen orientation code looks like and match it up to your needs.
