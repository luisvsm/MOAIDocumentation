Kinematic collisions

by KidDrew » Wed Jul 24, 2013 1:28 pm

I'm building a NES-style game, and in the NES style my bullets should travel across the screen at a constant pace and be immune to gravity. I'm trying to solve this by making my bullets kinematic, but I can't seem to detect collisions on the fixtures. My bullets are passing right through static bodies and not invoking my collision handler. If I make my bullets dynamic, then the collisions happen as expected and the handler is invoked.

Here's a code snippet:

Code: Select all
  1.  

  2.   this.body = world:addBody(MOAIBox2DBody.KINEMATIC, x, y)

  3.  

  4.   this.body:setFixedRotation(true)

  5.   this.body:setBullet(true)

  6.   this.fixture = this.body:addRect(-2, -2.5, 2, 2.5)

  7.   this.fixture.id = 'bullet'

  8.   this.fixture:setCollisionHandler(collision.handler, MOAIBox2DArbiter.BEGIN)

  9.  

  10.   this.body:setLinearVelocity(200,0)

  11.  



So two questions:
1. Is it true that fixtures on kinematic bodies don't detect collisions? Why?

2. Is there a way to make a dynamic body immune to gravity?

Re: Kinematic collisions

by ezraanderson » Wed Jul 24, 2013 7:17 pm

I struggled with box2d and bullets for along time then came to the conclusion that I would never get the results I wanted using box2d for my bullets. The Main annoyances for me was they would not move fast enough and always detect collisions.

Things I tried with bullets:
  • Use bullets as sensors then disable if they hit a wall.
  • Recycle your bullets the more kinetic/dynamic bodies in the world impact box2d performance
  • Make fixtures bullets.
  • tweak MOAIBox2DWorld.setUnitsToMeters()
  • Don't use box2d to calculate collision results, calculate my own impact ie:zombies stumble when shot
Result: Scrap box2d for bullets and write my own system which was a lot of effort but performs perfectly.

  • disabling gravity completely then setlinearvelocity on objects that are falling.

Re: Kinematic collisions

by KidDrew » Fri Jul 26, 2013 4:35 pm

I found another solution, just in case anyone is interested. I set my body to dynamic, so collisions are detected but gravity applies. To counter the gravity, I just set the Y velocity to 5 every update:

Code: Select all
  1.  

  2. local dx, dy = self.body:getLinearVelocity()

  3. self.body:setLinearVelocity(dx, 5)

  4.
