Re: MOAI, Hanappe and Rapanui

by ibisum » Wed Nov 14, 2012 12:12 pm

moaigui, Hanappe, and Rapanui - these are all quite decent little frameworks to solve various aspects of MOAI that weren't "filled in". All of these frameworks started from an itch to scratch, and its a testament to the wonders of MOAI that they were released/developed as rapidly as they were. moaigui might have its rough edges, but a cursory glance through its code reveals a well-structured attempt to bring a GUI to the party - and it does so acceptably well, given certain limitations of MOAI itself (text-rendering, specifically..)

Hanappe and Rapanui are really decent frameworks for a generalized container system, implementing features that are common to the games industry as a whole - standard patterns for using content systems, menu'ing akin to what users are used to in the simpler console platforms, etc. I'd say, if you want to learn MOAI, using any - or all - of these frameworks as a basis to gain an understanding of the system is a valid effort and worthwhile, to boot. There are some nice things to learn about in all of these frameworks, and as long as you have the SDK docs at hand, a good way to begin to understand the deep power of MOAI itself ..

One other thing - search for MOAI on github. There are some other projects that have published samples/sources hidden out there, and in particular - if I can find it - there was a samples project that really did wonderful things with just the basic MOAI functionality .. I'll see if I find the URL.
