]]> 2012-05-12T06:43:25+00:00 2012-05-12T06:43:25+00:00 /forums/viewtopic.php?t=772&p=3890#p3890 <![CDATA[Re: Distribution Questions]]> However, there are several ways to hide your game code.

You can precompile them with luac. To be able to load them normally, you need to append ?.luac to the end of package.path.
You need to be careful as lua bytecode is NOT portable. Most mobiles are 32bit little endian. For windows, stick to the 32bit distribution of lua and moai and it should work even on 64 bit machines. Linux can be a problem depending on distro(some ship with both 32 and 64 bit libraries, others dont). Im not sure about Mac. Do PowerPc(big endian) Mac still exists?

You can also zip all resources and append the zip file to your exe. The exe header contains a size field so extra content wont spoil it. As for zip, the file index is at the end so extra stuffs at the beginning is fine too. You can mount your exe file at the start and your game should work as normal. Most zip archivers have no problem recognizing it too so dont rely too much on this method.

Lua allows you to create a custom module loader. You can write your own in C/C++(so it's harder to decompile) and add it to the package.loaders array. That way, you can use some encryption for lua code.
All these are just obfuscation anyway. Anything can be cracked.

Somewhere in this forum, there's a tutorial on setting up a debugger which integrates with Eclipse or IntelliJ. It's called Koneki. Whatever socket related issues mentioned in those threads should be gone by now.
I personally use a commercial debugger called Decoda which works out of the box with any engine that uses Lua. It only needs a pdb file.
Recent moai-dev revision allows you to use your own error handler. I hook it up with this script: https://github.com/ignacio/StackTracePlus
It prints out value of local variables in addition to just the function names.
There is also a moai-ide which has debugging support. Although quite functional, it's still beta quality.
My custom host and recent versions of Moai's host can reload the project whenever a change to its file is detected. When I'm editing, I usually modify my scripts so that the game always go to the state I'm interested in (e.g: skip main menu and jump straight to a particular level). I can see how changes to the code affect the game. Coupled with a detailed stack trace(StacktracePlus), a debugger is not needed most of the time.
