My (admittedly limited) experience on 2D game animation involved putting a call to character or object animation in the game loop, where it's called every frame or loop step, and moving the object a tiny amount. "every loop, if the player is pressing X, move the cathead 3 pixels to the right"

by patrick » Mon Jun 20, 2011 5:28 pm

You can take either approach depending on what you're comfortable with. In fact, the first draft Rocket Lobster tutorial used stepwise animation instead of the animation helpers.

Try creating a MOAIThread and in the main function use prop:getLoc () and prop:setLoc () to move your objects. The default simulation step is 60fps, but there are methods in MOAISim to adjust that.
