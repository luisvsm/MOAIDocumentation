kubuk wrote:
Ok. I put files in one of the "standard" folders and it works now, but it doesn't show images, when I am trying to run it from ZeroBrane Studio. It works well, when I am running it from .bat file.


kubuk, you may want to check on two things. First of all, if you specify the custom location for your moai executable, make sure you specify the path to moai executable: I have an example in cfg/user-sample.lua:

-- to specify full path to moai *executable* if it's not in one of PATH folders
path.moai = 'd:/lua/moai/moai'

you can also specify 'd:/lua/moai/moai.exe' (but it should work without the extension too). This is probably why you are getting "access denied".

As to loading images and fonts, make sure that the project folder location (in the project pane on the left) points to your project folder. This folder will be set as the current one when you start your project from ZBS, so your script will find all the additional files it needs to load.

I've posted some details here (http://notebook.kulchenko.com/zerobrane ... ane-studio), but I'll make it clear that the path should be to the executable. Let me know if you run into any other issues.

Paul.
