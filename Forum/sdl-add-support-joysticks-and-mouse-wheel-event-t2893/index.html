Just a minor fix to your Joystick.ccp:

Code: Select all
  1. const Joystick::AXIS_MOTION Joystick::HandleAxisMotion(const SDL_Event & event) {

  2.    

  3.     if( event.jaxis.axis == 0 ) { //X axis

  4.         if( event.jaxis.value < -JOYSTICK_DEAD_ZONE ) {

  5.             jx = -1;

  6.         } else if( event.jaxis.value > JOYSTICK_DEAD_ZONE ) {

  7.             jx =  1;

  8.         } else {

  9.                         jx = 0; //set x position to zero only if during this SDL_event it goes under the dead zone

  10.                 }

  11.     } else if ( event.jaxis.axis == 1 ) { // Y axis

  12.         if( event.jaxis.value < -JOYSTICK_DEAD_ZONE ) {

  13.             jy = -1;

  14.         } else if( event.jaxis.value > JOYSTICK_DEAD_ZONE ) {

  15.             jy =  1;

  16.         } else {

  17.                         jy = 0; //set y position to zero only if during this SDL_event it goes under the dead zone

  18.                 }

  19.     }

  20.  

  21.     return std::make_pair(jx, jy);

  22. }



and then in Joystick.h add:
Code: Select all
  1. private:

  2.         ...

  3.         int jx=0;

  4.         int jy=0;



Essentially if you would move the joystick to the left (x=-1), any slightest nudge of the stick vertically (even if it goes under the JOYSTICK_DEAD_ZONE value) would've reset the horizontal position back to neutral (x=0), and the same goes the other way around. After that fixing that though, it works a-okay.

Also seems to work on Windows just fine, after removing C++11 dependencies. Haven't tested the mouse wheel yet.

Re: SDL : Add support joysticks and mouse wheel event

by ibisum » Wed Jun 17, 2015 9:37 am

I also have to fix Joystick support for my projects soon, and so my contribution:

Have a look at how antirez implemented joystick support some time ago in the LOAD81 project:

https://github.com/seclorum/load81/blob/master/load81.c

(Search for 'joystickXMovedEvent')

Basically, this implementation creates a local Lua table called "joysticks", with full details: # of joysticks available, and button state, analog and digital position of each attached device.

Then, on the MOAI/Lua side, all one needs to do is read the values from the table when necessary. On the 'when necessary' subject, I think it only needs to happen when a joystick move event is enqueued, i.e. get the joystick event at SDL layer, update the joystick table values, call the user-provided callback.

This method of pushing values into the table from the C joystick event side means you can support all kinds of stick configurations - for example, joysticks that only send 1,0,-1 across the axes, or joysticks that are really just buttons (i.e. left is ON, up is ON), analog joysticks that provide 0.0:1.0 scales, and so on. Some joysticks are gimballed, some reset-to-center, so enumerating the joysticks this way - rather than just passing the problem across to the Lua side - makes a little more sense, imho.
