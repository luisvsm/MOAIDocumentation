Re: ScriptDeck -- can it save its state?

by keelo » Wed Jan 30, 2013 12:52 pm

Here's one way to accomplish this: you could make a new class/thing with that

a) renders to a MOAIFrameBuffer (see the frame buffer sample), and then save the MOAITexture from that
b) reuses that texture until you call Invalidate() or something

The tricky bit is that you need to set up a viewport to render to a framebuffer; check out the framebuffer sample.

Context may help. What I'm doing:

https://itunes.apple.com/us/app/miracle ... 04748?mt=8

I'm doing some comparison tests to see whether I could do this better in Moai than I can in Corona. The biggest thing I want is native widgets, and I find no indication that they exist here either, but I would accept performance improvements as a valid reason to switch anyway, probably.

But the specific thing I'm doing is a display mode where there are three points bouncing around the screen, and every frame, I render a spiral out from the center to each of these points. In Corona, each set of three spirals becomes a "group", and I then adjust the alpha of previous groups to create a fading effect; each "group" contains a few hundred to maybe a thousand or so objects. In Moai, each set of lines is a single mesh object, and I could in theory adjust the alpha, except I don't know enough of the OpenGL shader vocabulary to have any clue how to spell "render this half-transparent" as a shader.

Anyway, I want to create meshes that I can then render stuff to. So:

Code: Select all
  1.   if not g then

  2.     g = {}

  3.     g.vbo = MOAIVertexBuffer.new()

  4.     g.vbo:setFormat(vbo_format)

  5.     g.vbo:reserveVerts((line_segments + 3) * set.points)

  6.     g.vbo:bless()

  7.     g.mesh = MOAIMesh.new()

  8.     g.mesh:setTexture(white_pixel)

  9.     g.mesh:setVertexBuffer(g.vbo)

  10.     g.mesh:setPrimType(MOAIMesh.GL_LINE_STRIP)

  11.     g.prop = MOAIProp.new()

  12.     g.prop:setDeck(g.mesh)

  13.     g.prop:setShader(MOAIShaderMgr.getShader(MOAIShaderMgr.MESH_SHADER))

  14.     g.prop:setCullMode(MOAIProp.CULL_BACK)

  15.  

  16.     layer:insertProp(g.prop)

  17.     g.prop:setBlendMode(MOAIProp2D.BLEND_ADD)

  18.   end

  19.   g.sublines = g.sublines or {}



I am not at all sure whether some of this is right, and I don't really know what some of this means. For instance, I don't know what MESH_SHADER does, or whether it has to be there; if it does, I assume that means I can't actually do the alpha fading I want to the object, although I could just specify alpha for each point -- but that would require rewriting all the points every time I want to change the alpha, and that would suck.

So I have code (largely unchanged) which generates three "sublines", each of which has an array of points, each of which is { x = N, y = M, color = c }, and a colorfor() which returns r, g, b triplets from an integer color. So then, when I want to update each mesh:

Code: Select all
  1.  

  2. local function draw_line(line)

  3.   line.vbo:reset()

  4.   for j = 1, #line.sublines do

  5.     subline = line.sublines&#91;j&#93;

  6.     line.vbo:writeFloat(subline.points&#91;1&#93;.x, subline.points&#91;1&#93;.y)

  7.     line.vbo:writeColor32(0, 0, 0, 0)

  8.     for i = 1, line_segments do

  9.       line.vbo:writeFloat(subline.points&#91;i&#93;.x, subline.points&#91;i&#93;.y)

  10.       line.vbo:writeColor32(colorfor(subline.points&#91;i&#93;.color))

  11.     end

  12.     line.vbo:writeFloat(subline.points&#91;line_segments&#93;.x, subline.points&#91;line_segments&#93;.y)

  13.     line.vbo:writeColor32(0, 0, 0, 0)

  14.   end

  15.   line.vbo:bless()

  16. end



This seems to do what I want. (And so far, it appears, the performance is a factor of 5 better, at least.)

I could probably get noticably better performance using the data buffers or something, but I wanted to do the bare minimum amount of code changing to get results.

I don't actually quite understand what :bless() does, or how to know when to use it, but this seems to work.
