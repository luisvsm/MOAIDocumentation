Re: Hack N Slash Lua code

by Vavius » Wed Oct 22, 2014 8:35 pm

Some interesting moments I've found in the source:
1) The component-entity structure, pretty common these days. Entities update their components directly (no Systems). Since this is Lua code, there will be no performance benefits from proper Entity-Component-System
2) Box2d physics
3) Cool post-processing shader things. Lots of blur, but too heavy for mobile
4) Full-featured code live reload. Looks like you can edit/add/delete any class and module without pausing the game
5) They also use layout.py, that outputs declarative lua code. I'm pretty sure that my layout.py is different, despite the same name
