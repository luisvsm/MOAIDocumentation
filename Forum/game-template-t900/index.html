FEATURES:

Drawing Hills with box2d shapes

Player objects
Move main player using HUD buttons
Click on players produce particles / apply Box2d physics
Click and drag player
Launch bullet from player using sling

Thanks for the feedback.

I have cleaned up the indentation.

It may make sense to break the code into separate files but I was using a bootstrap method to test across android and ipad without having to recompile the whole project and deploy all the time. It was easier to pull one file down than numerous directories.

It would be nice to refactor and encapsulate a lot of the code but I wasn't trying build a lua upon a lua framework - I just wanted all the components and pieces to put together a complete game.

It contains everything I've learned about Moai in the last 6 months and has the nuts and bolts to build a game. Perhaps its not so much a template but a collection of library functions.

Level 1
A controllable birdy character and a block of treasure - the bird is welded initially - camera is freeform and can be panned with swipes - has parallax background

Level 2
A controllable rock character with a random hilly platform - camera is locked to main character - has no parallax background

Level 3
A controllable rock character in gravity less environment - sling bullets from the rock character

Player1 = If clicked on and is welded will break weld and change texture of sprite
Player2 = Print to debug when clicked on
Player3 = Launches up and to the right with a confetti explosion
Player4 = On click,drag and release launches bullet

add_actor(x,y,width,height,density,friction,restitution,actorname,treasure,bodytype,texture_image,particle_texture,smoke_texture,welded,health,points,velocityx,velocityy)

add-actor is the core of everything and has the following attributes
x,y,width,height - size
density,friction,restitution - box2d attributes
actorname=name used in collisions
treasure=??
bodytype=box2d attribute
texture_image=tiled image of the sprite
particle_texture=image to use when explode is called
smoke_texture=image to use when smoke is called
welded=box2d attribute
health=amount of impact damage required before the actor is destroyed
points=number of points when actor is destroyed
velocityx,velocityy=box2 atribute to set initial velocity of actor

Round up of the levels created
Level1 - Platform demo with hud buttons for movement - jumping is only allowed when the character is not falling
Level2 - Mashup level - hud button move rock around - up fires a skull in the air - clicking on character creates diamond
Level3 - Asteroids - left and right rotate ship - forward moves forward in direction - clicking on ship launches bullet
Level4 - Space Invaders merge with Breakout - left and right move ship - up launches bullet - blocks are fixed until enough damage looses them
Level5 - Angry Birds type launch and smash
Level6 - Simple click and destroy blocks - similar blocks destroy each other - no hud
