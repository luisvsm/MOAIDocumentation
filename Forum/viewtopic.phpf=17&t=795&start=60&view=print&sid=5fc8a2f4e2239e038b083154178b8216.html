Posted: Tue Jan 29, 2013 8:46 am
by ibisum
I'd really rather continue working on adding Camera support this week, so please .. have at it!

You can see my work here, in case you haven't already: https://github.com/seclorum/moai-dev

Specifically, I modified the Android template sources to include the relevant keyboard handling code, and as well added a TextView in the Android layout which sits outside the view of the main window - this is key (pun intended) because it is the TextView that gets the IME event messages and passes them through to the Lua VM side of things .. you should pay close attention to the way that this view is instantiated and set to be outside the main view, with Margin settings.

With regards to my code, there were some changes in the Android host done by Zipline after I had already finished this work, specifically with regards to locking - so you can't just copy/paste my code from the Android host, alas - but it should be enough to give you a full idea of what it takes to make it work. Also I suggest that you should use the moaigui framework for testing, as this is the best way to see whats going on so far .. at least, its what I used while adding MOAIKeyboardAndroid support.

Then, on the flipside, there is the MOAIKeyboardAndroid.cpp/.h code, which by now should be very familiar to you - this is something you probably can just copy from my fork.

Anyway, looking forward to the next MOAI release! If you need help or want some testing, just say the word ..
