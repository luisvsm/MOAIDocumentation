I am a begginer in MOAI, Rapanui and mobile development.

1) Using RAPANUI, I need create some squares
(with the command square1 = RNFactory.createRect)
and then paint them with a gradient. Is there a ready resource in RAPANUI for this ?

2) If the answer to the first question is no, then I would like to adapt a LUA routine
which runs very well on Corona SDK, but for that I need to know how to draw a line with the RAPANUI
(in Corona SDK, the command is to create lines line1 = display.newLine ()).

3) If it is not possible to implement the solutions mentioned in (1) and (2),
I would use the API mentioned in:

/forums/post4657.html

With this API, I made a board, and I painted each square with a gradient,
using the following commands (this is just a sample):

local colors = {"# FF0000", "# FFCC00", "# 009900", "# 0099CC"}
Square1 Meshes2D.newRect = (x1, y1, x2, y2, colors)
mylayer: insertProp (Square1)



After creating the board, I created an image with the command
image1 = RNFactory.createImage ("imagem1.png")

However, the "image1" is "below" the board, and
I need to draw image1 OVER the board.

I think the reason is that the RAPANUI creates its own layer (Screen)
while in the above commands I have my own layer, called "mylayer."

Please see the problem in the following link (the "image1" is the turtle under
the board):

http://img90.imageshack.us/img90/5081/imagesample.png

I put this new funcion in the module "RNFActory.lua", in my rapanui-sdk folder (I think this new routine will be incorporated into the "official" version, in the future).
