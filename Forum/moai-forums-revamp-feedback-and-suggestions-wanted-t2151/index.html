MOAI Forums Revamp: Feedback and Suggestions Wanted

by naturally » Tue Nov 12, 2013 5:40 pm

Howdy MOAI, recently I was given the opportunity to help manage the forums as I've had some ideas on how to improve and expand them. Before I do anything I'd like to run my ideas by you guys (and girls), get your feedback, and mull it over before implementing anything.

This is the current forum layout:

CURRENT
-----------------

Moai Development Forums
- Moai SDK
- - Community Contributions
- Moai Cloud
- New Users
- Bug Reports

MOAI Community Forums
- Moai Roadmap
- Made With Moai


Proposed layout:

PROPOSED
------------------

MOAI DEVELOPMENT FORUMS:
- Moai SDK
- New Users
- Bug Reports

MOAI COMMUNITY FORUMS:
- Community Contributions
- - Flower / Hanappe
- Moai Roadmap
- - Moai Cloud
- Made With Moai

PLATFORM SPECIFICS:
- Android Devices
- iPhone / iPad / OSX
- Windows 8 / 7 / Vista / XP / Phone
- Linux 64bit / 32bit
- Chrome / Blackberry / HTML JS / Other

THIRD PARTY LIBARIES AND PLUGINS:
- Physics: Box2D / Chipmunk
- Sound: FMOD / Untz / etc
- Advertising and In-App Purchases

MORE:
- General Game Development
- Off Topic


Tasks will also include moving old threads into relevant forums, for search/archival purposes. I figure such an outstanding framework such as Flower deserves its own subforum (at least) and other frameworks+forks could have subforums as well. What do you think? Too many? Not enough? Too confusing? Keep in mind a forum doesn't have to be highly active or populated to be its own forum. If there is enough delineation it is worth separating the topics. My main goal is to diffuse the current MOAI SDK forum since it has turned into a catch-all for everything. Any suggestions welcome.

cheers, dave

by davek » Wed Nov 13, 2013 1:36 pm

Do you really want high activity in a forum like "Bug Reports", though?

I don't think people really expect every specialized subforum to be highly active. If we are trying to trick people into joining MOAI by having high activity in a few forums, thats not really the best strategy to lure developers, especially if it harms the actual users by bogging them down. Look at Box2d, it's still super popular, has lots of subforums, but not all of them are exploding... I doubt anyone has abandoned Box2d because their Scala subforum isn't active, for example. We already have low-activity forums like MOAI Cloud, so anyone looking for an SDK based on unanimous subforum activity is already long gone.

Anyways, relying on search for content organization is a flaw... that's like having a wiki with no TOC, if you take my analogy!
