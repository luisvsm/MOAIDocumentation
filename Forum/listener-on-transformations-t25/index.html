by patrick » Sat Apr 16, 2011 1:03 pm

A pretty typical construct I've seen is to spin lock on the isBusy() method of the actions that the animation helpers return:

function main ()
    print ( 'begin' )
    action = prop:moveRot ( 360, 1.5 )
    while action:isBusy () do coroutine.yield () end
    print ( 'end' )
end
thread = MOAIThread.new ()
thread:run ( main )

Anything that derives from MOAIAction has a few methods to check status: isBusy, isActive and isDone. isActive tells you if the action is currently in the action tree (and will receive updates). isDone tells you if the action has more work remaining. Some action types are never 'done.' isBusy tells you if the action is both in the action tree and not done.

There is a callback on MOAITimer that fires every time a loop is completed. I could certainly add one to MOAIEaseDriver (what the anim helpers return). Now that I think about it, I could change MOAIEaseDriver to subclass from MOAITimer, if you need more control over it. Could also add status change callbacks to all actions...

Let me know (in general) if you need more callbacks. I've been leaning toward polling for most things with callbacks used sparingly, but there are a few objects (like the input sensors) that support both callbacks and polling.

Listener on transformations

by patrick » Thu Apr 21, 2011 2:21 pm

OK, the new build is up. There's a class called MOAIEventSource that is a mix-in for any objects that need to publish events. You just register your handler using the event ID. Right now the only class that uses this new system is MOAIAction, and the only event it publishes is MOAIAction.EVENT_STOP. So:


action = prop:moveRot ( 360, 1.5 )

action:setListener ( MOAIAction.EVENT_STOP, function () print ( 'end' ) end )

I'll be converting some of the other callbacks over to use this system (though probably not the input device callbacks). In the mean time, if you need more events let me know.

Josh or I post a more philosophical discussion of callbacks vs. polling later. We've done a lot of internal experimenting with both and have Opinions.
