Thanks a lot for the great work ! I have no more excuses not to finish my game this time

Just a note. I'm using arch linux 32 bits and I had to add pthread, rt, ssl and crypto to the linker to successfully compile moai. My src/hosts/CMakeLists.txt now looks like this:
Code: Select all
  1. ...

  2. target_link_libraries(gluthost

  3.     aku

  4.     moaicore

  5.     utf8

  6.     jansson

  7.     pthread

  8.     rt

  9.     ssl

  10.     crypto

  11.     ${OPENGL_LIBRARIES}

  12.     ${GLUT_LIBRARIES}

  13.     GLEW

  14. )


I also had problems compiling MOAI with Chipmunk support, as the newest chipmunk's github code is completely out of sync with MOAI's lua wrappers, and I couldn't find a suitable precompiled package. Anyway, to remove Chipmunk support I changed the following files:

src/config-default/moaicore-config.h
Code: Select all
  1. #ifndef USE_CHIPMUNK

  2.         #define USE_CHIPMUNK 0

  3. #endif



Removed all MOAICp source files in src/moai-core/CMakeLists.txt and removed chipmunk from the linker:
Code: Select all
  1. add_library(moaicore STATIC

  2.     MOAI...

  3.     # MOAICpArbiter.cpp

  4.     # MOAICpXX.XX

  5.     ...

  6. )

  7.  

  8. target_link_libraries(moaicore

  9.     Box2D

  10.     tinyxml

  11.     jpeg

  12.     png

  13.     ${CURL_LIBRARIES}

  14.     ${FREETYPE_LIBRARIES}

  15. )



Finally I commented out the MOAICpSpace include in src/moaicore/MOAILayer.cpp
Code: Select all
  1. #include <moaicore/MOAICamera.h>

  2. // #include <moaicore/MOAICpSpace.h>

  3. #include <moaicore/MOAIDebugLines.h>



I think it would be nice to have a CMake option to enable/disable chipmunk support. What do you think ?
Thanks again !
