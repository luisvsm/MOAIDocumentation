MOAI on Debian

by spacepluk » Wed May 02, 2012 11:40 pm

UPDATE: It seems that for now it only works on 32bit systems.

Hi there,
I've updated the old cmake files to work with 1.0 and it seems to be working good so far. I've uploaded the changes to my fork in github. For some of the dependencies it's using the system libraries, so you'll need to install some packages to be able to compile.

Quick guide for Debian (wheezy):
# Install the dependencies:
sudo aptitude install freeglut3-dev libglew-dev libglu1-mesa-dev libxmu-dev libxi-dev chipmunk-dev libjpeg8-dev libpng-dev libfreetype6-dev liblua5.1-0-dev libjansson-dev libtinyxml-dev libcurl4-openssl-dev zlib1g-dev libexpat1-dev cmake build-essential

# Clone the repository:
git clone git://github.com/spacepluk/moai-beta.git
cd moai-beta
git checkout linux-cmake

# Compile:
mkdir build
cd build
cmake ..
make -j4


Have fun

Okay - for those folks reading this thread, we met in IRC and have been working out the problems I have had - which are related to the fact I'm running Ubuntu 12.04 - AMD64bit version, and haven't had the chance to set up the 32-bit development environment, including headers, yet. This has been a bit of a dead-end, because its not so easy to have a full-blown 32-bit development environment under Ubuntu 12.04-amd64, I guess, unless we go the full-blown chroot environment path. And I don't wanna do that right now, but in case someone else works it out, please let us know.

So now instead I am trying to install in an Ubuntu-11.04 VM (32bit), and for that I found that both libjansson-dev and libchipmunk-dev are not in the default repositories for 11.04, but are available through PPA's:

Code: Select all
  1.  

  2. sudo add-apt-repository ppa:josebagar/ppa

  3. sudo add-apt-repository ppa:petri/ppa

  4. sudo apt-get update

  5.  



After those three lines, I was able to install the packages smoothly as listed by spacepluk at the top of the thread .. fyi.
