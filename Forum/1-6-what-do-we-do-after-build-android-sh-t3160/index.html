Re: 1.6: What do we do after build-android.sh?

by ibisum » Thu Jun 18, 2015 11:44 am

As far as I can see there is no good reason not to be using moaiutil from now on (MOAI 1.6) to create new host projects ..

Code: Select all
  1.  

  2. $ cd moai-dev

  3. $ export PATH=`pwd`/util:$PATH

  4. $ mkdir NewProjectFolder && cd NewProjectFolder

  5. $ moaiutil host init

  6. $ #edit hostconfig.lua and make sure it looks right

  7. $ moaiutil host create android

  8. $ cd hosts/android # Edit settings, etc.

  9.  



So the general 'flow' is: clone MOAI, build the libraries for all host platforms (see bin/*.sh scripts), set up PATH as per above, use moaiutil to create new host projects ...

by monomon » Mon Jun 13, 2016 2:44 pm

It is worth pointing out that moaiutil has been renamed to pito.

Running util/pito without arguments provides good usage information.
It goes along the lines of:

Code: Select all
  1. export PATH=moai-dev/util:$PATH

  2. mkdir myHost

  3. cd myHost

  4. pito init

  5. -- edit hostconfig.lua ...

  6. pito host create android-gradle

  7. pito host build android-gradle



(haven't looked at what the other possible targets are besides gradle, since op was asking about alternatives)
