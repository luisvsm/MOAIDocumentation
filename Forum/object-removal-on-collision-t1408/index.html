Re: Object removal on collision

by stun » Thu Nov 22, 2012 7:33 pm

Have you found the answer yet? It's not an uncommon question about box2d if you google. If not I can probably help since I encountered the same issue learning Box2d a month ago! Caveat emptor: I'm assuming Rapanui is using box2d underneath. I could be wrong, but it seems likely!

Problem: You cannot change the box2d 'world state' in the collision handlers. You cannot add/remove objects, nor I believe change any of their properties (position, rotation, velocity, density, mass etc).

Reason: The box2d 'collisionHandlers' are poorly named. What they should be called is 'collisionNotification' perhaps. The collision handlers are called by box2d during it's simulation update steps. They may also be called multiple times on the same object (if it collides with multiple things). As they're called during the box2d simulation you cannot safely change any box2d body or fixture properties, nor add or remove objects because this will potentially break (or even crash) the simulation.

Solution: The solution is to defer your changes to the box2d world until after the simulation update completes. Fortunately in Lua this is trivial to do. I just do something like the following. This is MOAI pseudo code btw, since I'm not using Rapanui, but the principle still applies.

Code: Select all
  1.  

  2. -- The box2d collision handler for a 'ship'.

  3. function shipCollision(phase, shipFix, otherFix, arbiter)

  4.   if phase == MOAIBox2DArbiter.PRE_SOLVE then

  5.     game:defer(

  6.       function()

  7.         -- Ship has collided with something, kill it.

  8.         shipExplode(shipFix)

  9.       end

  10.     )

  11.   end  

  12. end

  13.  

  14. -- The 'defer' function that just builds a table of lambda functions to call later

  15. function game:defer(func)

  16.   table.insert(self._deferred, func)

  17. end

  18.  

  19. -- The game 'update' function which is called in a seperate MOAIThread after MOAISim has yielded back to it.

  20. funtion game:update()

  21.   -- Just call every deferred lambda function since the last update, in the order they arrived

  22.   for i = 1, #self._deferred do

  23.     self._deferred[i]()

  24.   end

  25. end

  26.  



Hope that helps.
