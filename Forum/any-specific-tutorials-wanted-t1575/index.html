That seems like a really good candidate. The way I usually explain it:

In lua, : is purely a syntactic convenience. foo:bar(a, b) is exactly the same as foo.bar(foo, a, b). If a class member function's definition lists its first argument as being of its own type, it is probably intended to be called with a colon.

by seebs » Sat Feb 02, 2013 12:13 pm

Muzz wrote:So using a colon passes the first argument as self? true....

Passes self as the first argument, yes.

Code: Select all
  1.  

  2. o = { data = "foo" }

  3.  

  4. function o.same_1(self)

  5.   print(self.data)

  6. end

  7.  

  8. function o:same2()

  9.   print(self.data)

  10. end

  11.  

  12. o.same_1(o)

  13. o:same1()

  14. o.same_2(o)

  15. o:same2()

  16.  



All four calls are identical. So are the two functions.

Oh, another thing I'd like to see more on:

The relationship between simulator loops and "frames".

Code: Select all
  1. MOAISim.openWindow("test", 480, 320)

  2. local viewport = MOAIViewport.new()

  3. viewport:setSize(480, 320)

  4. viewport:setScale(480, 320)

  5. layer = MOAILayer2D.new()

  6. layer:setViewport(viewport)

  7. MOAIRenderMgr.setRenderTable( { layer } )

  8.  

  9. local sim_cycles = 0

  10.  

  11. function every_frame()

  12.   while true do

  13.     coroutine.yield()

  14.     sim_cycles = sim_cycles + 1

  15.   end

  16. end

  17. frames = MOAICoroutine.new()

  18. frames:run(every_frame)

  19. function do_draw()

  20.   print(string.format("%d cycles, %.1f fps", sim_cycles, MOAISim.getPerformance()))

  21.   sim_cycles = 0

  22. end

  23. local sd = MOAIScriptDeck.new()

  24. sd:setDrawCallback(do_draw)

  25. sd:setRect(-64, -64, 64, 64)

  26. local prop = MOAIProp2D.new()

  27. prop:setDeck(sd)

  28. layer:insertProp(prop)



About 80-90% of the time, the output from this says "1 cycles", but occasionally I see either 0 or 2.

I am probably missing some useful concept here.
