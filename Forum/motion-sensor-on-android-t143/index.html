@AndyL I am also able to get sensor events in Lua. Check MoaiActivity.java in your Android host and make sure that the onSensorChanged callback is getting called and that it is successfully calling AKUEnqueueLevelEvent.

@milleniummike I'm not sure what version of the SDK you are using, but in my Android testing, the proper function signature for level event callbacks is "onLevelEvent ( x, y, z )". I'm working out of a development branch, so it's possible that something has changed recently that has altered the callback signature(s).

by sean » Sat Feb 11, 2012 1:03 am

So, I tested on Android and I'm getting callbacks CONSTANTLY. This may be device-dependent. Further, you can tweek the update rate in MoaiActivity.java. The samples host uses SensorManager.SENSOR_DELAY_NORMAL when it registers the listener, but Android supports a couple of faster rates - SENSOR_DELAY_FASTEST and SENSOR_DELAY_GAME.

Change the line:

accel.updateInterval = mAnimInterval

to

accel.updateInterval = mAnimInterval/60

to get one update per frame.

anddrewscott wrote:would be better memory wise (multiplying is better anyway + 1/60 is recurring decimal)


Moai (Lua) numbers are doubles. Every number uses the exact same amount of storage, regardless of value (be it 0.016, 1/60, 982349848439289234.6666[repeating], 0, etc.). Perhaps you're confusing the storage required for a number with the storage required for a string representation of a number?
