I just discovered 2 bugs with MOAIGfxDevice. The first bug is that MOAIDraw.drawPoints is only drawing the first point even if > 1 points are passed in. The code to reproduce this is:

Code: Select all
  1.  

  2. MOAISim.openWindow ( "test", 320, 480 )

  3.  

  4. viewport = MOAIViewport.new ()

  5. viewport:setSize ( 320, 480 )

  6. viewport:setScale ( 320, 480 )

  7.  

  8. layer = MOAILayer.new ()

  9. layer:setViewport ( viewport )

  10. MOAISim.pushRenderPass ( layer )

  11.  

  12. local points = {}

  13. for i=1,200 do

  14.         table.insert ( points, math.random(-160, 160) )

  15.         table.insert ( points, math.random(-240, 240) )

  16. end

  17.  

  18. function onDraw ()

  19.  

  20.         MOAIGfxDevice.setPenColor ( 1, 0, 0, 1 )

  21.         MOAIDraw.fillCircle ( 0, 0, 64, 32 )

  22.  

  23.         MOAIGfxDevice.setPenColor ( 1, 0, 0, 1 )

  24.         MOAIGfxDevice.setPenWidth ( 2 )

  25.         MOAIGfxDevice.setPointSize ( 5 )

  26.         MOAIDraw.drawPoints(points)

  27.  

  28. end

  29.  

  30. scriptDeck = MOAIScriptDeck.new ()

  31. scriptDeck:setRect ( -64, -64, 64, 64 )

  32. scriptDeck:setDrawCallback ( onDraw )

  33.  

  34. prop = MOAIProp.new ()

  35. prop:setDeck ( scriptDeck )

  36. layer:insertProp ( prop )

  37.  



I tracked down the issue to MOAIGfxDevice::SetPrimType in src/moaicore/gfx/MOAIGfxDevice.cpp

As is:

  3.  

  4.         if ( this->mPrimType != primType ) {

  5.  

  6.                 this->Flush ();

  7.                 this->mPrimType = primType;

  9.                 switch ( primType ) {

  14.

Fixed:
Code: Select all
  1.  

  2. void MOAIGfxDevice::SetPrimType ( u32 primType ) {

  3.  

  4.         if ( this->mPrimType != primType ) {

  5.  

  6.                 this->Flush ();

  7.                 this->mPrimType = primType;

  8.  

  9.                 switch ( primType ) {

  10.                

  11.                         case GL_LINES:

  12.                                 this->mPrimSize = 2;

  13.                                 break;

  14.                        

  15.                         case GL_TRIANGLES:

  16.                                 this->mPrimSize = 3;

  17.                                 break;

  18.                        

  19.                         case GL_POINTS:

  20.                         case GL_LINE_LOOP:

  21.                         case GL_LINE_STRIP:

  22.                         case GL_TRIANGLE_FAN:

  23.                         case GL_TRIANGLE_STRIP:

  24.                         default:

  25.                                 this->mPrimSize = 0;

  26.                                 break;

  27.                 }

  28.         }

  29. }



Note that this->mPrimSize should be 0 for GL_POINTS.

The second bug is a bit harder to spot. In fact, if the only primitive you draw is a point array then it works. This is because MOAIGfxDevice->mPrimType is being initialized as GL_POINTS but inside of MOAIGfxDevice::SetPrimType a check is being made

Code: Select all
  1.  

  2. if ( this->mPrimType != primType ) {    // line 1157 of MOAIGfxDevice.cpp

  3.  



that causes critical code to be skipped over.

The solution is to initialize mPrimType to a truly invalid value like -1

Code: Select all
  1.  

  2. mPrimType ( -1 ),       // line 748 of MOAIGfxDevice.cpp

  3.  



Be aware that the type of mPrimType is GLenum and thus you will get signed/unsigned warnings if you put in a -1.
