Code:
  1. module ( ..., package.seeall )

  2.  

  3. function new(pProgramName, pFPS)

  4.   -- Program State

  5.   local result = {}

  6.  

  7.   -- Global Settings

  8.   result.fps          = pFPS or 60

  9.   result.programName  = pProgramName or "test"

  10.   result.screenWidth  = MOAIEnvironment.screenWidth  or 1024

  11.   result.screenHeight = MOAIEnvironment.screenHeight or 1024

  12.   if result.screenWidth < result.screenHeight then

  13.     result.screenWidth, result.screenHeight = result.screenHeight, result.screenWidth

  14.   end

  15.  

  16.   -- Initialization

  17.   function result:init()

  18.     -- FPS

  19.     MOAISim.setStep( 1 / self.fps )

  20.  

  21.     -- Simulator Window

  22.     MOAISim.openWindow( self.programName, self.screenWidth, self.screenHeight )

  23.     print(              self.programName, self.screenWidth, self.screenHeight )

  24.  

  25.     -- Viewport

  26.     self.viewport = MOAIViewport.new ()

  27.     self.scaleX  = 2048

  28.     self.scaleY  = self.scaleX * self.screenHeight / self.screenWidth

  29.     self.viewport:setSize(  self.screenWidth, self.screenHeight )

  30.     self.viewport:setScale( self.scaleX,      self.scaleY       )

  31.  

  32.     -- Camera

  33.     self.camera = MOAICamera2D.new ()

  34.  

  35.     -- Done

  36.     return self

  37.   end

  38.  

  39.   -- Done

  40.   return result

  41. end

Moses wrote:
However, if you set an arbitrary resolution and enter fullscreen mode on a device with a different width / height everything is drawn offset.

You need to do some math to work with different aspect ratios. My game is running in landscape and the width of the screen is assumed to be fixed at "one screen". The art assumes a 1:1 square aspect ratio, but I reposition things vertically as needed. Here is an example:
Code:
  1. function loadButtonPlay(pState, pLayer, pPartition, pTileset, pScaleW, pScaleH)

  2.   local index            = 10

  3.   local x, y             = 0, (pState.scaleY / 2) - (pScaleH / 2)

  4.   local centerX, centerY = 0, 0

  5.   local result = loadButton( pState, pLayer, pPartition, pTileset,

  6.     index, x, y, centerX, centerY )

  7.   return result

  8. end


See line 3 where I set y.

getDeviceSize() was deprecated from MOAI 1.0 -> 1.1 and is going to be phased out. So you will see some samples that might still use it, but in 1.2 you have the MOAIEnvironment.screenWidth/Height constants, instead.

Be aware that these constants (well, most of the MOAIEnvironment table, actually) are not all properly filled out when working on MacOSX/Windows/Linux hosts - they're really only working on iOS/Android host targets right now - but this will probably be addressed in a near-future release (I haven't checked 1.2 to see if its the case with this version yet).
