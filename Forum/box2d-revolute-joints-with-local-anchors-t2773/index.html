Box2D Revolute Joints with local anchors

by stun » Mon Aug 18, 2014 12:39 am

I sometimes find it a little frustrating that the MOAI Box2D wrapper hides some useful details from you. I understand the motivation, which is to provide a simpler, cleaner interface to Box2D than Box2D's own API. Unfortunately sometimes that gets in the way.

Recently I was forced to implement a new MOAIBox2D:addRevoluteJointLocal function to allow Box2D revolute joint creation using 'local anchors' - that is anchors local to the relevant bodies, rather than specified in world coordinates for a single anchor as MOAIBox2D:addRevoluteJoint expects. Here's the snippet of code from moai-box2d/MOAIBox2DWorld.cpp

Code: Select all
  1. //----------------------------------------------------------------//

  2. /**     @name   addRevoluteJointLocal

  3.         @text   Create and add a joint to the world, using local anchors. See Box2D documentation.

  4.        

  5.         @in             MOAIBox2DWorld self

  6.         @in             MOAIBox2DBody bodyA

  7.         @in             MOAIBox2DBody bodyB

  8.         @in             number anchorA_X        in units, in world coordinates, converted to meters

  9.         @in             number anchorA_Y        in units, in world coordinates, converted to meters

  10.         @in             number anchorB_X        in units, in world coordinates, converted to meters

  11.         @in             number anchorB_Y        in units, in world coordinates, converted to meters

  12.         @out    MOAIBox2DJoint joint

  13. */

  14. int     MOAIBox2DWorld::_addRevoluteJointLocal( lua_State* L ) {

  15.         MOAI_LUA_SETUP ( MOAIBox2DWorld, "UUUNNNN" )

  16.        

  17.         if ( self->IsLocked ()) {

  18.                 MOAILog ( state, MOAILogMessages::MOAIBox2DWorld_IsLocked );

  19.                 return 0;

  20.         }

  21.        

  22.         MOAIBox2DBody* bodyA = state.GetLuaObject < MOAIBox2DBody >( 2, true );

  23.         MOAIBox2DBody* bodyB = state.GetLuaObject < MOAIBox2DBody >( 3, true );

  24.        

  25.         if ( !( bodyA && bodyB )) return 0;

  26.                

  27.         b2RevoluteJointDef jointDef;

  28.         jointDef.bodyA = bodyA->mBody;

  29.         jointDef.bodyB = bodyB->mBody;

  30.        

  31.         jointDef.localAnchorA.Set(

  32.                 state.GetValue < float >( 4, 0 ) * self->mUnitsToMeters,

  33.                 state.GetValue < float >( 5, 0 ) * self->mUnitsToMeters);

  34.                

  35.         jointDef.localAnchorB.Set(

  36.                 state.GetValue < float >( 6, 0 ) * self->mUnitsToMeters,

  37.                 state.GetValue < float >( 7, 0 ) * self->mUnitsToMeters);

  38.                

  39.         MOAIBox2DRevoluteJoint* joint = new MOAIBox2DRevoluteJoint ();

  40.         joint->SetJoint ( self->mWorld->CreateJoint ( &jointDef ));

  41.         joint->SetWorld ( self );

  42.         joint->LuaRetain ( bodyA );

  43.         joint->LuaRetain ( bodyB );

  44.         self->LuaRetain ( joint );

  45.        

  46.         joint->PushLuaUserdata ( state );

  47.         return 1;

  48. }

I found a good flow for quick patches is as follows:

Code: Select all
  1.  

  2. #assuming "upstream" is moai/moai-dev (main repo) and "origin" is stun/moai-dev (your fork)

  3. #you can always add upstream to your repo by running "git remote add \https://github.com/moai/moai-dev"

  4.  

  5. #grab a copy of main repo master and put it in a branch locally called pull_box2d_fix

  6. git fetch upstream master:pull_box2d_fix

  7.  

  8. #checkout the new branch

  9. git checkout pull_box2d_fix

  10.  

  11. #do the fix

  12. <cut & paste fix>

  13. git add <fixedfile>

  14. <test the fix>

  15.  

  16. #commit

  17. git commit -m "fix box2d revolute joints"

  18.  

  19. #push it to your github fork

  20. git push origin pull_box2d_fix

  21.  

  22. #jump on github and click the green thingy at the top to submit pull request against master on moai-dev

  23.  



This saves having to get your repo in order . Thanks again for the patch!
