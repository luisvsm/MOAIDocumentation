Re: Error Building: version_major and version_minor .h not f

by DanielWolf » Sat Sep 05, 2015 8:54 pm

The problem is that Moai version information used to live in "/src/config-default/". That's what the build scripts still expect. However, with commit 689087f691 (06/20/2015), Patrick seems to have moved the version information to "/util/sdk/". What's more, the file structure has changed as well, so this doesn't look line a trivial fix.

Is there anybody who understands both the build scripts and Patrick's new versioning system?
