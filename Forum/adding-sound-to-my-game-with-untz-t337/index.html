Adding Sound to my game with Untz

by bslayerw » Tue Jan 10, 2012 2:12 pm

Either there are some design flaws with the Untz audio system or I'm overlooking something. I'm going to dig deeper into Untz and see if I can't add these features…



- Ability to play the same sound multiple times without it restarting the sound. This seems like a design limitation in the audio mixer and how sounds manage state. Ideally when requesting to play a sound it could be added and be mixed. An example of this would be in my pinball game I'm porting over to MOAI. I have a "bumper" sound that plays at a different pitch based on how hard the ball hits the bumper. This is currently not possible with Untz unless I create a ton of different sounds effects and even then they might restart one that's already playing. In my own implementation (prior to porting) I'd use a circular buffer for all the OpenAL sources available and assign my sound data to one that's not in use, OpenAL handles the rest. 

- Ability to play sounds at a different pitch (AL_PITCH). As outlined in my example above. I do all sorts of crazy things with the sounds. For example my simple audio engine that's based on OpenAL (prior to porting to MOAI) I'd go into a "slo-mo" mode when activating a certain features. For a couple of seconds all physics is slowed down and all sounds have their pitch changes so play really slow. After a set time I put everything back to normal. 



input appreciated.

Adding Sound to my game with Untz

by eric » Wed Jan 11, 2012 5:39 pm

The way to play same sound multiple times would be to create multiple handles pointing to the same memory. Under the hood, they share the data, so you do not take a memory hit by doing this. 




local sound1 = MOAIUntzSound.new ()

sound1:load ( "mySound.wav" )

local sound2 = MOAIUntzSound.new ()

sound2:load ( "mySound.wav" )




The above will give you the ability to play the same sound overlapped multiple times without taking a memory hit. The pitch shifting feature is not yet possible in Untz. We will ask the Retronyms guys if this is something they would like to add.
